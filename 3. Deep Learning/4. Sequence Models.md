## 1. Sequential Data Modeling Intuition

### a. Intuition

Imagine reading a sentence: *"The bank by the river was flooded."*

A traditional feedforward neural network would look at each word in isolation. It sees "bank" and has no idea whether that means a financial institution or a riverbank. But you, as a reader, use **context** — the word "river" from earlier informs your interpretation of "bank."

This is the fundamental challenge of **sequential data**: each element's meaning depends on what came before it (and sometimes after it).

Sequential data appears everywhere:

| Domain | Examples |
|---|---|
| NLP | Sentences, documents, conversations |
| Finance | Stock prices, transaction histories |
| IoT / Energy | Sensor readings, electricity demand |
| Audio | Speech waveforms, music |
| Healthcare | ECGs, patient vitals over time |
| Video | Frame sequences |

**The key insight**: Order matters. Shuffling the elements destroys meaning.

### b. Mathematical Insight

For a sequence `x = [x₁, x₂, ..., xₜ]`, we want to model:

```
P(xₜ | x₁, x₂, ..., xₜ₋₁)
```

The joint probability of the sequence becomes:

```
P(x₁, x₂, ..., xₜ) = ∏ P(xₜ | x₁, ..., xₜ₋₁)
```

This is the **autoregressive** view of sequences. A good sequence model must efficiently represent this conditional dependence without requiring the model to grow with sequence length.

### c. How It Works (Step-by-Step)

1. **Represent each time step** as a vector (word embedding, sensor reading, etc.)
2. **Feed elements one at a time** into the model, in order
3. **Maintain a "memory" state** that summarizes everything seen so far
4. **Update memory** at each step using the new input
5. **Produce output** at each step (or only at the end, depending on the task)

### d. Visual Representation

**Types of sequential tasks:**

```
Many-to-One (Sentiment):        Many-to-Many (Translation):
x₁ → x₂ → x₃ → x₄ → [y]       x₁ → x₂ → [y₁ → y₂ → y₃]

One-to-Many (Music generation): Many-to-Many Synced (POS tagging):
[x] → y₁ → y₂ → y₃             x₁ → x₂ → x₃
                                 ↓    ↓    ↓
                                 y₁   y₂   y₃
```

### e. Python Implementation

```python
import numpy as np
import torch
import torch.nn as nn
from torch.utils.data import Dataset, DataLoader

# ─────────────────────────────────────────────
# Utility: Create a synthetic sequence dataset
# ─────────────────────────────────────────────
def generate_sine_sequence(n_samples=1000, seq_len=50, noise=0.05):
    """
    Generate a noisy sine wave dataset for sequence modeling demos.
    Task: predict the next value given a window of past values.
    """
    t = np.linspace(0, 4 * np.pi, n_samples + seq_len)
    series = np.sin(t) + noise * np.random.randn(len(t))

    X, y = [], []
    for i in range(n_samples):
        X.append(series[i : i + seq_len])
        y.append(series[i + seq_len])

    return np.array(X, dtype=np.float32), np.array(y, dtype=np.float32)


class SequenceDataset(Dataset):
    def __init__(self, X, y):
        # X: (N, seq_len), y: (N,)
        self.X = torch.tensor(X).unsqueeze(-1)  # → (N, seq_len, 1)
        self.y = torch.tensor(y).unsqueeze(-1)  # → (N, 1)

    def __len__(self):
        return len(self.X)

    def __getitem__(self, idx):
        return self.X[idx], self.y[idx]


# Quick demo
X, y = generate_sine_sequence()
dataset = SequenceDataset(X, y)
loader  = DataLoader(dataset, batch_size=32, shuffle=True)

sample_x, sample_y = next(iter(loader))
print(f"Input shape:  {sample_x.shape}")   # (32, 50, 1)
print(f"Target shape: {sample_y.shape}")   # (32, 1)
```

### f. When to Use Sequence Models

**Use when:**
- Data has a meaningful temporal or positional order
- Context from previous steps matters for current prediction
- You're working with variable-length inputs
- NLP, time-series forecasting, speech recognition, anomaly detection

**Avoid or rethink when:**
- Order is meaningless (tabular data → use gradient boosting)
- Sequences are very long (>500 steps) → consider Transformers instead
- You need real-time edge inference with strict latency → simpler models win

## 2. Recurrent Neural Networks (RNNs)

### a. Intuition

Think of an RNN as a person reading a book while taking notes. At each page, they:
1. Read the new content (input `xₜ`)
2. Look at their notes from the previous page (hidden state `hₜ₋₁`)
3. Update their notes (new hidden state `hₜ`)
4. Optionally write a summary (output `yₜ`)

The **notes** (`hₜ`) are the RNN's memory. The same set of weights is applied at **every time step** — this is called **weight sharing across time**.

### b. Mathematical Insight

At each time step `t`:

```
hₜ = tanh(Wₓₕ · xₜ + Wₕₕ · hₜ₋₁ + bₕ)
yₜ = Wₕᵧ · hₜ + bᵧ
```

Where:
- `xₜ ∈ ℝᵈ` — input at time step t (d-dimensional)
- `hₜ ∈ ℝʰ` — hidden state (h-dimensional memory)
- `Wₓₕ ∈ ℝʰˣᵈ` — input-to-hidden weights
- `Wₕₕ ∈ ℝʰˣʰ` — hidden-to-hidden weights (the recurrent connection)
- `tanh` — squashes values to [-1, 1], centered activation

**Training via Backpropagation Through Time (BPTT):**

The loss gradient flows backward through time steps:

```
∂L/∂Wₕₕ = Σₜ (∂Lₜ/∂hₜ · ∂hₜ/∂Wₕₕ)
```

The gradient at step `t` depends on the product of Jacobians across all prior steps — this is the root cause of the **vanishing/exploding gradient** problem.

### c. How It Works (Step-by-Step)

1. **Initialize** hidden state `h₀ = 0` (or learned)
2. **For each time step t = 1 to T:**
   - Compute new hidden state: `hₜ = tanh(Wₓₕ xₜ + Wₕₕ hₜ₋₁ + b)`
   - (Optionally) compute output: `yₜ = Wₕᵧ hₜ`
3. **After all steps**, use final `hₜ` or collect all `yₜ` depending on task
4. **Compute loss** against targets
5. **Backpropagate** gradients through all time steps (BPTT)
6. **Update weights** with optimizer

### d. Visual Representation

**Unrolled RNN through time:**

```
         x₁        x₂        x₃        x₄
         │         │         │         │
         ▼         ▼         ▼         ▼
h₀ ──► [RNN] ──► [RNN] ──► [RNN] ──► [RNN] ──► hₜ
         │         │         │         │
         ▼         ▼         ▼         ▼
        y₁        y₂        y₃        y₄

Each [RNN] cell applies: hₜ = tanh(Wₓₕxₜ + Wₕₕhₜ₋₁ + b)
Same weights W at every step → weight sharing
```

**Vanishing gradient problem visualized:**

```
t=100  t=99  t=98  ...  t=2   t=1
  ◄─────────────────────────────── Gradient flow
  0.001  0.01  0.1  ...  0.9   1.0   ← Gradients shrink exponentially
```

### e. Python Implementation

```python
import torch
import torch.nn as nn


# ─────────────────────────────────────────────
# Vanilla RNN from scratch (educational)
# ─────────────────────────────────────────────
class VanillaRNNCell(nn.Module):
    """
    A single RNN cell — the building block.
    Computes: hₜ = tanh(Wxh·xₜ + Whh·hₜ₋₁ + b)
    """
    def __init__(self, input_size: int, hidden_size: int):
        super().__init__()
        self.hidden_size = hidden_size
        # Combined weight matrix for efficiency
        self.W_xh = nn.Linear(input_size, hidden_size, bias=False)
        self.W_hh = nn.Linear(hidden_size, hidden_size, bias=True)
        self.tanh  = nn.Tanh()

    def forward(self, x_t, h_prev):
        # x_t:   (batch, input_size)
        # h_prev: (batch, hidden_size)
        return self.tanh(self.W_xh(x_t) + self.W_hh(h_prev))


class VanillaRNN(nn.Module):
    """
    Full RNN that unrolls a VanillaRNNCell across time.
    Returns: all hidden states + final hidden state
    """
    def __init__(self, input_size: int, hidden_size: int, output_size: int):
        super().__init__()
        self.hidden_size = hidden_size
        self.cell = VanillaRNNCell(input_size, hidden_size)
        self.fc   = nn.Linear(hidden_size, output_size)

    def forward(self, x, h0=None):
        # x: (batch, seq_len, input_size)
        batch_size, seq_len, _ = x.shape

        # Initialize hidden state
        if h0 is None:
            h0 = torch.zeros(batch_size, self.hidden_size, device=x.device)

        h_t = h0
        hidden_states = []

        for t in range(seq_len):
            h_t = self.cell(x[:, t, :], h_t)
            hidden_states.append(h_t.unsqueeze(1))

        # Stack all hidden states: (batch, seq_len, hidden_size)
        all_hidden = torch.cat(hidden_states, dim=1)

        # Output: use last hidden state for many-to-one tasks
        out = self.fc(h_t)
        return out, all_hidden, h_t


# ─────────────────────────────────────────────
# Using PyTorch built-in RNN (production-ready)
# ─────────────────────────────────────────────
class RNNForecaster(nn.Module):
    """
    Production-style RNN for univariate time-series forecasting.
    """
    def __init__(self, input_size=1, hidden_size=64, num_layers=1,
                 output_size=1, dropout=0.0):
        super().__init__()
        self.rnn = nn.RNN(
            input_size=input_size,
            hidden_size=hidden_size,
            num_layers=num_layers,
            batch_first=True,       # Input shape: (batch, seq, features)
            dropout=dropout if num_layers > 1 else 0.0,
            nonlinearity='tanh'
        )
        self.fc = nn.Linear(hidden_size, output_size)

    def forward(self, x):
        # x: (batch, seq_len, input_size)
        out, h_n = self.rnn(x)
        # out: (batch, seq_len, hidden_size)
        # h_n: (num_layers, batch, hidden_size)

        # Use the last time step output
        last_out = out[:, -1, :]       # (batch, hidden_size)
        return self.fc(last_out)       # (batch, output_size)


# ─────────────────────────────────────────────
# Training loop
# ─────────────────────────────────────────────
def train_model(model, loader, epochs=20, lr=1e-3):
    optimizer = torch.optim.Adam(model.parameters(), lr=lr)
    criterion = nn.MSELoss()
    model.train()

    for epoch in range(epochs):
        total_loss = 0.0
        for X_batch, y_batch in loader:
            optimizer.zero_grad()
            preds = model(X_batch)
            loss  = criterion(preds, y_batch)
            loss.backward()
            # Clip gradients to prevent exploding gradients
            nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
            optimizer.step()
            total_loss += loss.item()

        if (epoch + 1) % 5 == 0:
            avg_loss = total_loss / len(loader)
            print(f"Epoch [{epoch+1}/{epochs}] Loss: {avg_loss:.4f}")

    return model


# Demo run
X, y = generate_sine_sequence()
dataset = SequenceDataset(X, y)
loader  = DataLoader(dataset, batch_size=32, shuffle=True)

model = RNNForecaster(input_size=1, hidden_size=64)
model = train_model(model, loader, epochs=20)
```

| Scenario | Recommendation |
|---|---|
| Short sequences (<20 steps) | ✅ Vanilla RNN can work |
| Long sequences (>50 steps) | ❌ Vanishing gradient kills performance — use LSTM/GRU |
| Variable-length input | ✅ RNN handles naturally with masking |
| Educational / debugging | ✅ Great for understanding recurrence |
| Production NLP | ❌ Use Transformers or at minimum LSTM |

### g. Key Hyperparameters

| Parameter | Typical Range | Effect |
|---|---|---|
| `hidden_size` | 32 – 512 | More hidden units = more capacity, slower |
| `num_layers` | 1 – 3 | Depth = abstract features, risk of vanishing grad |
| `dropout` | 0.1 – 0.5 | Regularization (only between layers, not on last) |
| Learning rate | 1e-4 – 1e-2 | Adam usually works; lower for longer sequences |
| Gradient clipping | 0.5 – 5.0 | Critical for stability — always use it |

## 3. Long Short-Term Memory (LSTM)

### a. Intuition

Vanilla RNNs forget things over long sequences. The gradient shrinks at each time step, making it nearly impossible to remember something from 100 steps ago.

The LSTM, introduced by Hochreiter & Schmidhuber (1997), solves this with a **cell state** — a separate "conveyor belt" that runs straight through time, with minimal interference. Information can ride this belt from the beginning to the end of a sequence almost unchanged.

Think of the LSTM as a sophisticated note-taking system with three decisions to make at each step:
1. **Forget Gate**: "What from my old notes should I erase?"
2. **Input Gate**: "What new information should I write down?"
3. **Output Gate**: "What from my notes should I share with the next step?"

### b. Mathematical Insight

At each time step `t`, given input `xₜ` and previous state `(hₜ₋₁, cₜ₋₁)`:

```
fₜ = σ(Wf · [hₜ₋₁, xₜ] + bf)          # Forget gate
iₜ = σ(Wi · [hₜ₋₁, xₜ] + bi)          # Input gate
c̃ₜ = tanh(Wc · [hₜ₋₁, xₜ] + bc)      # Candidate cell state
cₜ = fₜ ⊙ cₜ₋₁ + iₜ ⊙ c̃ₜ             # Cell state update
oₜ = σ(Wo · [hₜ₋₁, xₜ] + bo)          # Output gate
hₜ = oₜ ⊙ tanh(cₜ)                    # Hidden state
```

Where:
- `σ` is the sigmoid function → squashes to [0,1] (acts as a gate, 0=closed, 1=open)
- `tanh` → squashes to [-1,1] (content values)
- `⊙` is element-wise multiplication
- `[hₜ₋₁, xₜ]` means concatenation

**Why this fixes vanishing gradients:**

The cell state update `cₜ = fₜ ⊙ cₜ₋₁ + iₜ ⊙ c̃ₜ` is **additive**. The gradient through this path is:

```
∂cₜ/∂cₜ₋₁ = fₜ  (element-wise)
```

If the forget gate learns to keep `fₜ ≈ 1`, gradients flow through unchanged — no vanishing!

### c. How It Works (Step-by-Step)

1. **Concatenate** `[hₜ₋₁, xₜ]` into a combined input vector
2. **Forget gate**: sigmoid decides what fraction of old cell state to keep
3. **Input gate**: sigmoid decides what fraction of new candidate to write
4. **Candidate**: tanh computes potential new values for the cell
5. **Update cell state**: multiply old cell by forget gate, add new candidate * input gate
6. **Output gate**: sigmoid decides what to expose as the new hidden state
7. **Hidden state**: output gate * tanh(cell state)
8. **Repeat** for next time step

### d. Visual Representation

```
                    ╔══════════════════════════════════════╗
                    ║           LSTM Cell                  ║
                    ║                                      ║
cₜ₋₁ ─────────────[×]──────────[+]──────────────────────── cₜ
                    │           │
                    │ forget    │ input
                   [f] = σ    [c̃] = tanh
                    │          ×
                   [i] = σ
                    │
hₜ₋₁ ──┐           └──────────────────────────────────── hₜ
        ├──► concat with xₜ
xₜ  ────┘              │
                        ▼
                   [o] = σ
                    │
                   tanh(cₜ) ×──────────────────────────── hₜ
                    ║                                      ║
                    ╚══════════════════════════════════════╝

Legend: [×] = multiply  [+] = add  σ = sigmoid  × = element-wise
```

**Gate behavior examples:**

```
Forget gate fₜ ≈ 0: Erase old memory (e.g., end of sentence → reset)
Forget gate fₜ ≈ 1: Keep old memory (e.g., subject noun for verb agreement)
Input gate iₜ ≈ 0: Ignore new input (e.g., filler word "the")
Input gate iₜ ≈ 1: Store new input (e.g., key entity like a name)
```

### e. Python Implementation

```python
import torch
import torch.nn as nn


# ─────────────────────────────────────────────
# LSTM Cell from scratch (educational)
# ─────────────────────────────────────────────
class LSTMCell(nn.Module):
    """
    Manual LSTM cell implementation.
    Computes one time step of the LSTM equations.
    """
    def __init__(self, input_size: int, hidden_size: int):
        super().__init__()
        self.hidden_size = hidden_size

        # All 4 gates in one weight matrix for efficiency
        # Concatenates: [forget, input, cell, output]
        self.W_gates = nn.Linear(input_size + hidden_size, 4 * hidden_size)

    def forward(self, x_t, state):
        h_prev, c_prev = state
        # x_t:   (batch, input_size)
        # h_prev: (batch, hidden_size)

        # Concatenate input and previous hidden state
        combined = torch.cat([x_t, h_prev], dim=1)   # (batch, input+hidden)

        # Compute all gates at once
        gates = self.W_gates(combined)                 # (batch, 4*hidden)

        # Split into 4 components
        f, i, g, o = gates.chunk(4, dim=1)

        # Apply activations
        f = torch.sigmoid(f)      # forget gate
        i = torch.sigmoid(i)      # input gate
        g = torch.tanh(g)         # candidate cell state
        o = torch.sigmoid(o)      # output gate

        # Update cell and hidden state
        c_t = f * c_prev + i * g
        h_t = o * torch.tanh(c_t)

        return h_t, c_t


# ─────────────────────────────────────────────
# Production LSTM for time-series forecasting
# ─────────────────────────────────────────────
class LSTMForecaster(nn.Module):
    """
    LSTM model for sequence-to-one prediction.
    Architecture: LSTM → Dropout → Linear
    """
    def __init__(self, input_size=1, hidden_size=128, num_layers=2,
                 output_size=1, dropout=0.2):
        super().__init__()
        self.lstm = nn.LSTM(
            input_size=input_size,
            hidden_size=hidden_size,
            num_layers=num_layers,
            batch_first=True,
            dropout=dropout if num_layers > 1 else 0.0,
            bidirectional=False
        )
        self.dropout = nn.Dropout(dropout)
        self.fc      = nn.Linear(hidden_size, output_size)

    def forward(self, x):
        # x: (batch, seq_len, input_size)
        # out: (batch, seq_len, hidden_size)
        # h_n: (num_layers, batch, hidden_size)
        # c_n: (num_layers, batch, hidden_size)
        out, (h_n, c_n) = self.lstm(x)

        # Take the last time step's output
        last = out[:, -1, :]           # (batch, hidden_size)
        last = self.dropout(last)
        return self.fc(last)           # (batch, output_size)


# ─────────────────────────────────────────────
# LSTM for sequence labeling (many-to-many)
# ─────────────────────────────────────────────
class LSTMTagger(nn.Module):
    """
    LSTM that produces an output at every time step.
    Use case: POS tagging, NER, anomaly detection per step.
    """
    def __init__(self, vocab_size, embed_dim, hidden_size, num_classes, num_layers=1):
        super().__init__()
        self.embed  = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.lstm   = nn.LSTM(embed_dim, hidden_size, num_layers,
                              batch_first=True, dropout=0.3 if num_layers > 1 else 0.0)
        self.fc     = nn.Linear(hidden_size, num_classes)

    def forward(self, x):
        # x: (batch, seq_len) — token indices
        embedded = self.embed(x)                    # (batch, seq_len, embed_dim)
        out, _   = self.lstm(embedded)              # (batch, seq_len, hidden_size)
        logits   = self.fc(out)                     # (batch, seq_len, num_classes)
        return logits


# ─────────────────────────────────────────────
# Demo: Training LSTM forecaster
# ─────────────────────────────────────────────
def demo_lstm():
    X, y    = generate_sine_sequence(n_samples=1000, seq_len=50)
    dataset = SequenceDataset(X, y)
    loader  = DataLoader(dataset, batch_size=32, shuffle=True)

    model   = LSTMForecaster(input_size=1, hidden_size=128, num_layers=2, dropout=0.2)
    print(f"Total parameters: {sum(p.numel() for p in model.parameters()):,}")

    model   = train_model(model, loader, epochs=30, lr=1e-3)
    return model
```

## 4. Gated Recurrent Unit (GRU)

### a. Intuition

The GRU (Cho et al., 2014) is LSTM's streamlined sibling. Where the LSTM has separate cell state and hidden state + three gates, the GRU **merges the cell and hidden state** and uses only **two gates**.

Think of it as the LSTM going on a diet: it removes the output gate and merges the forget/input gates into a single "update gate." The result is a model that's:
- Faster to train (fewer parameters)
- Often just as good as LSTM in practice
- Easier to understand and debug

### b. Mathematical Insight

At each time step `t`:

```
zₜ = σ(Wz · [hₜ₋₁, xₜ])      # Update gate (blends old and new)
rₜ = σ(Wr · [hₜ₋₁, xₜ])      # Reset gate (how much old state matters)
h̃ₜ = tanh(W · [rₜ ⊙ hₜ₋₁, xₜ])  # Candidate hidden state
hₜ = (1 - zₜ) ⊙ hₜ₋₁ + zₜ ⊙ h̃ₜ  # Final hidden state
```

**Key insight about the update gate:**

```
hₜ = (1 - zₜ) ⊙ hₜ₋₁ + zₜ ⊙ h̃ₜ
        ─────────────────  ─────────
          Keep old info      Add new info
```

When `zₜ → 0`: keep the old state unchanged (skip update — like a skip connection)
When `zₜ → 1`: completely replace with new candidate

This is a **linear interpolation** between old and new state, which is gradient-friendly.

### c. How It Works (Step-by-Step)

1. **Reset gate `rₜ`**: Decides how much of previous hidden state to use when computing candidate
2. **Candidate `h̃ₜ`**: Compute new candidate state, partially ignoring old state via reset gate
3. **Update gate `zₜ`**: Decides the mix between old hidden state and new candidate
4. **Final hidden state**: Linear combination of old and new via update gate
5. **No separate cell state** — everything is in `hₜ`

### d. Visual Representation

```
GRU Cell:

hₜ₋₁ ──┬────────────────────────────────────[1-z]──[+]──► hₜ
        │                                             │
        ├──► [z gate: σ] ◄── xₜ ──────────────[z]──►┘
        │
        ├──► [r gate: σ] ◄── xₜ
        │         │
        └────[×]──┘
              │
              ▼
         [h̃: tanh] ◄── xₜ

LSTM has 4 weight matrices; GRU has 3 → ~25% faster
```

**GRU vs LSTM parameter count:**

```
LSTM: 4 × (input_size + hidden_size) × hidden_size
GRU:  3 × (input_size + hidden_size) × hidden_size

For input=100, hidden=256:
LSTM: 4 × 356 × 256 = 364,544 parameters
GRU:  3 × 356 × 256 = 273,408 parameters  (25% fewer)
```

### e. Python Implementation

```python
import torch
import torch.nn as nn


# ─────────────────────────────────────────────
# GRU Cell from scratch
# ─────────────────────────────────────────────
class GRUCell(nn.Module):
    """Manual GRU cell. Demonstrates the 2-gate design."""
    def __init__(self, input_size: int, hidden_size: int):
        super().__init__()
        self.hidden_size = hidden_size

        # Update and reset gates
        self.W_z = nn.Linear(input_size + hidden_size, hidden_size)  # update gate
        self.W_r = nn.Linear(input_size + hidden_size, hidden_size)  # reset gate
        self.W_h = nn.Linear(input_size + hidden_size, hidden_size)  # candidate

    def forward(self, x_t, h_prev):
        combined = torch.cat([x_t, h_prev], dim=1)

        z = torch.sigmoid(self.W_z(combined))      # update gate
        r = torch.sigmoid(self.W_r(combined))      # reset gate

        # Candidate uses reset gate to partially ignore h_prev
        combined_r = torch.cat([x_t, r * h_prev], dim=1)
        h_tilde    = torch.tanh(self.W_h(combined_r))

        # Interpolate between old and new
        h_t = (1 - z) * h_prev + z * h_tilde
        return h_t


# ─────────────────────────────────────────────
# Production GRU forecaster
# ─────────────────────────────────────────────
class GRUForecaster(nn.Module):
    """GRU-based time series forecaster. Generally preferred over vanilla RNN."""
    def __init__(self, input_size=1, hidden_size=128, num_layers=2,
                 output_size=1, dropout=0.2):
        super().__init__()
        self.gru = nn.GRU(
            input_size=input_size,
            hidden_size=hidden_size,
            num_layers=num_layers,
            batch_first=True,
            dropout=dropout if num_layers > 1 else 0.0
        )
        self.norm    = nn.LayerNorm(hidden_size)   # Stabilizes training
        self.dropout = nn.Dropout(dropout)
        self.fc      = nn.Linear(hidden_size, output_size)

    def forward(self, x):
        out, h_n = self.gru(x)
        last = out[:, -1, :]
        last = self.norm(last)
        last = self.dropout(last)
        return self.fc(last)


# ─────────────────────────────────────────────
# GRU for text classification
# ─────────────────────────────────────────────
class GRUClassifier(nn.Module):
    """
    GRU-based text classifier.
    Uses mean pooling over all time steps (often better than last state).
    """
    def __init__(self, vocab_size, embed_dim=128, hidden_size=256,
                 num_classes=2, num_layers=2, dropout=0.3):
        super().__init__()
        self.embed  = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.gru    = nn.GRU(embed_dim, hidden_size, num_layers,
                             batch_first=True,
                             bidirectional=False,
                             dropout=dropout if num_layers > 1 else 0.0)
        self.pool   = lambda out: out.mean(dim=1)  # Mean pooling
        self.dropout = nn.Dropout(dropout)
        self.fc     = nn.Linear(hidden_size, num_classes)

    def forward(self, x, lengths=None):
        emb  = self.embed(x)                       # (batch, seq, embed)
        out, _ = self.gru(emb)                     # (batch, seq, hidden)
        pooled = out.mean(dim=1)                   # (batch, hidden) — mean pooling
        return self.fc(self.dropout(pooled))       # (batch, num_classes)


# ─────────────────────────────────────────────
# Comparing GRU vs LSTM parameter counts
# ─────────────────────────────────────────────
def compare_parameters(input_size=100, hidden_size=256, num_layers=2):
    lstm_model = nn.LSTM(input_size, hidden_size, num_layers, batch_first=True)
    gru_model  = nn.GRU(input_size,  hidden_size, num_layers, batch_first=True)

    lstm_params = sum(p.numel() for p in lstm_model.parameters())
    gru_params  = sum(p.numel() for p in gru_model.parameters())

    print(f"LSTM parameters: {lstm_params:,}")
    print(f"GRU parameters:  {gru_params:,}")
    print(f"GRU savings:     {(1 - gru_params/lstm_params)*100:.1f}%")

compare_parameters()
```

## 5. Bidirectional RNNs

### a. Intuition

Standard RNNs process sequences left-to-right. But many tasks benefit from knowing the **future context** when making a decision about the present.

Consider: *"I _____ the deposit."* — Without the following word, "withdrew" or "made" are both plausible. But if you see *"I _____ the deposit before the bank closed,"* future context helps disambiguate.

A **Bidirectional RNN** runs two RNNs in parallel:
- One **forward** (left-to-right, seeing past)
- One **backward** (right-to-left, seeing future)

Their outputs are concatenated, giving the model complete context from both directions at every position.

### b. Mathematical Insight

At each time step `t`:

```
Forward pass:   →hₜ = RNN(xₜ, →hₜ₋₁)    # left to right
Backward pass:  ←hₜ = RNN(xₜ, ←hₜ₊₁)   # right to left

Combined:       ĥₜ = [→hₜ; ←hₜ]          # concatenation
                yₜ = W · ĥₜ + b
```

The combined representation `ĥₜ` at position `t` has seen:
- All tokens from `1 to t` (via forward)
- All tokens from `T to t` (via backward)

This doubles the hidden size in the output. If each direction has `h` units, the output is `2h` dimensional.

### c. How It Works (Step-by-Step)

1. **Forward RNN**: Process sequence from left to right, collect `→h₁, →h₂, ..., →hₜ`
2. **Backward RNN**: Process same sequence from right to left, collect `←h₁, ←h₂, ..., ←hₜ`
3. **Concatenate**: At each position `t`, concatenate `[→hₜ; ←hₜ]` → vector of size `2 × hidden`
4. **Output layer**: Apply linear transformation to each concatenated vector
5. **For many-to-one**: Often concatenate only `→hₜ` and `←h₁` (the "boundary" states)

> ⚠️ **Critical limitation**: Bidirectional RNNs **cannot be used for autoregressive generation** or real-time inference where future tokens aren't available. They're best for classification and labeling where the full sequence is available.

### d. Visual Representation

```
Forward RNN:
x₁ ──► [→] ──► [→] ──► [→] ──► [→]
         →h₁    →h₂    →h₃    →h₄

Backward RNN:
x₁ ◄── [←] ◄── [←] ◄── [←] ◄── [←]
        ←h₁    ←h₂    ←h₃    ←h₄

Combined output at each position:
         ĥ₁     ĥ₂     ĥ₃     ĥ₄
         │       │       │       │
    [→h₁;←h₁] [→h₂;←h₂] [→h₃;←h₃] [→h₄;←h₄]

Each ĥₜ has seen the ENTIRE sequence from both directions.
```

### e. Python Implementation

```python
import torch
import torch.nn as nn
import torch.nn.functional as F


# ─────────────────────────────────────────────
# Bidirectional LSTM for text classification
# ─────────────────────────────────────────────
class BiLSTMClassifier(nn.Module):
    """
    Bidirectional LSTM for sentiment/document classification.
    Uses attention pooling to aggregate sequence representations.
    """
    def __init__(self, vocab_size, embed_dim=128, hidden_size=256,
                 num_classes=2, num_layers=2, dropout=0.3, pad_idx=0):
        super().__init__()
        self.embed  = nn.Embedding(vocab_size, embed_dim, padding_idx=pad_idx)
        self.drop_embed = nn.Dropout(dropout)

        self.lstm   = nn.LSTM(
            embed_dim, hidden_size, num_layers,
            batch_first=True,
            bidirectional=True,     # The key flag
            dropout=dropout if num_layers > 1 else 0.0
        )
        # Bidirectional doubles output size
        bidir_size = hidden_size * 2

        # Attention layer to pool sequence
        self.attention = nn.Linear(bidir_size, 1)

        self.fc     = nn.Sequential(
            nn.Dropout(dropout),
            nn.Linear(bidir_size, hidden_size),
            nn.ReLU(),
            nn.Dropout(dropout),
            nn.Linear(hidden_size, num_classes)
        )

    def attention_pool(self, out, mask=None):
        """
        Soft attention pooling over the sequence dimension.
        Learns which time steps are most important.
        """
        # out: (batch, seq, 2*hidden)
        scores = self.attention(out).squeeze(-1)  # (batch, seq)

        if mask is not None:
            # Mask padding positions with -inf before softmax
            scores = scores.masked_fill(mask == 0, float('-inf'))

        weights = F.softmax(scores, dim=1)        # (batch, seq)
        context = (weights.unsqueeze(-1) * out).sum(dim=1)  # (batch, 2*hidden)
        return context

    def forward(self, x, lengths=None):
        # x: (batch, seq_len) — token indices
        emb  = self.drop_embed(self.embed(x))     # (batch, seq, embed)

        # Pack padded sequences for variable-length efficiency
        if lengths is not None:
            emb = nn.utils.rnn.pack_padded_sequence(
                emb, lengths.cpu(), batch_first=True, enforce_sorted=False)

        out, (h_n, c_n) = self.lstm(emb)

        if lengths is not None:
            out, _ = nn.utils.rnn.pad_packed_sequence(out, batch_first=True)

        # Attention pooling
        context = self.attention_pool(out)         # (batch, 2*hidden)
        return self.fc(context)                    # (batch, num_classes)


# ─────────────────────────────────────────────
# Bidirectional GRU for NER / sequence labeling
# ─────────────────────────────────────────────
class BiGRUTagger(nn.Module):
    """
    Bidirectional GRU for token-level classification (NER, POS, etc.)
    CRF layer can be added on top for structured prediction.
    """
    def __init__(self, vocab_size, embed_dim=100, hidden_size=128,
                 num_tags=10, num_layers=1, dropout=0.5):
        super().__init__()
        self.embed = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.gru   = nn.GRU(embed_dim, hidden_size, num_layers,
                            batch_first=True, bidirectional=True,
                            dropout=dropout if num_layers > 1 else 0.0)
        self.dropout = nn.Dropout(dropout)
        self.fc      = nn.Linear(hidden_size * 2, num_tags)

    def forward(self, x):
        emb  = self.embed(x)                       # (batch, seq, embed)
        out, _ = self.gru(emb)                     # (batch, seq, 2*hidden)
        out  = self.dropout(out)
        return self.fc(out)                        # (batch, seq, num_tags)


# ─────────────────────────────────────────────
# Inspecting the forward and backward states
# ─────────────────────────────────────────────
def inspect_bidirectional():
    """Shows how to extract forward and backward hidden states separately."""
    model = nn.LSTM(input_size=10, hidden_size=32,
                    num_layers=1, batch_first=True, bidirectional=True)

    x   = torch.randn(4, 20, 10)   # (batch=4, seq=20, features=10)
    out, (h_n, c_n) = model(x)

    # h_n shape: (num_directions * num_layers, batch, hidden_size)
    # = (2, 4, 32) for 1 layer, bidirectional
    h_forward  = h_n[0]  # (batch, 32) — forward final state
    h_backward = h_n[1]  # (batch, 32) — backward final state

    # For many-to-one, concatenate boundary states
    h_combined = torch.cat([h_forward, h_backward], dim=1)  # (batch, 64)

    print(f"Output:           {out.shape}")        # (4, 20, 64)
    print(f"Forward state:    {h_forward.shape}")  # (4, 32)
    print(f"Backward state:   {h_backward.shape}") # (4, 32)
    print(f"Combined:         {h_combined.shape}") # (4, 64)

inspect_bidirectional()
```

## 6. Deep RNNs

### a. Intuition

Just as deep feedforward networks learn hierarchical features (edges → shapes → objects), **deep RNNs** stack multiple recurrent layers to learn hierarchical temporal representations.

- Layer 1 might capture: short-term patterns, local syntax
- Layer 2 might capture: phrase-level structure, medium-term trends
- Layer 3 might capture: sentence-level or document-level semantics, long-term trends

Think of it like a jazz band: the drummer keeps the beat (Layer 1), the bassist adds harmony structure (Layer 2), and the soloist improvises meaningful melodies over both (Layer 3).

### b. Mathematical Insight

In a deep RNN with `L` layers:

```
Layer 1:   h₁ₜ = RNN₁(xₜ,  h₁ₜ₋₁)
Layer 2:   h²ₜ = RNN₂(h¹ₜ, h²ₜ₋₁)
Layer 3:   h³ₜ = RNN₃(h²ₜ, h³ₜ₋₁)
...
Output:    yₜ  = W · hᴸₜ + b
```

Each layer takes the **output sequence** of the previous layer as its **input sequence**. The gradients must flow through both time (T steps) and depth (L layers), making training more challenging.

**Dropout between layers** is critical for regularization in deep RNNs.

### c. How It Works (Step-by-Step)

1. **Input sequence** enters Layer 1: processes all T time steps
2. **Layer 1 outputs** `h¹₁...h¹ₜ` — these become the input sequence to Layer 2
3. **Layer 2 processes** the Layer 1 output sequence, produces `h²₁...h²ₜ`
4. **Continue stacking** layers as needed
5. **Final layer output** is passed to the task-specific head (classifier, regressor)
6. **Dropout is applied** between layers during training to prevent co-adaptation

### d. Visual Representation

```
Deep RNN (3 layers):

                  y₁    y₂    y₃    y₄
                  │     │     │     │
Layer 3:  h₀³─►[RNN]─►[RNN]─►[RNN]─►[RNN]
                  │     │     │     │
Layer 2:  h₀²─►[RNN]─►[RNN]─►[RNN]─►[RNN]
                  │     │     │     │
Layer 1:  h₀¹─►[RNN]─►[RNN]─►[RNN]─►[RNN]
                  ▲     ▲     ▲     ▲
                  x₁    x₂    x₃    x₄

Dropout ~~~~ applied between layers
```

**Residual connections in deep RNNs (advanced):**

```
h²ₜ = RNN₂(h¹ₜ, h²ₜ₋₁) + h¹ₜ    # Skip connection
```

This mirrors ResNet's idea — helps gradient flow in very deep models.

### e. Python Implementation

```python
import torch
import torch.nn as nn


# ─────────────────────────────────────────────
# Deep stacked LSTM with residual connections
# ─────────────────────────────────────────────
class DeepRNNLayer(nn.Module):
    """
    A single layer in a deep RNN stack.
    Optionally applies a residual connection.
    """
    def __init__(self, input_size, hidden_size, cell_type='lstm',
                 dropout=0.3, residual=False):
        super().__init__()
        self.residual     = residual and (input_size == hidden_size)
        self.dropout      = nn.Dropout(dropout)
        self.norm         = nn.LayerNorm(hidden_size)

        if cell_type == 'lstm':
            self.rnn = nn.LSTM(input_size, hidden_size, batch_first=True)
        elif cell_type == 'gru':
            self.rnn = nn.GRU(input_size, hidden_size, batch_first=True)
        else:
            self.rnn = nn.RNN(input_size, hidden_size, batch_first=True)

    def forward(self, x, state=None):
        out, state = self.rnn(x, state)         # out: (batch, seq, hidden)
        out = self.dropout(out)
        out = self.norm(out)

        if self.residual:
            out = out + x                        # Residual connection

        return out, state


class DeepRNN(nn.Module):
    """
    Deep RNN with configurable layers, cell type, and residual connections.
    Suitable for complex sequence modeling tasks.
    """
    def __init__(self, input_size, hidden_size, num_layers, output_size,
                 cell_type='lstm', dropout=0.3, residual=False):
        super().__init__()
        self.layers = nn.ModuleList()

        for i in range(num_layers):
            in_size = input_size if i == 0 else hidden_size
            self.layers.append(
                DeepRNNLayer(in_size, hidden_size, cell_type,
                             dropout=dropout if i < num_layers - 1 else 0.0,
                             residual=residual and i > 0)
            )

        self.fc = nn.Linear(hidden_size, output_size)

    def forward(self, x):
        # x: (batch, seq_len, input_size)
        out = x
        for layer in self.layers:
            out, _ = layer(out)

        # Use last time step
        return self.fc(out[:, -1, :])


# ─────────────────────────────────────────────
# Encoder-Decoder with deep RNNs (Seq2Seq)
# ─────────────────────────────────────────────
class Encoder(nn.Module):
    """
    Deep bidirectional LSTM encoder.
    Used for machine translation, summarization, etc.
    """
    def __init__(self, input_size, embed_dim, hidden_size, num_layers, dropout=0.3):
        super().__init__()
        self.embed  = nn.Embedding(input_size, embed_dim, padding_idx=0)
        self.lstm   = nn.LSTM(embed_dim, hidden_size, num_layers,
                              batch_first=True, bidirectional=True,
                              dropout=dropout if num_layers > 1 else 0.0)
        # Project from bidirectional to match decoder hidden size
        self.proj_h = nn.Linear(hidden_size * 2, hidden_size)
        self.proj_c = nn.Linear(hidden_size * 2, hidden_size)
        self.num_layers = num_layers

    def forward(self, src):
        emb  = self.embed(src)
        out, (h_n, c_n) = self.lstm(emb)
        # out: (batch, src_len, 2*hidden)

        # Combine forward and backward final states for decoder init
        # h_n: (2*num_layers, batch, hidden)
        batch = h_n.size(1)
        h_n = h_n.view(self.num_layers, 2, batch, -1)
        h_n = torch.cat([h_n[:, 0], h_n[:, 1]], dim=-1)       # (layers, batch, 2*hidden)
        h_n = torch.tanh(self.proj_h(h_n))                    # (layers, batch, hidden)

        c_n = c_n.view(self.num_layers, 2, batch, -1)
        c_n = torch.cat([c_n[:, 0], c_n[:, 1]], dim=-1)
        c_n = torch.tanh(self.proj_c(c_n))

        return out, (h_n, c_n)


class Decoder(nn.Module):
    """Unidirectional LSTM decoder for seq2seq."""
    def __init__(self, output_size, embed_dim, hidden_size, num_layers, dropout=0.3):
        super().__init__()
        self.embed  = nn.Embedding(output_size, embed_dim, padding_idx=0)
        self.lstm   = nn.LSTM(embed_dim + hidden_size * 2, hidden_size,
                              num_layers, batch_first=True,
                              dropout=dropout if num_layers > 1 else 0.0)
        self.fc     = nn.Linear(hidden_size, output_size)

    def forward(self, tgt, enc_out, state):
        # tgt: (batch, 1) — one token at a time
        emb  = self.embed(tgt)                     # (batch, 1, embed)

        # Simple context: mean of encoder outputs
        ctx  = enc_out.mean(dim=1, keepdim=True)   # (batch, 1, 2*hidden)
        inp  = torch.cat([emb, ctx], dim=-1)       # (batch, 1, embed+2*hidden)

        out, state = self.lstm(inp, state)
        logits = self.fc(out)                      # (batch, 1, vocab_size)
        return logits, state


# ─────────────────────────────────────────────

        params = sum(p.numel() for p in model.parameters())

        print(f"Layers: {cfg['num_layers']} | Params: {params:,} | Output: {out.shape}")

benchmark_depth()
```

## 7. Cross-Topic Relationships

### The Evolution of Sequence Models

Understanding how these ideas connect helps you make better architectural decisions.

```
Sequential Data Problem
        │
        ▼
Vanilla RNN  ──────────────────────────────────────
"I need memory across time steps"                  │
        │                                          │
        ▼ Problem: Vanishing Gradients             │
LSTM  ──────────────────────────────────           │
"Add gates + cell state for long memory"  │        │
        │                           │      │        │
        ▼ Simplification            │      │        │
GRU                                 │      │        │
"Merge cell+hidden, 2 gates not 3" │      │        │
        │                           │      │        │
        ▼ Add context direction      ▼      ▼        ▼
Bidirectional RNN / LSTM / GRU ─────────────────────
"See past AND future simultaneously"               │
        │                                          │
        ▼ Add depth                                │
Deep RNNs (Stacked LSTM/GRU/BiLSTM)               │
"Learn hierarchical temporal features"             │
        │                                          │
        ▼ (Natural next step)                      │
Attention Mechanisms ◄─────────────────────────────┘
"Let the model focus on relevant time steps"
        │
        ▼
Transformer
"Self-attention at scale, parallel computation"
```

**Key relationships to internalize:**

1. **RNN → LSTM/GRU**: Adding gates is the fix for vanishing gradients
2. **LSTM → GRU**: GRU proves you can often achieve the same results with less complexity
3. **Unidirectional → Bidirectional**: More context, but requires full sequence upfront
4. **Single layer → Deep**: Hierarchical feature learning at the cost of compute
5. **Deep RNN → Transformer**: Transformers replace sequential processing with parallel attention

**Task-to-Architecture mapping:**

```
Task                        Best RNN Architecture
─────────────────────────────────────────────────
Text classification         BiLSTM / BiGRU + pooling
Token labeling (NER)        BiLSTM-CRF
Time-series forecasting     LSTM / GRU (unidirectional!)
Speech recognition          Deep BiLSTM → CTC
Machine translation         BiLSTM encoder + LSTM decoder
Language modeling           LSTM (unidirectional)
Anomaly detection           LSTM autoencoder
```

```python
import numpy as np
import pandas as pd
from datetime import datetime, timedelta

def generate_energy_dataset(n_days=365 * 2, seed=42):
    """
    Generate synthetic electricity demand data with:
    - Daily and weekly seasonality
    - Weather effects (temperature)
    - Holiday effects
    - Random noise
    """
    np.random.seed(seed)

    # Hourly timestamps
    start = datetime(2022, 1, 1)
    timestamps = [start + timedelta(hours=i) for i in range(n_days * 24)]

    n = len(timestamps)
    t = np.arange(n)

    # Time features
    hour_of_day  = np.array([ts.hour for ts in timestamps])
    day_of_week  = np.array([ts.weekday() for ts in timestamps])
    month        = np.array([ts.month for ts in timestamps])

    # Daily pattern: peaks at 8am and 7pm
    daily = (
        1.0 * np.sin(2 * np.pi * hour_of_day / 24 - np.pi / 2) +
        0.5 * np.sin(4 * np.pi * hour_of_day / 24)
    )

    # Weekly pattern: lower on weekends
    weekly = np.where(day_of_week >= 5, -0.3, 0.2)

    # Seasonal pattern: higher in summer and winter
    seasonal = 0.5 * np.cos(2 * np.pi * t / (365 * 24) + np.pi)

    # Temperature (correlated with season, affects demand)
    temperature = 20 + 15 * np.cos(2 * np.pi * t / (365 * 24) + np.pi) + \
                  3 * np.random.randn(n)

    # Temperature effect on demand (U-shaped: high when too hot or cold)
    temp_effect = 0.05 * (temperature - 18) ** 2 / 50

    # Holiday effect (random days, demand drops 30%)
    is_holiday = np.zeros(n)
    holiday_days = np.random.choice(n_days, size=12, replace=False)
    for hday in holiday_days:
        is_holiday[hday * 24 : (hday + 1) * 24] = 1

    # Base demand
    base_demand = 5000  # MW

    demand = (base_demand +
              800 * daily +
              400 * weekly +
              300 * seasonal +
              500 * temp_effect -
              600 * is_holiday +
              100 * np.random.randn(n))

    # Clip to realistic range
    demand = np.clip(demand, 2000, 10000)

    df = pd.DataFrame({
        'timestamp':   timestamps,
        'demand_mw':   demand,
        'temperature': temperature,
        'hour':        hour_of_day,
        'day_of_week': day_of_week,
        'month':       month,
        'is_holiday':  is_holiday,
        'is_weekend':  (day_of_week >= 5).astype(int)
    })

    return df
```

```python
def engineer_features(df: pd.DataFrame) -> pd.DataFrame:
    """
    Create features for LSTM forecasting:
    - Cyclic time encodings (sin/cos)
    - Lag features
    - Rolling statistics
    """
    df = df.copy()

    # ── Cyclic encoding: avoids discontinuity (hour 23 → 0) ──
    df['hour_sin']   = np.sin(2 * np.pi * df['hour'] / 24)
    df['hour_cos']   = np.cos(2 * np.pi * df['hour'] / 24)
    df['dow_sin']    = np.sin(2 * np.pi * df['day_of_week'] / 7)
    df['dow_cos']    = np.cos(2 * np.pi * df['day_of_week'] / 7)
    df['month_sin']  = np.sin(2 * np.pi * df['month'] / 12)
    df['month_cos']  = np.cos(2 * np.pi * df['month'] / 12)

    # ── Lag features (previous demand values) ──
    for lag in [1, 2, 3, 6, 12, 24, 48, 168]:
        df[f'demand_lag_{lag}'] = df['demand_mw'].shift(lag)

    # ── Rolling statistics ──
    df['demand_roll_24h_mean'] = df['demand_mw'].shift(1).rolling(24).mean()
    df['demand_roll_24h_std']  = df['demand_mw'].shift(1).rolling(24).std()
    df['demand_roll_7d_mean']  = df['demand_mw'].shift(1).rolling(168).mean()

    # ── Drop rows with NaN from lagging ──
    df = df.dropna().reset_index(drop=True)

    print(f"Features created: {df.shape[1]} columns")
    print(f"Records after lag creation: {len(df):,}")
    return df
```

```python
# Dataset class for sliding window approach

class EnergyDataset(Dataset):
    """
    Creates (X, y) pairs using a sliding window.
    X: past `seq_len` hours of features
    y: next `horizon` hours of demand
    """
    def __init__(self, df, feature_cols, target_col,
                 seq_len=72, horizon=24):
        self.seq_len  = seq_len
        self.horizon  = horizon
        self.features = df[feature_cols].values.astype(np.float32)
        self.targets  = df[target_col].values.astype(np.float32)

    def __len__(self):
        return len(self.features) - self.seq_len - self.horizon + 1

    def __getitem__(self, idx):
        x = self.features[idx : idx + self.seq_len]
        y = self.targets[idx + self.seq_len : idx + self.seq_len + self.horizon]
        return torch.tensor(x), torch.tensor(y)

def temporal_split(df, train_ratio=0.7, val_ratio=0.15):
    n       = len(df)
    t_end   = int(n * train_ratio)
    v_end   = int(n * (train_ratio + val_ratio))
    return df[:t_end], df[t_end:v_end], df[v_end:]

class ForecastLSTM(nn.Module):
    def __init__(self, input_size, hidden_size=128, num_layers=2,
                 horizon=24, dropout=0.2):
        super().__init__()
        self.lstm = nn.LSTM(input_size, hidden_size, num_layers,
                            batch_first=True,
                            dropout=dropout if num_layers > 1 else 0.0)
        self.fc = nn.Sequential(
            nn.Dropout(dropout),
            nn.Linear(hidden_size, hidden_size // 2),
            nn.ReLU(),
            nn.Linear(hidden_size // 2, horizon)
        )

    def forward(self, x):
        out, _ = self.lstm(x)
        return self.fc(out[:, -1, :])


class ForecastGRU(nn.Module):
    def __init__(self, input_size, hidden_size=128, num_layers=2,
                 horizon=24, dropout=0.2):
        super().__init__()
        self.gru = nn.GRU(input_size, hidden_size, num_layers,
                          batch_first=True,
                          dropout=dropout if num_layers > 1 else 0.0)
        self.fc = nn.Sequential(
            nn.Dropout(dropout),
            nn.Linear(hidden_size, hidden_size // 2),
            nn.ReLU(),
            nn.Linear(hidden_size // 2, horizon)
        )

    def forward(self, x):
        out, _ = self.gru(x)
        return self.fc(out[:, -1, :])


class ForecastDeepLSTM(nn.Module):
    """Deep LSTM with layer normalization and residual connections."""
    def __init__(self, input_size, hidden_size=256, num_layers=3,
                 horizon=24, dropout=0.3):
        super().__init__()
        self.lstm = nn.LSTM(input_size, hidden_size, num_layers,
                            batch_first=True,
                            dropout=dropout if num_layers > 1 else 0.0)
        self.norm = nn.LayerNorm(hidden_size)
        self.fc = nn.Sequential(
            nn.Dropout(dropout),
            nn.Linear(hidden_size, hidden_size),
            nn.GELU(),
            nn.Dropout(dropout),
            nn.Linear(hidden_size, horizon)
        )

    def forward(self, x):
        out, _ = self.lstm(x)
        last = self.norm(out[:, -1, :])
        return self.fc(last)

class EarlyStopping:
    def __init__(self, patience=5, min_delta=1e-4):
        self.patience  = patience
        self.min_delta = min_delta
        self.counter   = 0
        self.best      = None
        self.stop      = False

    def __call__(self, val_loss, model, path='best_model.pt'):
        if self.best is None or val_loss < self.best - self.min_delta:
            self.best    = val_loss
            self.counter = 0
            torch.save(model.state_dict(), path)
        else:
            self.counter += 1
            if self.counter >= self.patience:
                self.stop = True


def train_forecaster(model, train_loader, val_loader, epochs=50, lr=1e-3):
    optimizer     = torch.optim.AdamW(model.parameters(), lr=lr, weight_decay=1e-5)
    scheduler     = torch.optim.lr_scheduler.ReduceLROnPlateau(
                        optimizer, patience=3, factor=0.5, verbose=False)
    criterion     = nn.MSELoss()
    early_stop    = EarlyStopping(patience=7)
    history       = {'train': [], 'val': []}

    for epoch in range(epochs):
        # Training
        model.train()
        train_loss = 0.0
        for X, y in train_loader:
            optimizer.zero_grad()
            pred = model(X)
            loss = criterion(pred, y)
            loss.backward()
            nn.utils.clip_grad_norm_(model.parameters(), 1.0)
            optimizer.step()
            train_loss += loss.item()
        train_loss /= len(train_loader)

        # Validation
        model.eval()
        val_loss = 0.0
        with torch.no_grad():
            for X, y in val_loader:
                pred = model(X)
                val_loss += criterion(pred, y).item()
        val_loss /= len(val_loader)

        scheduler.step(val_loss)
        early_stop(val_loss, model)
        history['train'].append(train_loss)
        history['val'].append(val_loss)

        if (epoch + 1) % 10 == 0:
            print(f"Epoch {epoch+1:3d} | Train: {train_loss:.4f} | Val: {val_loss:.4f}")

        if early_stop.stop:
            print(f"Early stopping at epoch {epoch+1}")
            break

    return history
```

```python
# ─────────────────────────────────────────────
# Evaluation
# ─────────────────────────────────────────────
def evaluate_model(model, loader, scaler_y):
    """Evaluate on test set and return inverse-scaled metrics."""
    model.eval()
    all_preds, all_targets = [], []

    with torch.no_grad():
        for X, y in loader:
            pred = model(X).numpy()
            all_preds.append(pred)
            all_targets.append(y.numpy())

    preds   = np.vstack(all_preds)    # (N, horizon)
    targets = np.vstack(all_targets)

    # Inverse transform: reshape to 2D, transform, reshape back
    preds_inv   = scaler_y.inverse_transform(preds.reshape(-1, 1)).reshape(preds.shape)
    targets_inv = scaler_y.inverse_transform(targets.reshape(-1, 1)).reshape(targets.shape)

    mae  = mean_absolute_error(targets_inv.flatten(), preds_inv.flatten())
    rmse = np.sqrt(mean_squared_error(targets_inv.flatten(), preds_inv.flatten()))
    mape = np.mean(np.abs((targets_inv - preds_inv) / (targets_inv + 1e-8))) * 100

    return {'MAE': mae, 'RMSE': rmse, 'MAPE': mape}
```

```python
from collections import Counter
import torch
from torch.utils.data import Dataset, DataLoader
from torch.nn.utils.rnn import pad_sequence

class Vocabulary:
    """
    Simple word-level vocabulary with padding and unknown tokens.
    """
    def __init__(self, min_freq=2):
        self.min_freq  = min_freq
        self.word2idx  = {'<PAD>': 0, '<UNK>': 1}
        self.idx2word  = {0: '<PAD>', 1: '<UNK>'}

    def build(self, texts):
        word_counts = Counter()
        for text in texts:
            word_counts.update(text.split())

        for word, count in word_counts.items():
            if count >= self.min_freq:
                idx = len(self.word2idx)
                self.word2idx[word] = idx
                self.idx2word[idx]  = word

        print(f"Vocabulary size: {len(self.word2idx):,}")
        return self

    def encode(self, text, max_len=128):
        tokens = text.split()[:max_len]
        return [self.word2idx.get(w, 1) for w in tokens]  # 1 = UNK


class ReviewDataset(Dataset):
    LABEL_MAP = {'negative': 0, 'neutral': 1, 'positive': 2}

    def __init__(self, df, vocab, max_len=128):
        self.texts   = [vocab.encode(t, max_len) for t in df['clean_review']]
        self.labels  = [self.LABEL_MAP[s] for s in df['sentiment']]
        self.lengths = [len(t) for t in self.texts]

    def __len__(self):
        return len(self.texts)

    def __getitem__(self, idx):
        return (torch.tensor(self.texts[idx],  dtype=torch.long),
                torch.tensor(self.labels[idx], dtype=torch.long),
                self.lengths[idx])


def collate_fn(batch):
    """Custom collate to pad variable-length sequences."""
    texts, labels, lengths = zip(*batch)
    padded = pad_sequence(texts, batch_first=True, padding_value=0)
    return padded, torch.stack(labels), torch.tensor(lengths)
```

**Step 5–9: Models, Training, Evaluation**

```python
from sklearn.metrics import classification_report, confusion_matrix


# ─────────────────────────────────────────────
# Model 1: LSTM Classifier
# ─────────────────────────────────────────────
class LSTMSentiment(nn.Module):
    def __init__(self, vocab_size, embed_dim=128, hidden_size=256,
                 num_layers=2, num_classes=3, dropout=0.4):
        super().__init__()
        self.embed = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.lstm  = nn.LSTM(embed_dim, hidden_size, num_layers,
                             batch_first=True, bidirectional=False,
                             dropout=dropout if num_layers > 1 else 0.0)
        self.fc    = nn.Sequential(
            nn.Dropout(dropout),
            nn.Linear(hidden_size, num_classes)
        )

    def forward(self, x, lengths=None):
        emb       = self.embed(x)
        out, _    = self.lstm(emb)
        last      = out[:, -1, :]    # Last time step
        return self.fc(last)


# ─────────────────────────────────────────────
# Model 2: BiLSTM with Attention
# ─────────────────────────────────────────────
class BiLSTMAttentionSentiment(nn.Module):
    def __init__(self, vocab_size, embed_dim=128, hidden_size=256,
                 num_layers=2, num_classes=3, dropout=0.4):
        super().__init__()
        self.embed     = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.lstm      = nn.LSTM(embed_dim, hidden_size, num_layers,
                                 batch_first=True, bidirectional=True,
                                 dropout=dropout if num_layers > 1 else 0.0)
        self.attention = nn.Linear(hidden_size * 2, 1)
        self.fc        = nn.Sequential(
            nn.Dropout(dropout),
            nn.Linear(hidden_size * 2, hidden_size),
            nn.ReLU(),
            nn.Dropout(dropout),
            nn.Linear(hidden_size, num_classes)
        )

    def forward(self, x, lengths=None):
        emb    = self.embed(x)                      # (B, T, E)
        out, _ = self.lstm(emb)                     # (B, T, 2H)

        # Attention
        scores  = self.attention(out).squeeze(-1)   # (B, T)
        weights = torch.softmax(scores, dim=1)      # (B, T)
        context = (weights.unsqueeze(-1) * out).sum(dim=1)  # (B, 2H)

        return self.fc(context)


# ─────────────────────────────────────────────
# Model 3: Stacked BiGRU (Deep)
# ─────────────────────────────────────────────
class DeepBiGRUSentiment(nn.Module):
    def __init__(self, vocab_size, embed_dim=128, hidden_size=128,
                 num_layers=3, num_classes=3, dropout=0.4):
        super().__init__()
        self.embed = nn.Embedding(vocab_size, embed_dim, padding_idx=0)
        self.gru   = nn.GRU(embed_dim, hidden_size, num_layers,
                            batch_first=True, bidirectional=True,
                            dropout=dropout if num_layers > 1 else 0.0)
        # Max + mean pooling
        self.fc = nn.Sequential(
            nn.Dropout(dropout),
            nn.Linear(hidden_size * 4, hidden_size),
            nn.ReLU(),
            nn.Dropout(dropout),
            nn.Linear(hidden_size, num_classes)
        )

    def forward(self, x, lengths=None):
        emb    = self.embed(x)
        out, _ = self.gru(emb)                      # (B, T, 2H)

        # Max pooling + Mean pooling concatenation
        max_pool  = out.max(dim=1).values           # (B, 2H)
        mean_pool = out.mean(dim=1)                 # (B, 2H)
        pooled    = torch.cat([max_pool, mean_pool], dim=1)  # (B, 4H)

        return self.fc(pooled)


# ─────────────────────────────────────────────
# Training loop for classification
# ─────────────────────────────────────────────
def train_classifier(model, train_loader, val_loader, epochs=30, lr=3e-4):
    optimizer  = torch.optim.AdamW(model.parameters(), lr=lr, weight_decay=1e-4)
    scheduler  = torch.optim.lr_scheduler.CosineAnnealingLR(optimizer, T_max=epochs)
    criterion  = nn.CrossEntropyLoss()
    early_stop = EarlyStopping(patience=5)
    best_acc   = 0.0

    for epoch in range(epochs):
        # Train
        model.train()
        train_correct, train_total, train_loss = 0, 0, 0.0
        for X, y, lengths in train_loader:
            optimizer.zero_grad()
            logits = model(X, lengths)
            loss   = criterion(logits, y)
            loss.backward()
            nn.utils.clip_grad_norm_(model.parameters(), 1.0)
            optimizer.step()

            train_correct += (logits.argmax(1) == y).sum().item()
            train_total   += len(y)
            train_loss    += loss.item()

        # Validate
        model.eval()
        val_correct, val_total, val_loss = 0, 0, 0.0
        with torch.no_grad():
            for X, y, lengths in val_loader:
                logits   = model(X, lengths)
                loss     = criterion(logits, y)
                val_correct += (logits.argmax(1) == y).sum().item()
                val_total   += len(y)
                val_loss    += loss.item()

        val_acc  = val_correct / val_total
        val_loss = val_loss / len(val_loader)
        scheduler.step()
        early_stop(val_loss, model)

        if val_acc > best_acc:
            best_acc = val_acc

        if (epoch + 1) % 5 == 0:
            print(f"Epoch {epoch+1:3d} | "
                  f"Train Acc: {train_correct/train_total:.3f} | "
                  f"Val Acc: {val_acc:.3f}")

        if early_stop.stop:
            print(f"Early stopping at epoch {epoch+1}")
            break

    print(f"Best val accuracy: {best_acc:.3f}")
    return model


# ─────────────────────────────────────────────
# Evaluation with full classification report
# ─────────────────────────────────────────────
def evaluate_classifier(model, loader):
    model.eval()
    all_preds, all_labels = [], []
    with torch.no_grad():
        for X, y, lengths in loader:
            preds = model(X, lengths).argmax(1)
            all_preds.extend(preds.numpy())
            all_labels.extend(y.numpy())

    print(classification_report(all_labels, all_preds,
                                 target_names=['negative', 'neutral', 'positive']))
    return all_preds, all_labels


# ─────────────────────────────────────────────
# Train and compare

}
```

## 10. Common Mistakes & Pitfalls

### 1. Data Leakage in Time-Series

```python
# ❌ WRONG: Random split leaks future data into training
from sklearn.model_selection import train_test_split
train, test = train_test_split(df, test_size=0.2, random_state=42)  # BAD for time-series!

# ✅ CORRECT: Temporal split — always train on past, test on future
n = len(df)
train = df[:int(n * 0.7)]
val   = df[int(n * 0.7):int(n * 0.85)]
test  = df[int(n * 0.85):]
```

### 2. Forgetting to Scale the Target

```python
# ❌ WRONG: Only scaling features
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)
# y left unscaled → loss values huge → unstable training

# ✅ CORRECT: Scale target separately (so you can inverse-transform)
scaler_X = StandardScaler().fit(X_train)
scaler_y = StandardScaler().fit(y_train.reshape(-1, 1))

X_train_s = scaler_X.transform(X_train)
y_train_s = scaler_y.transform(y_train.reshape(-1, 1)).flatten()
```

### 3. Not Clipping Gradients

```python
# ❌ WRONG: Training without gradient clipping
loss.backward()
optimizer.step()  # Exploding gradients will destabilize training

# ✅ CORRECT: Always clip gradients for RNN/LSTM/GRU
loss.backward()
nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
optimizer.step()
```

### 4. Using Bidirectional RNN for Forecasting

```python
# ❌ WRONG: BiLSTM for time-series prediction
# This creates data leakage — the backward pass sees future values!
class BadForecaster(nn.Module):
    def __init__(self):
        super().__init__()
        self.lstm = nn.LSTM(1, 64, batch_first=True, bidirectional=True)  # BUG!

# ✅ CORRECT: Use unidirectional LSTM for forecasting
class GoodForecaster(nn.Module):
    def __init__(self):
        super().__init__()
        self.lstm = nn.LSTM(1, 64, batch_first=True, bidirectional=False)
```

### 5. Ignoring Padding in Loss Computation

```python
# ❌ WRONG: Loss includes padding positions
logits = model(padded_seqs)           # (B, T, C)
loss   = criterion(logits.reshape(-1, C), labels.reshape(-1))  # Includes PAD positions

# ✅ CORRECT: Mask padding in loss
criterion = nn.CrossEntropyLoss(ignore_index=0)   # 0 = PAD token index
loss = criterion(logits.reshape(-1, C), labels.reshape(-1))
```

### 6. Stacking Too Many Layers Without Regularization

```python
# ❌ WRONG: Deep model without dropout
model = nn.LSTM(input_size, hidden_size, num_layers=6, batch_first=True)
# Will overfit on small datasets

# ✅ CORRECT: Dropout between layers
model = nn.LSTM(input_size, hidden_size, num_layers=6,
                batch_first=True, dropout=0.3)   # dropout applied between layers
# Also: add layer norm, reduce hidden_size, use early stopping
```

### 7. Wrong Batch Dimension Ordering

```python
# ❌ WRONG: Forgetting batch_first=True
model = nn.LSTM(input_size, hidden_size)    # Default: (seq, batch, features)
x = torch.randn(32, 100, 10)               # (batch, seq, features)
out, _ = model(x)                          # Shape mismatch!

# ✅ CORRECT: Always set batch_first=True OR reshape accordingly
model = nn.LSTM(input_size, hidden_size, batch_first=True)
out, _ = model(x)  # x: (batch, seq, features) ✓
```

### 8. Not Using PackedSequence for Variable-Length Inputs

```python
# ❌ WRONG: LSTM processes padding tokens as real input (wastes compute, hurts quality)
out, _ = lstm(padded_seq)

# ✅ CORRECT: Pack, process, unpack
packed = nn.utils.rnn.pack_padded_sequence(
    padded_seq, lengths, batch_first=True, enforce_sorted=False)
out_packed, _ = lstm(packed)
out, _ = nn.utils.rnn.pad_packed_sequence(out_packed, batch_first=True)
```

## 11. Interview Preparation

### Conceptual Questions

**Q1: Why do vanilla RNNs suffer from the vanishing gradient problem, and how do LSTMs solve it?**

> In vanilla RNNs, the gradient at time step `t` depends on the product of Jacobians across all previous steps: `∂L/∂h₁ = (∂hₜ/∂h₁)·(∂L/∂hₜ)`. Since `∂hₜ/∂hₜ₋₁ = Wₕₕᵀ · diag(tanh'(hₜ₋₁))`, and tanh derivative is at most 1, this product rapidly approaches 0 for long sequences (or explodes if the spectral radius of `Wₕₕ` > 1). LSTMs introduce a **cell state** that is updated via addition: `cₜ = fₜ ⊙ cₜ₋₁ + iₜ ⊙ c̃ₜ`. The gradient through this additive path is `∂cₜ/∂cₜ₋₁ = fₜ`, which is the forget gate — if the forget gate learns to be ~1, gradients flow unchanged over many steps.

**Q2: When would you choose GRU over LSTM?**

> GRU is preferred when: (1) training data is limited (fewer parameters means less overfitting), (2) inference speed is a priority (25% fewer parameters → faster), (3) you're iterating quickly on experiments. Empirically, GRU matches LSTM on ~80% of benchmarks. I'd choose LSTM when: the task explicitly requires tracking long-range dependencies (e.g., coreferencing pronouns across hundreds of words), or when I have experimental evidence that LSTM outperforms GRU on my specific dataset.

**Q3: What is BPTT and why is truncated BPTT used?**

> BPTT (Backpropagation Through Time) unrolls the RNN across all time steps and computes gradients via the chain rule. For very long sequences, this requires storing all intermediate activations (memory-intensive) and computing very long gradient chains (computationally expensive and numerically unstable). Truncated BPTT limits the unrolling to `k` steps: the forward pass is full, but gradients are only propagated back `k` steps. This trades gradient accuracy for tractability. Common values are k=20–100.

**Q4: Why can't you use a Bidirectional RNN for language generation?**

> Language generation is autoregressive: at each step, you generate `xₜ` using only `x₁...xₜ₋₁`. A bidirectional RNN's backward pass processes tokens from right to left, which means it uses `xₜ₊₁...xₙ` — tokens that haven't been generated yet. This creates a circular dependency (you need future tokens to generate the current token). BiRNNs are only valid when the full input sequence is available at inference time (classification, tagging, translation encoding).

**Q5: How does the forget gate work and what happens if it's saturated at 0?**

> The forget gate `fₜ = σ(Wf[hₜ₋₁, xₜ] + b)` outputs values in (0,1) and element-wise multiplies the previous cell state: `cₜ = fₜ ⊙ cₜ₋₁ + ...`. If `fₜ ≈ 0` (saturated closed), the cell completely forgets all previous memory — this is appropriate at natural boundaries like sentence ends. If `fₜ ≈ 1` (saturated open), memory is preserved. At initialization, forget gate biases are often set to 1.0 (a good practice) so the LSTM starts with a bias toward remembering.

### Scenario-Based Questions

**Q6: You're building a fraud detection system on transaction sequences. Each customer has 1–500 transactions. How do you design the model?**

> Key considerations: (1) Variable-length sequences → use PackedSequence or masking. (2) Class imbalance (fraud is rare) → use weighted cross-entropy or focal loss. (3) Real-time requirement → unidirectional LSTM (can process each new transaction as it arrives, streaming inference). (4) I'd use a 2-layer LSTM with hidden size 128, take the last hidden state, add a fraud-specific feature layer (transaction amount, time since last txn, velocity features), then classify. Add attention to let the model focus on suspicious transactions. Monitor for concept drift with a sliding evaluation window.

**Q7: Your LSTM training loss diverges after a few epochs. What do you check?**

> Systematic debugging: (1) **Gradient clipping** — did you set it? Try max_norm=0.5. (2) **Learning rate** — too high? Try 1e-4. (3) **NaN check** — any `inf` in inputs (log of 0, division by 0 in features)? (4) **Batch size** — very small batches have high variance. (5) **Weight initialization** — PyTorch defaults are usually fine, but explicit `xavier_uniform_` for linear layers helps. (6) **Layer norm** — add between LSTM output and downstream layers. Always plot gradient norms per layer to identify where the explosion occurs.

**Q8: You need to deploy a BiLSTM sentiment model that needs to process 10,000 reviews/minute. How do you scale?**

> Strategy: (1) **Batching**: batch reviews in groups of 64 (BiLSTM inference is highly parallelizable). (2) **ONNX export**: 2-3x faster inference than PyTorch eager mode. (3) **TorchScript**: for deployment without Python overhead. (4) **Horizontal scaling**: sentiment classification is stateless — run N replicas behind a load balancer. (5) **Async workers**: use Celery or similar for non-blocking processing. 10,000/min = ~167/sec, easily achievable with a single ONNX model on modern CPU. For higher throughput, use GPU batching.

**Q9: How would you handle very long documents (5,000+ words) with an LSTM?**

> Options: (1) **Hierarchical LSTM**: sentence-level LSTM produces sentence embeddings, document-level LSTM processes those. (2) **Truncation**: use the first/last N tokens (often adequate for classification). (3) **Chunking with aggregation**: split into overlapping windows, run LSTM on each, aggregate with mean/max pooling. (4) **Longformer or BigBird**: Transformer variants designed for long sequences — often superior for 1000+ token inputs. (5) **Sliding window with state transfer**: carry the hidden state from one chunk to the next for continuity.
