### 1.1 Representation Learning & Autoencoders

#### a. Intuition

Imagine you're trying to describe a human face to someone over a slow phone connection. You don't send a raw photo — that's too much data. Instead, you say: *"tall forehead, narrow eyes, sharp jawline, brown skin, slight smile."* That compressed description is the **representation** (or **latent code**) of the face. From that description, the listener reconstructs a mental image.

An **autoencoder** is a neural network that learns to do exactly this — automatically. It compresses input data into a compact representation, then decompresses it back to the original. The magic is that it learns *what features matter* without any labels.

The two halves are:
- **Encoder**: Input → Compressed code (bottleneck)
- **Decoder**: Compressed code → Reconstructed input

The bottleneck forces the network to learn the most important structure in the data.

#### b. Mathematical Insight

Given input **x**, the autoencoder learns:

```
z = f_encoder(x)        # encoding: x → z (latent space)
x̂ = f_decoder(z)       # decoding: z → x̂ (reconstruction)
```

The training objective is to minimize **reconstruction loss**:

```
L = ||x - x̂||²         # Mean Squared Error (for continuous data)
L = -Σ [x·log(x̂) + (1-x)·log(1-x̂)]   # Binary Cross-Entropy (for binary/image data)
```

The network learns parameters θ (encoder) and φ (decoder) to minimize this loss end-to-end via backpropagation. There are no labels — the input itself serves as the supervision signal.

#### c. How It Works (Step-by-Step)

1. **Forward pass**: Feed input x through the encoder → get latent code z
2. **Decode**: Feed z through the decoder → get reconstruction x̂
3. **Compute loss**: Measure difference between x and x̂
4. **Backpropagate**: Gradients flow through decoder, then encoder
5. **Update weights**: Both encoder and decoder improve simultaneously
6. **Repeat**: Until reconstruction loss converges

#### d. Visual Representation

```
Input          Encoder             Latent         Decoder          Output
 x             (compress)          Space          (expand)          x̂
                                     z
[784-dim]  →  [512] → [256]  →  [32-dim]  →  [256] → [512]  →  [784-dim]
 (image)      (hidden layers)    (bottleneck)   (hidden layers)   (reconstructed)

                        ↑ Information Bottleneck ↑
                     Forces learning of structure
```

```
Loss = ||x - x̂||²
Gradient flows: x̂ → Decoder → z → Encoder → x
```

#### e. Python Implementation

```python
import numpy as np
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader
from torchvision import datasets, transforms
import matplotlib.pyplot as plt


# ─────────────────────────────────────────────
# 1. Autoencoder Architecture
# ─────────────────────────────────────────────

class Autoencoder(nn.Module):
    """
    Vanilla fully-connected autoencoder.
    
    Args:
        input_dim: size of input (e.g., 784 for 28x28 MNIST images)
        latent_dim: size of the bottleneck representation
    """
    def __init__(self, input_dim: int = 784, latent_dim: int = 32):
        super().__init__()
        
        # Encoder: compresses input to latent space
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, 512),
            nn.ReLU(),
            nn.Linear(512, 256),
            nn.ReLU(),
            nn.Linear(256, latent_dim),  # bottleneck
        )
        
        # Decoder: reconstructs from latent space
        self.decoder = nn.Sequential(
            nn.Linear(latent_dim, 256),
            nn.ReLU(),
            nn.Linear(256, 512),
            nn.ReLU(),
            nn.Linear(512, input_dim),
            nn.Sigmoid(),  # output in [0, 1] for normalized images
        )

    def encode(self, x: torch.Tensor) -> torch.Tensor:
        return self.encoder(x)

    def decode(self, z: torch.Tensor) -> torch.Tensor:
        return self.decoder(z)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        z = self.encode(x)
        x_hat = self.decode(z)
        return x_hat


# ─────────────────────────────────────────────
# 2. Training Loop
# ─────────────────────────────────────────────

def train_autoencoder(model, dataloader, epochs=20, lr=1e-3, device='cpu'):
    """Train autoencoder with MSE reconstruction loss."""
    model = model.to(device)
    optimizer = optim.Adam(model.parameters(), lr=lr)
    loss_fn = nn.MSELoss()
    
    history = []
    for epoch in range(epochs):
        epoch_loss = 0.0
        for batch_x, _ in dataloader:  # ignore labels — unsupervised!
            batch_x = batch_x.view(batch_x.size(0), -1).to(device)  # flatten
            
            optimizer.zero_grad()
            x_hat = model(batch_x)          # forward pass
            loss = loss_fn(x_hat, batch_x)  # reconstruction loss
            loss.backward()                 # backprop
            optimizer.step()               # update weights
            
            epoch_loss += loss.item()
        
        avg_loss = epoch_loss / len(dataloader)
        history.append(avg_loss)
        if (epoch + 1) % 5 == 0:
            print(f"Epoch [{epoch+1}/{epochs}] Loss: {avg_loss:.4f}")
    
    return history


# ─────────────────────────────────────────────
# 3. Quick Demo (MNIST)
# ─────────────────────────────────────────────

def main():
    transform = transforms.Compose([transforms.ToTensor()])
    train_data = datasets.MNIST('./data', train=True, download=True, transform=transform)
    loader = DataLoader(train_data, batch_size=128, shuffle=True)
    
    model = Autoencoder(input_dim=784, latent_dim=32)
    history = train_autoencoder(model, loader, epochs=20)
    
    print(f"\nFinal reconstruction loss: {history[-1]:.4f}")
    print(f"Latent space dimensionality: 32 (from 784) = {32/784*100:.1f}% of original")

if __name__ == "__main__":
    main()
```

#### f. When to Use / Avoid

| Use When | Avoid When |
|---|---|
| No labels available and you need to learn structure | You have labels — supervised methods will be stronger |
| Anomaly detection (high reconstruction error = anomaly) | You need probabilistic outputs or sampling |
| Dimensionality reduction with non-linear structure | PCA would suffice (linear relationships only) |
| Pre-training representations for downstream tasks | Real-time inference on very constrained hardware |
| Data compression for structured data | You need generated samples (use VAE or GAN) |

#### g. Key Hyperparameters

| Hyperparameter | Effect | Typical Range |
|---|---|---|
| `latent_dim` | Compression ratio; smaller = more compression, less fidelity | 2–256 depending on task |
| `learning_rate` | Speed of convergence | 1e-4 to 1e-3 with Adam |
| `batch_size` | Training stability vs. speed | 64–512 |
| Architecture depth | Representational capacity | 2–5 hidden layers |
| Activation functions | Non-linearity; ReLU almost always works | ReLU, LeakyReLU |
| Loss function | MSE for continuous; BCE for binary | Task-dependent |

### 1.2 Denoising Autoencoders

#### a. Intuition

A vanilla autoencoder can "cheat" — if the bottleneck is large enough, it simply learns to copy the input. The denoising autoencoder (DAE) makes the task harder and more meaningful: **deliberately corrupt the input, then ask the network to reconstruct the clean original.**

Think of it like teaching a student by showing them smudged handwriting and asking them to reconstruct what was written clearly. To do well, the student *must* understand the underlying structure of letters — they can't just memorize the smudges.

This corruption-and-reconstruction objective forces the model to learn **robust, stable representations** — ones that capture the data's true structure, not incidental noise.

#### b. Mathematical Insight

Let x be the clean input. We apply a corruption process to get x̃:

```
x̃ = corrupt(x)     # add noise, zero out pixels, etc.
z  = encoder(x̃)    # encode the corrupted version
x̂  = decoder(z)    # try to reconstruct the CLEAN original
```

Loss objective: reconstruct the **clean** x from the **corrupted** x̃:

```
L_DAE = ||x - decoder(encoder(x̃))||²
```

This is subtly but importantly different from a vanilla autoencoder. The gradient signal now pushes the model toward representations that are invariant to the type of corruption used.

Common corruption strategies:
- **Gaussian noise**: x̃ = x + ε, where ε ~ N(0, σ²)
- **Masking noise**: randomly zero out p% of input dimensions
- **Salt-and-pepper noise**: randomly set pixels to 0 or 1

#### c. How It Works (Step-by-Step)

1. Take clean input x from the dataset
2. Apply corruption to create x̃ (e.g., add Gaussian noise)
3. Feed x̃ through the encoder → z
4. Feed z through the decoder → x̂
5. Compute reconstruction loss between x̂ and the **clean** x (not x̃)
6. Backpropagate and update
7. At inference: optionally pass clean or noisy data through the encoder

#### d. Visual Representation

```
Clean Input (x)          Corrupted Input (x̃)
  [████████]    →  add noise  →   [██░████░]
                                       │
                                   Encoder
                                       │
                                   Latent z
                                       │
                                   Decoder
                                       │
                               Reconstruction x̂
                                       │
                        Loss = ||x - x̂||²   ← compare with CLEAN x
```

The key insight: the model never sees the reconstruction loss against x̃. It must learn to "fill in the gaps" from the clean data's structure.

#### e. Python Implementation

```python
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader
from torchvision import datasets, transforms


class DenoisingAutoencoder(nn.Module):
    """
    Denoising Autoencoder.
    Learns robust representations by reconstructing clean inputs from corrupted ones.
    """
    def __init__(self, input_dim: int = 784, latent_dim: int = 64):
        super().__init__()
        
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, 512),
            nn.ReLU(),
            nn.Dropout(0.2),          # additional regularization
            nn.Linear(512, 256),
            nn.ReLU(),
            nn.Linear(256, latent_dim),
        )
        
        self.decoder = nn.Sequential(
            nn.Linear(latent_dim, 256),
            nn.ReLU(),
            nn.Linear(256, 512),
            nn.ReLU(),
            nn.Linear(512, input_dim),
            nn.Sigmoid(),
        )

    def forward(self, x_corrupted: torch.Tensor) -> torch.Tensor:
        z = self.encoder(x_corrupted)
        return self.decoder(z)


def add_gaussian_noise(x: torch.Tensor, noise_factor: float = 0.3) -> torch.Tensor:
    """Add Gaussian noise to input, then clamp to valid range [0, 1]."""
    noise = torch.randn_like(x) * noise_factor
    return torch.clamp(x + noise, 0.0, 1.0)


def add_masking_noise(x: torch.Tensor, mask_fraction: float = 0.3) -> torch.Tensor:
    """Randomly zero out a fraction of input dimensions."""
    mask = torch.bernoulli(torch.ones_like(x) * (1 - mask_fraction))
    return x * mask


def train_denoising_autoencoder(
    model, 
    dataloader, 
    epochs=20, 
    lr=1e-3, 
    noise_type='gaussian',
    noise_factor=0.3,
    device='cpu'
):
    """
    Train the DAE.
    
    Key difference from vanilla AE: we corrupt x before feeding to model,
    but compute loss against the CLEAN x.
    """
    model = model.to(device)
    optimizer = optim.Adam(model.parameters(), lr=lr)
    loss_fn = nn.MSELoss()
    
    noise_fn = add_gaussian_noise if noise_type == 'gaussian' else add_masking_noise
    
    for epoch in range(epochs):
        epoch_loss = 0.0
        for x_clean, _ in dataloader:
            x_clean = x_clean.view(x_clean.size(0), -1).to(device)
            
            # Step 1: Corrupt the input
            x_noisy = noise_fn(x_clean, noise_factor)
            
            # Step 2: Reconstruct from corrupted version
            x_hat = model(x_noisy)
            
            # Step 3: Loss against CLEAN input (critical distinction)
            loss = loss_fn(x_hat, x_clean)
            
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()
            
            epoch_loss += loss.item()
        
        avg_loss = epoch_loss / len(dataloader)
        if (epoch + 1) % 5 == 0:
            print(f"Epoch [{epoch+1}/{epochs}] DAE Loss: {avg_loss:.4f}")
```

#### f. When to Use / Avoid

| Use When | Avoid When |
|---|---|
| You want more robust representations than vanilla AE | The data has no natural noise structure to exploit |
| Pre-training for downstream classification | Noise type significantly differs from real corruption |
| Image denoising as an end goal | The task requires exact pixel-level fidelity |
| Learning features robust to missing data | Computational cost must be minimized |

#### g. Key Hyperparameters

| Hyperparameter | Effect |
|---|---|
| `noise_factor` / `mask_fraction` | Higher = harder task, more robust features, but may hurt if too high |
| `noise_type` | Gaussian for continuous data; masking for structured missingness |
| `latent_dim` | Same as vanilla AE — controls compression |
| Dropout in encoder | Complementary regularization to the noise corruption |

### 1.3 Sparse Autoencoders

#### a. Intuition

Neuroscience tells us that in the brain, only a small fraction of neurons fire for any given stimulus. This **sparse coding** is believed to be efficient and interpretable — each neuron specializes in detecting a specific feature.

Sparse autoencoders impose this same principle on neural networks. The constraint: at any given time, only a **small number of neurons in the latent layer should be active**. The rest should be near zero.

Why does this help? It forces the model to learn **disentangled, specialized features** — each latent dimension captures one specific pattern. This makes representations more interpretable and prevents neurons from encoding redundant or overlapping information.

#### b. Mathematical Insight

The sparse autoencoder adds a **sparsity penalty** to the reconstruction loss:

```
L_sparse = L_reconstruction + β · Ω(z)
```

Where Ω(z) is the sparsity penalty. Two common forms:

**L1 regularization** (promotes exact zeros):
```
Ω(z) = ||z||₁ = Σᵢ |zᵢ|
```

**KL-divergence sparsity** (constrains average activation):
```
Ω(z) = KL(ρ || ρ̂) = Σⱼ [ρ·log(ρ/ρ̂ⱼ) + (1-ρ)·log((1-ρ)/(1-ρ̂ⱼ))]
```

Where ρ is the target sparsity (e.g., 0.05, meaning neurons fire 5% of the time) and ρ̂ⱼ is the observed average activation of neuron j across the batch.

β controls the strength of the sparsity constraint. Higher β → sparser but potentially worse reconstruction.

#### c. How It Works (Step-by-Step)

1. Forward pass: x → encoder → z → decoder → x̂
2. Compute reconstruction loss: ||x - x̂||²
3. Compute average activation ρ̂ⱼ for each neuron j in the latent layer across the batch
4. Compute KL penalty: how much does ρ̂ differ from target ρ?
5. Total loss = reconstruction + β × sparsity penalty
6. Backpropagate total loss
7. Update weights — the penalty pushes most activations toward zero

#### d. Visual Representation

```
Latent Activations (before sparsity):    [0.8, 0.7, 0.6, 0.5, 0.4, 0.3, ...]
                                          ← many neurons active

Latent Activations (after sparsity):    [0.0, 0.0, 0.9, 0.0, 0.0, 0.8, ...]
                                          ← only a few neurons active
                                          ← each captures ONE specific feature

Feature specialization:
  z[2] = 0.9 → "detects diagonal edges"
  z[5] = 0.8 → "detects circular blobs"
  z[others] ≈ 0 → inactive for this input
```

#### e. Python Implementation

```python
import torch
import torch.nn as nn
import torch.optim as optim


class SparseAutoencoder(nn.Module):
    """
    Sparse Autoencoder using KL-divergence sparsity constraint.
    
    Forces most latent neurons to be near-zero, creating specialized features.
    """
    def __init__(self, input_dim: int = 784, latent_dim: int = 128):
        super().__init__()
        
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, 256),
            nn.ReLU(),
            nn.Linear(256, latent_dim),
            nn.Sigmoid(),  # IMPORTANT: Sigmoid ensures activations in [0,1] for KL penalty
        )
        
        self.decoder = nn.Sequential(
            nn.Linear(latent_dim, 256),
            nn.ReLU(),
            nn.Linear(256, input_dim),
            nn.Sigmoid(),
        )

    def forward(self, x: torch.Tensor):
        z = self.encoder(x)
        x_hat = self.decoder(z)
        return x_hat, z  # return both reconstruction AND latent code


def kl_divergence_sparsity(z: torch.Tensor, rho: float = 0.05) -> torch.Tensor:
    """
    KL divergence between target sparsity ρ and observed average activation ρ̂.
    
    Penalizes neurons that fire more (or less) than the target rate ρ.
    
    Args:
        z: latent activations [batch_size, latent_dim]
        rho: target average activation (sparsity target, e.g., 0.05 = 5%)
    
    Returns:
        KL penalty (scalar)
    """
    # Average activation of each neuron across the batch
    rho_hat = z.mean(dim=0)  # shape: [latent_dim]
    
    # Clamp to avoid log(0)
    rho_hat = torch.clamp(rho_hat, 1e-8, 1 - 1e-8)
    
    # KL(ρ || ρ̂) for each neuron, summed across latent dimensions
    kl = rho * torch.log(rho / rho_hat) + (1 - rho) * torch.log((1 - rho) / (1 - rho_hat))
    return kl.sum()


def train_sparse_autoencoder(
    model, 
    dataloader, 
    epochs=20, 
    lr=1e-3, 
    beta=1e-3,    # sparsity penalty weight
    rho=0.05,     # target sparsity
    device='cpu'
):
    """
    Train with combined reconstruction + sparsity loss.
    
    beta controls the trade-off:
    - Low beta → prioritize reconstruction (less sparse)
    - High beta → prioritize sparsity (potentially worse reconstruction)
    """
    model = model.to(device)
    optimizer = optim.Adam(model.parameters(), lr=lr)
    
    for epoch in range(epochs):
        total_loss = recon_loss_total = sparse_loss_total = 0.0
        
        for x, _ in dataloader:
            x = x.view(x.size(0), -1).to(device)
            
            # Forward
            x_hat, z = model(x)
            
            # Reconstruction loss
            recon_loss = nn.MSELoss()(x_hat, x)
            
            # Sparsity penalty
            sparse_loss = kl_divergence_sparsity(z, rho=rho)
            
            # Combined loss
            loss = recon_loss + beta * sparse_loss
            
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()
            
            total_loss += loss.item()
            recon_loss_total += recon_loss.item()
            sparse_loss_total += sparse_loss.item()
        
        n = len(dataloader)
        if (epoch + 1) % 5 == 0:
            print(f"Epoch {epoch+1} | Total: {total_loss/n:.4f} | "
                  f"Recon: {recon_loss_total/n:.4f} | "
                  f"Sparse: {sparse_loss_total/n:.4f}")


def analyze_sparsity(model, dataloader, device='cpu'):
    """Analyze how sparse the learned representations actually are."""
    model.eval()
    all_activations = []
    
    with torch.no_grad():
        for x, _ in dataloader:
            x = x.view(x.size(0), -1).to(device)
            _, z = model(x)
            all_activations.append(z.cpu())
    
    all_z = torch.cat(all_activations, dim=0)  # [N, latent_dim]
    
    avg_activation = all_z.mean(dim=0)  # average per neuron
    fraction_active = (all_z > 0.1).float().mean(dim=0)  # fraction of samples where neuron fires
    
    print(f"Average activation: {avg_activation.mean():.4f}")
    print(f"Neurons with avg activation < 0.1: {(avg_activation < 0.1).sum().item()}")
    print(f"Average fraction of samples where any neuron fires: {fraction_active.mean():.4f}")
    
    return avg_activation, fraction_active
```

| Use When | Avoid When |
|---|---|
| You need interpretable, specialized features | Perfect reconstruction is more important than sparsity |
| Feature selection / finding disentangled factors | Very low-dimensional latent spaces (not enough neurons to be sparse) |
| Inspired by neuroscience (dictionary learning) | Computational cost is a concern (β tuning adds overhead) |
| Over-complete autoencoders (latent_dim > input_dim) | You need probabilistic sampling |

#### g. Key Hyperparameters

| Hyperparameter | Effect |
|---|---|
| `rho` | Target sparsity. 0.05 means neurons should fire ~5% of the time |
| `beta` | Weight of sparsity penalty vs. reconstruction. Tune carefully |
| `latent_dim` | Can be larger than input_dim in sparse setting (over-complete) |
| Sigmoid vs. ReLU | Sigmoid enforces [0,1] range needed for KL penalty |

### 1.4 Variational Autoencoders (VAE)

#### a. Intuition

Here's the problem with regular autoencoders for **generation**: the latent space is not smooth or structured. If you pick a random point z in the latent space and decode it, you often get garbage — because the encoder may have learned a fragmented, discontinuous mapping.

A **Variational Autoencoder** (VAE) fixes this by making the latent space *probabilistic*. Instead of encoding x to a fixed point z, it encodes x to a **distribution** over z — specifically, a Gaussian with mean μ and variance σ².

Then, to get a latent code, you **sample** from that distribution. This forces the model to learn a smooth, continuous latent space where nearby points decode to semantically similar outputs.

The result: you can sample *any* point from the latent space and get a coherent, realistic output. This is what makes VAEs **generative** models — they learn the distribution of the data, not just how to compress and reconstruct it.

#### b. Mathematical Insight

The VAE optimizes the **Evidence Lower BOund (ELBO)**:

```
ELBO = E[log p(x|z)] - KL(q(z|x) || p(z))
         ↑                    ↑
  Reconstruction term    Regularization term
  (maximize likelihood)  (keep encoder close to standard Normal)
```

In practice, we **minimize** the negative ELBO:

```
L_VAE = L_reconstruction + β · KL(q(z|x) || N(0, I))
```

Where the KL divergence has a closed-form solution for Gaussians:

```
KL = -½ · Σᵢ [1 + log(σᵢ²) - μᵢ² - σᵢ²]
```

The **reparameterization trick** makes this differentiable:

```
z = μ + σ · ε,   ε ~ N(0, I)
```

Instead of sampling z directly (not differentiable), we sample ε (a fixed noise) and compute z via a differentiable operation. Gradients can flow through μ and σ.

#### c. How It Works (Step-by-Step)

1. **Encode**: Feed x through encoder → output (μ, log σ²)
2. **Sample**: z = μ + exp(0.5 · log σ²) · ε, where ε ~ N(0, I)
3. **Decode**: Feed z through decoder → x̂
4. **Reconstruction loss**: MSE or BCE between x̂ and x
5. **KL loss**: Penalize deviation of q(z|x) from N(0, I)
6. **Total loss**: reconstruction + β × KL
7. **Backprop**: Gradients flow through z via reparameterization trick

#### d. Visual Representation

```
               ┌─────────────────────────────────────────┐
               │             ENCODER                      │
    x  →  [Conv/Linear layers]  →  μ, log σ²             │
               └─────────────────────────────────────────┘
                                      │
                    Reparameterization trick:
                    z = μ + σ·ε,  ε ~ N(0,I)  ← differentiable!
                                      │
               ┌─────────────────────────────────────────┐
               │             DECODER                      │
    z  →  [Linear/Conv layers]  →  x̂                    │
               └─────────────────────────────────────────┘
                    
Loss = ||x - x̂||² + β·KL(N(μ,σ²) || N(0,1))
         ↑ reconstruction         ↑ regularization

Latent Space (2D example):
  ┌─────────────────────────┐
  │    . .  5s  . .         │ ← digits form clusters
  │  .3s.  .  .2s.          │   but clusters OVERLAP smoothly
  │    .  .1s.  .           │ ← can interpolate between clusters
  │  .0s. .   .9s.          │   and get valid outputs
  └─────────────────────────┘
```

#### e. Python Implementation

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
import torch.optim as optim
from torch.utils.data import DataLoader
from torchvision import datasets, transforms


class VAE(nn.Module):
    """
    Variational Autoencoder.
    
    Learns a probabilistic latent space, enabling:
    1. Generation of new samples
    2. Smooth interpolation in latent space
    3. Structured, meaningful representations
    """
    def __init__(self, input_dim: int = 784, hidden_dim: int = 256, latent_dim: int = 20):
        super().__init__()
        self.latent_dim = latent_dim
        
        # Encoder: maps input to distribution parameters
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
        )
        # Separate heads for mean and log-variance
        self.fc_mu = nn.Linear(hidden_dim, latent_dim)       # μ(x)
        self.fc_log_var = nn.Linear(hidden_dim, latent_dim)  # log σ²(x)
        
        # Decoder: maps latent z to reconstruction
        self.decoder = nn.Sequential(
            nn.Linear(latent_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, hidden_dim),
            nn.ReLU(),
            nn.Linear(hidden_dim, input_dim),
            nn.Sigmoid(),
        )

    def encode(self, x: torch.Tensor):
        """Encode to distribution parameters (μ, log σ²)."""
        h = self.encoder(x)
        mu = self.fc_mu(h)
        log_var = self.fc_log_var(h)
        return mu, log_var

    def reparameterize(self, mu: torch.Tensor, log_var: torch.Tensor) -> torch.Tensor:
        """
        Reparameterization trick: z = μ + σ·ε
        
        This is the key to making VAE differentiable.
        We sample ε from N(0, I) and compute z deterministically.
        Gradients can flow through μ and σ (but not through ε sampling).
        """
        if self.training:
            std = torch.exp(0.5 * log_var)  # σ = exp(0.5 · log σ²)
            eps = torch.randn_like(std)      # ε ~ N(0, I)
            return mu + eps * std            # z = μ + ε·σ
        else:
            return mu  # at inference, use the mean directly

    def decode(self, z: torch.Tensor) -> torch.Tensor:
        return self.decoder(z)

    def forward(self, x: torch.Tensor):
        mu, log_var = self.encode(x)
        z = self.reparameterize(mu, log_var)
        x_hat = self.decode(z)
        return x_hat, mu, log_var

    def sample(self, n_samples: int, device='cpu') -> torch.Tensor:
        """Generate new samples by sampling from the prior N(0, I)."""
        z = torch.randn(n_samples, self.latent_dim).to(device)
        with torch.no_grad():
            return self.decode(z)

    def interpolate(self, x1: torch.Tensor, x2: torch.Tensor, steps: int = 10) -> torch.Tensor:
        """Interpolate between two inputs in latent space."""
        mu1, _ = self.encode(x1)
        mu2, _ = self.encode(x2)
        
        interpolated = []
        for alpha in torch.linspace(0, 1, steps):
            z = (1 - alpha) * mu1 + alpha * mu2  # linear interpolation
            x_interp = self.decode(z)
            interpolated.append(x_interp)
        
        return torch.stack(interpolated)


def vae_loss(x: torch.Tensor, x_hat: torch.Tensor, mu: torch.Tensor, 
             log_var: torch.Tensor, beta: float = 1.0):
    """
    VAE Loss = Reconstruction Loss + β · KL Divergence
    
    - Reconstruction: how well did we reconstruct the input?
    - KL: how close is q(z|x) to the prior p(z) = N(0, I)?
    
    beta=1: standard VAE
    beta>1: β-VAE, emphasizes disentanglement
    """
    # Binary cross-entropy for reconstruction (pixel-wise)
    recon_loss = F.binary_cross_entropy(x_hat, x, reduction='sum')
    
    # KL divergence (closed form for Gaussians)
    # KL(N(μ,σ²) || N(0,1)) = -½ Σ(1 + log σ² - μ² - σ²)
    kl_loss = -0.5 * torch.sum(1 + log_var - mu.pow(2) - log_var.exp())
    
    return recon_loss + beta * kl_loss, recon_loss, kl_loss


def train_vae(model, dataloader, epochs=30, lr=1e-3, beta=1.0, device='cpu'):
    """Train the VAE with ELBO objective."""
    model = model.to(device)
    optimizer = optim.Adam(model.parameters(), lr=lr)
    
    for epoch in range(epochs):
        total_loss = recon_total = kl_total = 0.0
        
        for x, _ in dataloader:
            x = x.view(x.size(0), -1).to(device)
            
            x_hat, mu, log_var = model(x)
            loss, recon_loss, kl_loss = vae_loss(x, x_hat, mu, log_var, beta=beta)
            
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()
            
            total_loss += loss.item()
            recon_total += recon_loss.item()
            kl_total += kl_loss.item()
        
        n = len(dataloader.dataset)
        if (epoch + 1) % 5 == 0:
            print(f"Epoch {epoch+1} | Total: {total_loss/n:.2f} | "
                  f"Recon: {recon_total/n:.2f} | KL: {kl_total/n:.2f}")
```

### 1.5 Latent Space Intuition

#### a. Intuition

The latent space is the "internal language" a model uses to describe data. Think of it as a map, where each point represents a different data sample, and **nearby points represent similar things**.

In a well-trained model:
- All images of "3" cluster together
- The cluster for "3" is near the cluster for "8" (similar shapes), far from "1"
- Moving in one direction in latent space changes one attribute (e.g., digit thickness)
- Moving in another direction changes a different attribute (e.g., slant)

This is the dream of **disentanglement** — a latent space where dimensions have interpretable semantic meaning.

#### b. Mathematical Insight

The latent space is a manifold embedded in ℝᵈ (d-dimensional real space). The encoder learns a mapping:

```
f: X → Z   (data space to latent space)
```

For this to be useful, we want the manifold to be:
- **Continuous**: nearby points in Z decode to similar outputs
- **Complete**: every point in Z decodes to a valid output
- **Structured**: semantically similar inputs map to nearby regions in Z

Only VAEs and GANs explicitly enforce these properties. Vanilla AEs just learn compression — the latent space may be fragmented.

#### c. Visual Representation

```
Vanilla AE latent space:           VAE latent space:
  (2D projection)                    (2D projection)

  ·· 3s ··                            ...333...
  5s    ·  ·                         .555..222.
  ·· 2s    8s ·                      ..888..77.
     ·  1s                            .......  
  (scattered, sparse, gaps)          (continuous, smooth, no gaps)

     ↑ Random point here                ↑ Random point here
     might decode to garbage            will decode to valid output
```

#### d. Key Properties to Understand

**Interpolation**: In a good latent space, linear interpolation between two points produces a meaningful transition:
```
z_interp = (1-α)·z₁ + α·z₂,   α ∈ [0, 1]
```

Moving from z₁ (image of "2") to z₂ (image of "5") should produce intermediate images that gradually transform from 2 to 5.

**Arithmetic**: Famous example from Word2Vec: King - Man + Woman ≈ Queen. This works because the latent space captures semantic structure. The same can happen with image VAEs: Smiling Woman - Woman + Man ≈ Smiling Man.

### 1.6 Reconstruction vs Generation

#### a. Intuition

This distinction is critical and often confused:

**Reconstruction** (Autoencoders, DAE, Sparse AE):
- Given an input x, produce a faithful copy x̂
- The model has seen x; it's trying to reproduce it
- Quality metric: how close is x̂ to x?

**Generation** (VAE, GAN):
- Produce a new, realistic sample that was never seen during training
- The model is sampling from a learned distribution P(x)
- Quality metric: does it look realistic? Does it have diversity?

Think of it as the difference between a **photocopier** (reconstruction) and a **novelist** (generation). The novelist writes original stories in the style they've learned; they don't reproduce text they've read.

#### b. Technical Distinction

```
Reconstruction model:
  Training: minimize ||x - AE(x)||
  Inference: x → [Encoder] → z → [Decoder] → x̂
  Usage: compression, denoising, anomaly detection

Generation model:
  Training: minimize -ELBO (VAE) or adversarial loss (GAN)
  Inference: z ~ p(z) → [Decoder/Generator] → x_new
  Usage: data augmentation, creative generation, density estimation
```

A VAE does **both**: it reconstructs during training (ELBO), but can also generate at inference by sampling from N(0, I).

### 1.7 Generative Adversarial Networks (GANs)

#### a. Intuition

GANs are one of the most creative ideas in deep learning. The setup is a **game between two adversaries**:

- **Generator (G)**: A counterfeiter who creates fake data (images, audio, etc.) from random noise. Gets better over time at fooling the Detective.
- **Discriminator (D)**: A detective who tries to distinguish real data from the Generator's fakes.

They train together in a zero-sum game: the better the Generator gets at fooling the Discriminator, the harder the Discriminator has to try. The better the Discriminator gets at detecting fakes, the better the Generator has to become. Over time, the Generator learns to produce outputs so realistic that the Discriminator can't tell real from fake.

At equilibrium (Nash equilibrium), the Generator has learned the true data distribution P(x). The Discriminator outputs 0.5 for everything — it literally cannot distinguish real from fake.

#### b. Mathematical Insight

The GAN objective is a minimax game:

```
min_G max_D V(D, G) = E_{x~p_data}[log D(x)] + E_{z~p_z}[log(1 - D(G(z)))]
```

Breaking this down:
- `E[log D(x)]`: Discriminator should output 1 (real) for real data — maximize this
- `E[log(1 - D(G(z)))]`: Discriminator should output 0 (fake) for generated data — maximize this
- Generator wants to minimize this: make D(G(z)) → 1 (fool the Discriminator)

In practice, Generator loss is often replaced with:

```
L_G = -E[log D(G(z))]   ← non-saturating loss, avoids vanishing gradients
```

#### c. How It Works (Step-by-Step)

**Training alternates between two phases:**

**Phase 1: Train Discriminator** (freeze Generator)
1. Sample real data x from training set
2. Sample noise z ~ N(0, I), generate fake data x_fake = G(z)
3. Compute D(x) → should be close to 1 (real)
4. Compute D(x_fake) → should be close to 0 (fake)
5. Loss: -[log D(x) + log(1 - D(x_fake))]
6. Update Discriminator weights only

**Phase 2: Train Generator** (freeze Discriminator)
1. Sample noise z ~ N(0, I)
2. Generate x_fake = G(z)
3. Run x_fake through (frozen) Discriminator: D(x_fake)
4. Loss: -log(D(x_fake)) — want D to think the fake is real
5. Update Generator weights only

Repeat these two phases for thousands of iterations.

#### d. Visual Representation

```
                    ┌──────────────────┐
 z ~ N(0,I) ──────►│   GENERATOR (G)  │──────────► x_fake
 (random noise)     └──────────────────┘
                                                        │
                    ┌──────────────────┐               │
 x_real ────────────┤  DISCRIMINATOR   │◄──────────────┘
 (from dataset)     │      (D)         │
                    └──────────────────┘
                            │
                   D(x) → P(real | input)
                   
Training signals:
  For D: D(x_real) → 1, D(x_fake) → 0
  For G: D(G(z)) → 1  (fool D into thinking it's real)
  
Over training:
  Early:  G produces blurry noise
  Mid:    G produces blurry but recognizable shapes
  Late:   G produces sharp, photorealistic outputs
```

#### e. Python Implementation

```python
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader
from torchvision import datasets, transforms
import numpy as np


# ─────────────────────────────────────────────
# 1. Generator Architecture
# ─────────────────────────────────────────────

class Generator(nn.Module):
    """
    Fully-connected Generator for MNIST.
    
    Maps random noise z → fake image.
    Uses BatchNorm for training stability.
    """
    def __init__(self, latent_dim: int = 100, output_dim: int = 784):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(latent_dim, 256),
            nn.BatchNorm1d(256),
            nn.LeakyReLU(0.2),
            
            nn.Linear(256, 512),
            nn.BatchNorm1d(512),
            nn.LeakyReLU(0.2),
            
            nn.Linear(512, 1024),
            nn.BatchNorm1d(1024),
            nn.LeakyReLU(0.2),
            
            nn.Linear(1024, output_dim),
            nn.Tanh(),  # outputs in [-1, 1]; normalize real data to match
        )

    def forward(self, z: torch.Tensor) -> torch.Tensor:
        return self.net(z)


# ─────────────────────────────────────────────
# 2. Discriminator Architecture
# ─────────────────────────────────────────────

class Discriminator(nn.Module):
    """
    Fully-connected Discriminator.
    
    Maps image → probability of being real.
    Uses Dropout for regularization (prevents D from being too perfect).
    """
    def __init__(self, input_dim: int = 784):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(input_dim, 1024),
            nn.LeakyReLU(0.2),
            nn.Dropout(0.3),
            
            nn.Linear(1024, 512),
            nn.LeakyReLU(0.2),
            nn.Dropout(0.3),
            
            nn.Linear(512, 256),
            nn.LeakyReLU(0.2),
            nn.Dropout(0.3),
            
            nn.Linear(256, 1),
            nn.Sigmoid(),  # output: probability of being real
        )

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.net(x)


# ─────────────────────────────────────────────
# 3. GAN Training Loop
# ─────────────────────────────────────────────

def train_gan(
    generator, 
    discriminator, 
    dataloader, 
    epochs: int = 50,
    lr: float = 2e-4,
    latent_dim: int = 100,
    device: str = 'cpu',
    n_critic: int = 1   # how many D steps per G step
):
    """
    Standard GAN training loop.
    
    Key insight: we alternate between training D and G.
    D trains to maximize log(D(x)) + log(1-D(G(z)))
    G trains to maximize log(D(G(z)))  [non-saturating]
    """
    generator = generator.to(device)
    discriminator = discriminator.to(device)
    
    # Separate optimizers for G and D
    opt_G = optim.Adam(generator.parameters(), lr=lr, betas=(0.5, 0.999))
    opt_D = optim.Adam(discriminator.parameters(), lr=lr, betas=(0.5, 0.999))
    
    criterion = nn.BCELoss()
    
    # Fixed noise for visualization (to track G's progress)
    fixed_noise = torch.randn(64, latent_dim).to(device)
    
    history = {'d_loss': [], 'g_loss': []}
    
    for epoch in range(epochs):
        d_loss_epoch = g_loss_epoch = 0.0
        
        for real_x, _ in dataloader:
            batch_size = real_x.size(0)
            real_x = real_x.view(batch_size, -1).to(device)
            
            # Labels: 1 for real, 0 for fake
            real_labels = torch.ones(batch_size, 1).to(device)
            fake_labels = torch.zeros(batch_size, 1).to(device)
            
            # ──────────────────────────────────────
            # Phase 1: Train Discriminator
            # ──────────────────────────────────────
            # (train n_critic times for each G step)
            for _ in range(n_critic):
                # Real data loss
                d_real_pred = discriminator(real_x)
                d_real_loss = criterion(d_real_pred, real_labels)
                
                # Fake data loss
                z = torch.randn(batch_size, latent_dim).to(device)
                fake_x = generator(z).detach()  # .detach() = don't backprop into G
                d_fake_pred = discriminator(fake_x)
                d_fake_loss = criterion(d_fake_pred, fake_labels)
                
                d_loss = d_real_loss + d_fake_loss
                
                opt_D.zero_grad()
                d_loss.backward()
                opt_D.step()
            
            # ──────────────────────────────────────
            # Phase 2: Train Generator
            # ──────────────────────────────────────
            z = torch.randn(batch_size, latent_dim).to(device)
            fake_x = generator(z)
            
            # G wants D to predict fake images as REAL
            g_pred = discriminator(fake_x)
            g_loss = criterion(g_pred, real_labels)  # want D(G(z)) → 1
            
            opt_G.zero_grad()
            g_loss.backward()
            opt_G.step()
            
            d_loss_epoch += d_loss.item()
            g_loss_epoch += g_loss.item()
        
        n = len(dataloader)
        history['d_loss'].append(d_loss_epoch / n)
        history['g_loss'].append(g_loss_epoch / n)
        
        if (epoch + 1) % 10 == 0:
            print(f"Epoch [{epoch+1}/{epochs}] | D_loss: {d_loss_epoch/n:.4f} | "
                  f"G_loss: {g_loss_epoch/n:.4f}")
    
    return history


# ─────────────────────────────────────────────
# 4. Monitoring GAN Training Health
# ─────────────────────────────────────────────

def check_gan_health(d_loss: float, g_loss: float) -> str:
    """
    Quick heuristic to check if GAN training is on track.
    Ideal: both losses should hover around 0.693 (log(2)).
    """
    equilibrium = 0.693  # -log(0.5)
    
    if d_loss < 0.2:
        return "WARNING: D is too strong — G may be unable to learn"
    elif g_loss > 5.0:
        return "WARNING: G is failing badly — check architecture or lr"
    elif abs(d_loss - equilibrium) < 0.1 and abs(g_loss - equilibrium) < 0.1:
        return "HEALTHY: Near Nash equilibrium"
    else:
        return f"Training in progress — D:{d_loss:.3f}, G:{g_loss:.3f}"
```

### 1.7.1 Mode Collapse (Conceptual)

**Mode collapse** is the most notorious failure mode of GAN training. 

The Generator "discovers" that fooling the Discriminator only requires generating a small subset of the real data distribution. For example, on a digit dataset, the Generator might produce excellent-quality 7s and nothing else — because 7s are enough to fool the Discriminator.

**Why it happens**: The Generator is playing a game. If it finds a strategy that consistently fools D (even if the strategy is narrow), it has no incentive to explore other modes.

```
Real data distribution:
  p_data(x) = 10 classes of digits [0-9]
  
After mode collapse:
  p_G(x) ≈ just "7"s and "3"s
  
D can't distinguish: the 7s and 3s look real
G has no signal to generate other classes
```

**Signs of mode collapse**: Generated samples are diverse in batch but across training iterations, you see the same samples repeatedly. Diversity metrics (like FID or IS) plateau or worsen.

**Mitigation strategies**:
- **Minibatch discrimination**: Let D see batches, not just individual samples
- **Unrolled GANs**: G trains against future D states
- **Wasserstein GANs** (WGAN): Replace BCE loss with Wasserstein distance
- **Diversity-encouraging losses**: Penalize identical outputs

---

### 1.7.2 Training Instability (Conceptual)

GAN training is notoriously unstable for several reasons:

**Vanishing gradients**: If D becomes too good early in training, D(G(z)) ≈ 0 for all generated samples. Then log(1 - D(G(z))) ≈ log(1) = 0, and G receives zero gradient signal — it can't learn.

**Mode collapse** (as above).

**Oscillation**: G and D can enter a cycle where each defeats the other's recent improvements, never converging to equilibrium.

**Stabilization techniques**:

| Technique | Description |
|---|---|
| **Label smoothing** | Use 0.9 instead of 1.0 for real labels; prevents D from becoming overconfident |
| **Spectral normalization** | Normalize weight matrices in D to Lipschitz constant 1 |
| **Gradient penalty** (WGAN-GP) | Add penalty on gradients of D to enforce Lipschitz constraint |
| **Two time-scale update rule** | Different learning rates for G and D |
| **Progressive growing** | Start with low-res, progressively increase resolution |

### 1.8 GAN Variants

#### a. DCGAN (Deep Convolutional GAN)

**What it is**: Replace fully-connected layers with convolutional layers. This made GAN training significantly more stable and produced much better image results (2015, Radford et al.).

**Key architectural rules**:
- Replace pooling with strided convolutions (D) and fractional-strided (transposed) convolutions (G)
- Use BatchNorm in both G and D (except G output layer and D input layer)
- Use ReLU in G, LeakyReLU in D
- No fully-connected hidden layers

```python
class DCGANGenerator(nn.Module):
    """DCGAN Generator: maps z to image using transposed convolutions."""
    def __init__(self, latent_dim: int = 100, ngf: int = 64, n_channels: int = 1):
        super().__init__()
        self.net = nn.Sequential(
            # Input: z vector (latent_dim,) → reshape to (latent_dim, 1, 1) before this
            nn.ConvTranspose2d(latent_dim, ngf * 8, 4, 1, 0, bias=False),
            nn.BatchNorm2d(ngf * 8),
            nn.ReLU(True),
            # → (ngf*8, 4, 4)
            
            nn.ConvTranspose2d(ngf * 8, ngf * 4, 4, 2, 1, bias=False),
            nn.BatchNorm2d(ngf * 4),
            nn.ReLU(True),
            # → (ngf*4, 8, 8)
            
            nn.ConvTranspose2d(ngf * 4, ngf * 2, 4, 2, 1, bias=False),
            nn.BatchNorm2d(ngf * 2),
            nn.ReLU(True),
            # → (ngf*2, 16, 16)
            
            nn.ConvTranspose2d(ngf * 2, ngf, 4, 2, 1, bias=False),
            nn.BatchNorm2d(ngf),
            nn.ReLU(True),
            # → (ngf, 32, 32)
            
            nn.ConvTranspose2d(ngf, n_channels, 4, 2, 1, bias=False),
            nn.Tanh(),
            # → (n_channels, 64, 64)
        )

    def forward(self, z: torch.Tensor) -> torch.Tensor:
        z = z.view(z.size(0), -1, 1, 1)  # reshape to spatial
        return self.net(z)


class DCGANDiscriminator(nn.Module):
    """DCGAN Discriminator: classifies image as real or fake."""
    def __init__(self, ndf: int = 64, n_channels: int = 1):
        super().__init__()
        self.net = nn.Sequential(
            # Input: (n_channels, 64, 64)
            nn.Conv2d(n_channels, ndf, 4, 2, 1, bias=False),
            nn.LeakyReLU(0.2, inplace=True),
            # → (ndf, 32, 32)
            
            nn.Conv2d(ndf, ndf * 2, 4, 2, 1, bias=False),
            nn.BatchNorm2d(ndf * 2),
            nn.LeakyReLU(0.2, inplace=True),
            # → (ndf*2, 16, 16)
            
            nn.Conv2d(ndf * 2, ndf * 4, 4, 2, 1, bias=False),
            nn.BatchNorm2d(ndf * 4),
            nn.LeakyReLU(0.2, inplace=True),
            # → (ndf*4, 8, 8)
            
            nn.Conv2d(ndf * 4, ndf * 8, 4, 2, 1, bias=False),
            nn.BatchNorm2d(ndf * 8),
            nn.LeakyReLU(0.2, inplace=True),
            # → (ndf*8, 4, 4)
            
            nn.Conv2d(ndf * 8, 1, 4, 1, 0, bias=False),
            nn.Sigmoid(),
            # → (1, 1, 1) = probability
        )

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.net(x).view(-1, 1)
```

#### b. Conditional GAN (cGAN)

**What it is**: Condition both G and D on additional information y (e.g., a class label). This gives you **control** over what the Generator produces.

Regular GAN: z → G → x (random output)
Conditional GAN: (z, y) → G → x_y (output conditioned on class y)

```python
class ConditionalGenerator(nn.Module):
    """
    Conditional GAN Generator.
    Inputs: noise z + class label y (embedded)
    Output: image of the specified class
    """
    def __init__(self, latent_dim=100, n_classes=10, embed_dim=50, output_dim=784):
        super().__init__()
        
        # Embed the class label into a dense vector
        self.label_embedding = nn.Embedding(n_classes, embed_dim)
        
        self.net = nn.Sequential(
            nn.Linear(latent_dim + embed_dim, 256),
            nn.BatchNorm1d(256),
            nn.LeakyReLU(0.2),
            nn.Linear(256, 512),
            nn.BatchNorm1d(512),
            nn.LeakyReLU(0.2),
            nn.Linear(512, output_dim),
            nn.Tanh(),
        )

    def forward(self, z: torch.Tensor, labels: torch.Tensor) -> torch.Tensor:
        # Embed labels and concatenate with noise
        label_emb = self.label_embedding(labels)
        inp = torch.cat([z, label_emb], dim=1)
        return self.net(inp)


class ConditionalDiscriminator(nn.Module):
    """
    Conditional GAN Discriminator.
    Must distinguish real/fake while accounting for the conditioning label.
    """
    def __init__(self, input_dim=784, n_classes=10, embed_dim=50):
        super().__init__()
        
        self.label_embedding = nn.Embedding(n_classes, embed_dim)
        
        self.net = nn.Sequential(
            nn.Linear(input_dim + embed_dim, 512),
            nn.LeakyReLU(0.2),
            nn.Dropout(0.3),
            nn.Linear(512, 256),
            nn.LeakyReLU(0.2),
            nn.Dropout(0.3),
            nn.Linear(256, 1),
            nn.Sigmoid(),
        )

    def forward(self, x: torch.Tensor, labels: torch.Tensor) -> torch.Tensor:
        label_emb = self.label_embedding(labels)
        inp = torch.cat([x, label_emb], dim=1)
        return self.net(inp)
```

#### c. CycleGAN (Conceptual)

**What it is**: CycleGAN enables **unpaired image-to-image translation**. You have two domains (e.g., horses and zebras) with no paired examples, and you want to learn to translate between them.

**Key innovation**: Cycle consistency loss. If you translate image X to domain Y, then translate back to domain X, you should get the original image back.

```
x → G(x→y) → y_fake → F(y→x) → x_reconstructed ≈ x
```

Loss = adversarial_loss(D_Y, G) + adversarial_loss(D_X, F) + λ·||F(G(x)) - x||₁ + λ·||G(F(y)) - y||₁

Applications: Horse ↔ Zebra, Summer ↔ Winter, Photo ↔ Painting, Day ↔ Night, MRI ↔ CT.

```
                ┌──────────┐              ┌──────────┐
  x (Horse) ───►│  G: X→Y  │──► ŷ (Zebra)│  F: Y→X  │──► x̂ ≈ x
                └──────────┘              └──────────┘
                                  
Cycle consistency: ||F(G(x)) - x||₁  should be small
```

#### d. StyleGAN (Conceptual)

**What it is**: NVIDIA's StyleGAN (2019, Karras et al.) produces photorealistic high-resolution face images (1024×1024). It's the model behind "thispersondoesnotexist.com".

**Key innovations**:
1. **Mapping network**: Instead of feeding z directly to G, a mapping network transforms z → w (intermediate latent space), which is better disentangled
2. **AdaIN (Adaptive Instance Normalization)**: Style code w is injected at each layer via scale/shift normalization, controlling different levels of detail
3. **Progressive growing**: Training starts at 4×4 resolution, progressively doubles to 1024×1024
4. **Stochastic variation**: Independent Gaussian noise is added to each layer, controlling fine details (hair strands, pores)

```
z ~ N(0,I) → [Mapping Network (8 FC layers)] → w
                                                 │
                                         AdaIN injection
                                         at each layer
                                                 │
4×4 → 8×8 → 16×16 → ... → 1024×1024  ← Progressive layers
 ↑       ↑       ↑
Coarse style  Mid style  Fine style
(pose, shape) (hair, eyes) (texture, color)
```

## 4. Cross-Topic Relationships

Understanding how these topics connect is as important as understanding each topic individually.

```
                         REPRESENTATION LEARNING
                               │
                    ┌──────────┴──────────┐
                    │                     │
             RECONSTRUCTION           GENERATION
                    │                     │
          ┌─────────┼─────────┐           ├─────────────┐
          │         │         │           │             │
       Vanilla    Denoising  Sparse      VAE           GAN
          AE        AE        AE          │             │
          │         │         │      Structured    Adversarial
       Basic     Robust    Interpretable  Latent      Training
     compression features    features    Space          │
          │                              │         ┌────┴────┐
          └──────────────────────────────┘         │         │
                         │                       DCGAN     cGAN
                    Latent Space              CycleGAN  StyleGAN
                    Intuition
```

**Evolution of ideas:**

1. **Vanilla AE** → demonstrates that compression is possible without labels
2. **Denoising AE** → forces more robust features by corrupting input
3. **Sparse AE** → forces interpretable features by constraining activations
4. **VAE** → makes latent space continuous and probabilistic → enables generation
5. **GAN** → abandons reconstruction entirely, learns generation through competition
6. **DCGAN** → scales GANs to images with convolutional architecture
7. **cGAN** → adds controllability to GAN generation
8. **CycleGAN** → applies GAN idea to unpaired translation with cycle consistency
9. **StyleGAN** → achieves state-of-the-art quality through hierarchical style control

**Key tension**: Reconstruction quality vs. generation quality. Autoencoders excel at faithful reconstruction but struggle at diverse generation. GANs produce the sharpest generated images but can't reconstruct. VAEs sit in the middle — they generate and reconstruct, but generated images are sometimes blurry.

### Project 1: Manufacturing Defect Detection with Autoencoders

#### a. Problem Statement

A semiconductor manufacturer produces wafers and needs to detect surface defects automatically. **Defects are extremely rare** (< 1% of wafers), making it impractical to train a supervised classifier — there simply aren't enough labeled defect examples.

**Business question**: Can we flag anomalous wafers for human inspection without labeled defect data?

**Solution approach**: Train an autoencoder on **only normal wafers**. The model learns to reconstruct normal wafers well. When a defective wafer is fed in, the reconstruction error is high — because the autoencoder has never seen that pattern before. High error → anomaly.

#### b. Dataset

**Real dataset**: MVTec Anomaly Detection Dataset
- URL: [https://www.mvtec.com/company/research/datasets/mvtec-ad](https://www.mvtec.com/company/research/datasets/mvtec-ad)
- 15 categories of industrial objects and textures
- Training: normal samples only; Test: normal + anomalous

```python
def generate_wafer_dataset(
    n_normal: int = 5000,
    n_anomaly: int = 100,
    n_features: int = 64,
    random_state: int = 42
):
    """
    Simulate wafer surface measurements.
    Normal wafers: measurements from a tight distribution.
    Anomalous wafers: contain unusual spikes or shifted patterns.
    
    In practice, these would be 2D grayscale images of wafer surfaces.
    Here we simulate them as feature vectors for simplicity.
    """
    rng = np.random.RandomState(random_state)
    
    # Normal wafers: measurements drawn from consistent distribution
    X_normal = rng.randn(n_normal, n_features) * 0.3
    # Add some correlated structure (simulating surface patterns)
    X_normal += np.sin(np.linspace(0, 2 * np.pi, n_features))
    
    # Anomalous wafers: contain defects (spikes, shifts, etc.)
    X_anomaly = rng.randn(n_anomaly, n_features) * 0.3
    X_anomaly += np.sin(np.linspace(0, 2 * np.pi, n_features))
    
    # Add defect signatures: random spikes at random positions
    for i in range(n_anomaly):
        spike_idx = rng.randint(0, n_features, size=5)
        X_anomaly[i, spike_idx] += rng.uniform(2, 5, size=5)  # large spike

    y_anomaly = np.ones(n_anomaly)

    y = np.concatenate([y_normal, y_anomaly])

    X_min, X_max = X.min(axis=0), X.max(axis=0)
    X_normalized = (X - X_min) / (X_max - X_min + 1e-8)
    
    return X_normalized.astype(np.float32), y.astype(int), X_min, X_max
```

```python
class AnomalyAutoencoder(nn.Module):
    """
    Autoencoder for anomaly detection.
    
    Normal inputs → low reconstruction error
    Anomalous inputs → high reconstruction error
    
    The bottleneck forces the model to learn only normal patterns.
    Anomalous patterns don't fit this learned representation → high error.
    """
    def __init__(self, input_dim: int, latent_dim: int = 16):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Linear(input_dim, 64),
            nn.BatchNorm1d(64),
            nn.ReLU(),
            nn.Dropout(0.1),
            nn.Linear(64, 32),
            nn.ReLU(),
            nn.Linear(32, latent_dim),
        )
        self.decoder = nn.Sequential(
            nn.Linear(latent_dim, 32),
            nn.ReLU(),
            nn.Linear(32, 64),
            nn.BatchNorm1d(64),
            nn.ReLU(),
            nn.Linear(64, input_dim),
        )

    def forward(self, x):
        return self.decoder(self.encoder(x))

    def reconstruction_error(self, x: torch.Tensor) -> torch.Tensor:
        """Compute per-sample reconstruction error (MSE)."""
        with torch.no_grad():
            x_hat = self.forward(x)
            errors = ((x - x_hat) ** 2).mean(dim=1)
        return errors


def train_model(model, train_loader, epochs=50, lr=1e-3, device='cpu'):
    model = model.to(device)
    optimizer = optim.Adam(model.parameters(), lr=lr, weight_decay=1e-5)
    scheduler = optim.lr_scheduler.ReduceLROnPlateau(optimizer, patience=5)
    loss_fn = nn.MSELoss()
    
    train_losses = []
    for epoch in range(epochs):
        model.train()
        epoch_loss = 0.0
        for (batch_x,) in train_loader:
            batch_x = batch_x.to(device)
            x_hat = model(batch_x)
            loss = loss_fn(x_hat, batch_x)
            
            optimizer.zero_grad()
            loss.backward()
            optimizer.step()
            epoch_loss += loss.item()
        
        avg = epoch_loss / len(train_loader)
        train_losses.append(avg)
        scheduler.step(avg)
        
        if (epoch + 1) % 10 == 0:
            print(f"  Epoch {epoch+1}/{epochs} | Loss: {avg:.5f}")
    
    return train_losses

def evaluate_anomaly_detector(model, X_test_t, y_test, model_name="AE", device='cpu'):
    """
    Evaluate autoencoder-based anomaly detector.
    
    Key insight: we don't need a fixed threshold upfront.
    We evaluate using threshold-free metrics (AUC-ROC, AUC-PR).
    """
    X_test_t = X_test_t.to(device)
    model = model.to(device)
    
    # Reconstruction errors = anomaly scores
    errors = model.reconstruction_error(X_test_t).cpu().numpy()
    
    # Threshold-free metrics
    auc_roc = roc_auc_score(y_test, errors)
    auc_pr = average_precision_score(y_test, errors)
    
    # Threshold selection: maximize F1 on validation
    # In practice, this threshold would be set by business requirements
    # (e.g., "flag top 5% of samples for inspection")
    thresholds = np.percentile(errors, np.arange(80, 100, 1))
    best_f1 = 0
    best_threshold = None
    
    from sklearn.metrics import f1_score
    for thresh in thresholds:
        y_pred = (errors > thresh).astype(int)
        f1 = f1_score(y_test, y_pred, zero_division=0)
        if f1 > best_f1:
            best_f1 = f1
            best_threshold = thresh
    
    y_pred_best = (errors > best_threshold).astype(int)
    
    print(f"\n{'='*40}")
    print(f"Model: {model_name}")
    print(f"{'='*40}")
    print(f"AUC-ROC: {auc_roc:.4f}")
    print(f"AUC-PR:  {auc_pr:.4f}")
    print(f"Best F1 (threshold={best_threshold:.4f}): {best_f1:.4f}")
    print(f"\nClassification Report:")
    print(classification_report(y_test, y_pred_best, 
                                target_names=['Normal', 'Anomaly']))
    
    return {
        'model_name': model_name,
        'auc_roc': auc_roc,
        'auc_pr': auc_pr,
        'best_f1': best_f1,
        'errors': errors,
        'threshold': best_threshold
    }
```

#### d. Deployment Considerations

**Batch vs Real-time**:
- **Batch**: Run nightly on all wafers produced during the day. Reconstruct each wafer's measurements, flag those above threshold for next-day re-inspection.
- **Real-time**: If wafers are measured inline during production, the AE inference is fast enough for real-time scoring (< 1ms per sample on CPU).

**FastAPI Deployment**:

```python
# deployment/api.py
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
import torch
import numpy as np
import joblib
from pathlib import Path

app = FastAPI(title="Wafer Anomaly Detector", version="1.0")

# Load model and scaler at startup
MODEL_PATH = Path("models/ae_best.pt")
SCALER_PATH = Path("models/scaler.pkl")

class WaferMeasurement(BaseModel):
    measurements: list[float]  # 64-dim feature vector
    wafer_id: str

class AnomalyResult(BaseModel):
    wafer_id: str
    reconstruction_error: float
    is_anomaly: bool
    confidence: float

# Global model state
model = None
scaler = None
THRESHOLD = 0.85  # Set from validation; stored in config

@app.on_event("startup")
async def load_model():
    global model, scaler
    model = torch.load(MODEL_PATH, map_location='cpu')
    model.eval()
    scaler = joblib.load(SCALER_PATH)

@app.post("/predict", response_model=AnomalyResult)
async def predict_anomaly(wafer: WaferMeasurement):
    if len(wafer.measurements) != 64:
        raise HTTPException(400, f"Expected 64 features, got {len(wafer.measurements)}")
    
    # Preprocess
    x = np.array(wafer.measurements, dtype=np.float32).reshape(1, -1)
    x_scaled = scaler.transform(x)
    x_tensor = torch.FloatTensor(x_scaled)
    
    # Inference
    with torch.no_grad():
        x_hat = model(x_tensor)
        error = ((x_tensor - x_hat) ** 2).mean().item()
    
    # Score to probability (simple sigmoid scaling)
    confidence = 1 / (1 + np.exp(-(error - THRESHOLD) * 10))
    
    return AnomalyResult(
        wafer_id=wafer.wafer_id,
        reconstruction_error=round(error, 6),
        is_anomaly=error > THRESHOLD,
        confidence=round(confidence, 4)
    )

@app.get("/health")
async def health():
    return {"status": "ok", "model_loaded": model is not None}
```

```python
import torch
import torch.nn as nn
import torch.optim as optim
from torch.utils.data import DataLoader, TensorDataset
import numpy as np
import matplotlib.pyplot as plt
from typing import Tuple


# ═══════════════════════════════════════════════════════════
# STEP 1: DATA LOADING & PREPROCESSING
# ═══════════════════════════════════════════════════════════

def prepare_face_data(X_faces: np.ndarray, batch_size: int = 64) -> DataLoader:
    """
    Prepare face images for DCGAN training.
    
    Key: normalize to [-1, 1] to match Generator's Tanh output.
    """
    # Add channel dimension: (N, H, W) → (N, 1, H, W)
    X_tensor = torch.FloatTensor(X_faces).unsqueeze(1)
    
    # Normalize from [0, 1] to [-1, 1] (matches Tanh activation in Generator)
    X_tensor = X_tensor * 2 - 1
    
    dataset = TensorDataset(X_tensor)
    loader = DataLoader(dataset, batch_size=batch_size, shuffle=True, num_workers=0)
    
    print(f"Loaded {len(X_faces)} images, shape: {X_tensor.shape}")
    print(f"Pixel range: [{X_tensor.min():.2f}, {X_tensor.max():.2f}]")
    
    return loader


# ═══════════════════════════════════════════════════════════
# STEP 2: DCGAN ARCHITECTURE
# ═══════════════════════════════════════════════════════════

def weights_init(m):
    """
    Initialize weights according to DCGAN paper.
    Conv and BatchNorm layers get specific initialization.
    """
    classname = m.__class__.__name__
    if 'Conv' in classname:
        nn.init.normal_(m.weight.data, 0.0, 0.02)
    elif 'BatchNorm' in classname:
        nn.init.normal_(m.weight.data, 1.0, 0.02)
        nn.init.constant_(m.bias.data, 0)


class FaceGenerator(nn.Module):
    """
    DCGAN Generator for 64×64 grayscale faces.
    
    Architecture: z (100,) → (ngf*8, 4, 4) → ... → (1, 64, 64)
    Each ConvTranspose2d doubles the spatial resolution.
    """
    def __init__(self, latent_dim: int = 100, ngf: int = 64, n_channels: int = 1):
        super().__init__()
        self.latent_dim = latent_dim
        
        self.net = nn.Sequential(
            # Block 1: latent → (ngf*8, 4, 4)
            nn.ConvTranspose2d(latent_dim, ngf * 8, kernel_size=4, stride=1, padding=0, bias=False),
            nn.BatchNorm2d(ngf * 8),
            nn.ReLU(True),
            
            # Block 2: (ngf*8, 4, 4) → (ngf*4, 8, 8)
            nn.ConvTranspose2d(ngf * 8, ngf * 4, kernel_size=4, stride=2, padding=1, bias=False),
            nn.BatchNorm2d(ngf * 4),
            nn.ReLU(True),
            
            # Block 3: (ngf*4, 8, 8) → (ngf*2, 16, 16)
            nn.ConvTranspose2d(ngf * 4, ngf * 2, kernel_size=4, stride=2, padding=1, bias=False),
            nn.BatchNorm2d(ngf * 2),
            nn.ReLU(True),
            
            # Block 4: (ngf*2, 16, 16) → (ngf, 32, 32)
            nn.ConvTranspose2d(ngf * 2, ngf, kernel_size=4, stride=2, padding=1, bias=False),
            nn.BatchNorm2d(ngf),
            nn.ReLU(True),
            
            # Output: (ngf, 32, 32) → (n_channels, 64, 64)
            nn.ConvTranspose2d(ngf, n_channels, kernel_size=4, stride=2, padding=1, bias=False),
            nn.Tanh(),  # output in [-1, 1]
        )
        
        self.apply(weights_init)

    def forward(self, z: torch.Tensor) -> torch.Tensor:
        z = z.view(z.size(0), self.latent_dim, 1, 1)  # reshape to spatial
        return self.net(z)


class FaceDiscriminator(nn.Module):
    """
    DCGAN Discriminator for 64×64 grayscale faces.
    
    Architecture: (1, 64, 64) → ... → (ndf*8, 4, 4) → scalar
    Each Conv2d halves the spatial resolution.
    """
    def __init__(self, ndf: int = 64, n_channels: int = 1):
        super().__init__()
        
        self.net = nn.Sequential(
            # No BatchNorm on first layer (DCGAN paper recommendation)
            # (n_channels, 64, 64) → (ndf, 32, 32)
            nn.Conv2d(n_channels, ndf, kernel_size=4, stride=2, padding=1, bias=False),
            nn.LeakyReLU(0.2, inplace=True),
            
            # (ndf, 32, 32) → (ndf*2, 16, 16)
            nn.Conv2d(ndf, ndf * 2, kernel_size=4, stride=2, padding=1, bias=False),
            nn.BatchNorm2d(ndf * 2),
            nn.LeakyReLU(0.2, inplace=True),
            
            # (ndf*2, 16, 16) → (ndf*4, 8, 8)
            nn.Conv2d(ndf * 2, ndf * 4, kernel_size=4, stride=2, padding=1, bias=False),
            nn.BatchNorm2d(ndf * 4),
            nn.LeakyReLU(0.2, inplace=True),
            
            # (ndf*4, 8, 8) → (ndf*8, 4, 4)
            nn.Conv2d(ndf * 4, ndf * 8, kernel_size=4, stride=2, padding=1, bias=False),
            nn.BatchNorm2d(ndf * 8),
            nn.LeakyReLU(0.2, inplace=True),
            
            # (ndf*8, 4, 4) → (1, 1, 1)
            nn.Conv2d(ndf * 8, 1, kernel_size=4, stride=1, padding=0, bias=False),
            nn.Sigmoid(),
        )
        
        self.apply(weights_init)

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.net(x).view(-1, 1)


# ═══════════════════════════════════════════════════════════
# STEP 3: DCGAN TRAINING WITH MONITORING
# ═══════════════════════════════════════════════════════════

class GANTrainer:
    """
    Encapsulates DCGAN training with monitoring and checkpointing.
    """
    def __init__(
        self,
        generator: nn.Module,
        discriminator: nn.Module,
        latent_dim: int = 100,
        lr: float = 2e-4,
        betas: Tuple = (0.5, 0.999),
        device: str = 'cpu'
    ):
        self.G = generator.to(device)
        self.D = discriminator.to(device)
        self.latent_dim = latent_dim
        self.device = device
        
        # DCGAN uses Adam with lower beta1 than default
        self.opt_G = optim.Adam(self.G.parameters(), lr=lr, betas=betas)
        self.opt_D = optim.Adam(self.D.parameters(), lr=lr, betas=betas)
        
        self.criterion = nn.BCELoss()
        
        # Fixed noise for tracking G's progress visually
        self.fixed_noise = torch.randn(16, latent_dim).to(device)
        
        self.history = {
            'd_loss': [], 'g_loss': [],
            'd_real_acc': [], 'd_fake_acc': []
        }

    def train_epoch(self, dataloader: DataLoader) -> dict:
        """Train one epoch, return per-batch metrics."""
        self.G.train()
        self.D.train()
        
        d_losses, g_losses = [], []
        d_real_accs, d_fake_accs = [], []
        
        for (real_images,) in dataloader:
            batch_size = real_images.size(0)
            real_images = real_images.to(self.device)
            
            real_labels = torch.ones(batch_size, 1).to(self.device) * 0.9  # label smoothing
            fake_labels = torch.zeros(batch_size, 1).to(self.device)
            
            # ────────────────────────────────────────
            # Train Discriminator
            # ────────────────────────────────────────
            self.opt_D.zero_grad()
            
            # Real images
            d_real = self.D(real_images)
            d_real_loss = self.criterion(d_real, real_labels)
            
            # Fake images (detach from G graph)
            z = torch.randn(batch_size, self.latent_dim).to(self.device)
            fake_images = self.G(z).detach()
            d_fake = self.D(fake_images)
            d_fake_loss = self.criterion(d_fake, fake_labels)
            
            d_loss = d_real_loss + d_fake_loss
            d_loss.backward()
            self.opt_D.step()
            
            # ────────────────────────────────────────
            # Train Generator
            # ────────────────────────────────────────
            self.opt_G.zero_grad()
            
            z = torch.randn(batch_size, self.latent_dim).to(self.device)
            fake_images = self.G(z)
            g_pred = self.D(fake_images)
            
            # Non-saturating loss: G maximizes log(D(G(z)))
            g_loss = self.criterion(g_pred, torch.ones(batch_size, 1).to(self.device))
            g_loss.backward()
            self.opt_G.step()
            
            # Track metrics
            d_losses.append(d_loss.item())
            g_losses.append(g_loss.item())
            d_real_accs.append((d_real > 0.5).float().mean().item())
            d_fake_accs.append((d_fake < 0.5).float().mean().item())
        
        return {
            'd_loss': np.mean(d_losses),
            'g_loss': np.mean(g_losses),
            'd_real_acc': np.mean(d_real_accs),
            'd_fake_acc': np.mean(d_fake_accs),
        }

    def train(self, dataloader: DataLoader, epochs: int = 100, save_every: int = 10):
        """Full training loop with monitoring."""
        print(f"Starting DCGAN training for {epochs} epochs")
        print(f"Generator params: {sum(p.numel() for p in self.G.parameters()):,}")
        print(f"Discriminator params: {sum(p.numel() for p in self.D.parameters()):,}")
        
        for epoch in range(epochs):
            metrics = self.train_epoch(dataloader)
            
            for k, v in metrics.items():
                self.history[k].append(v)
            
            if (epoch + 1) % 10 == 0:
                print(
                    f"Epoch [{epoch+1}/{epochs}] | "
                    f"D_loss: {metrics['d_loss']:.4f} | "
                    f"G_loss: {metrics['g_loss']:.4f} | "
                    f"D(real): {metrics['d_real_acc']:.3f} | "
                    f"D(fake): {metrics['d_fake_acc']:.3f}"
                )
                
                # Health check
                health = self._check_health(metrics)
                if health != 'OK':
                    print(f"  ⚠️  {health}")
        
        print("\nTraining complete!")
        return self.history

    def _check_health(self, metrics: dict) -> str:
        """Heuristic health check for GAN training."""
        if metrics['d_real_acc'] > 0.99 and metrics['d_fake_acc'] > 0.99:
            return "Discriminator too dominant — Generator may not be learning"
        if metrics['g_loss'] > 5.0:
            return "Generator loss too high — consider reducing D update frequency"
        if metrics['d_loss'] < 0.1:
            return "Discriminator too strong — increase G complexity or reduce D lr"
        return "OK"

    def generate_samples(self, n: int = 16) -> torch.Tensor:
        """Generate n samples for visualization."""
        self.G.eval()
        with torch.no_grad():
            z = torch.randn(n, self.latent_dim).to(self.device)
            samples = self.G(z)
        self.G.train()
        return samples.cpu()


# ═══════════════════════════════════════════════════════════
# STEP 4: EVALUATION METRICS FOR GENERATIVE MODELS
# ═══════════════════════════════════════════════════════════

def compute_inception_score_proxy(
    generated_samples: torch.Tensor,
    real_samples: torch.Tensor
) -> dict:
    """
    Proxy metrics for GAN evaluation (without full Inception network).
    
    Real evaluations use:
    - FID (Fréchet Inception Distance): gold standard
    - IS (Inception Score): diversity + quality
    - SSIM: structural similarity (reconstruction tasks)
    
    Here we use simple proxies for demonstration.
    """
    gen_np = generated_samples.numpy().reshape(len(generated_samples), -1)
    real_np = real_samples.numpy().reshape(len(real_samples), -1)
    
    # 1. Pixel statistics divergence
    gen_mean, gen_std = gen_np.mean(), gen_np.std()
    real_mean, real_std = real_np.mean(), real_np.std()
    
    stat_divergence = abs(gen_mean - real_mean) + abs(gen_std - real_std)
    
    # 2. Sample diversity (std across generated samples)
    diversity = gen_np.std(axis=0).mean()
    
    # 3. Coverage: do generated samples cover the real distribution?
    from sklearn.neighbors import NearestNeighbors
    nn = NearestNeighbors(n_neighbors=1)
    nn.fit(real_np[:200])  # fit on subset of real
    distances, _ = nn.kneighbors(gen_np[:200])
    coverage_score = 1 / (1 + distances.mean())  # higher = better coverage
    
    metrics = {
        'pixel_stat_divergence': round(stat_divergence, 4),
        'sample_diversity': round(diversity, 4),
        'coverage_score': round(coverage_score, 4),
    }
    
    print("\nGenerative Model Evaluation Metrics:")
    print(f"  Pixel stat divergence (lower=better): {metrics['pixel_stat_divergence']}")
    print(f"  Sample diversity (higher=better): {metrics['sample_diversity']}")
    print(f"  Coverage score (higher=better): {metrics['coverage_score']}")
    
    return metrics


# ═══════════════════════════════════════════════════════════
# STEP 5: RUN FULL PIPELINE
# ═══════════════════════════════════════════════════════════

def run_dcgan_pipeline():
    device = 'cuda' if torch.cuda.is_available() else 'cpu'
    LATENT_DIM = 100
    BATCH_SIZE = 64
    EPOCHS = 50   # Use 200+ for real results
    
    print("=== DCGAN Face Generation Pipeline ===\n")
    
    # Data
    X_faces = generate_synthetic_face_dataset(n_samples=5000, img_size=64)
```

## 7. Common Mistakes & Pitfalls

### Autoencoder Pitfalls

**1. Making the latent space too large**
The autoencoder will learn to copy the input (identity function) if the bottleneck is not constraining enough. Always start smaller than you think you need and increase if reconstruction is too poor.

**2. Using MSE for image reconstruction when you care about sharpness**
MSE averages predictions, producing blurry reconstructions. Consider perceptual loss (VGG features) or SSIM loss for sharper results.

**3. Assuming a vanilla AE's latent space is suitable for generation**
It's not. The latent space of a vanilla AE is fragmented. Use a VAE if you need to generate new samples by sampling from the latent space.

**4. Forgetting to use the scaler at inference time**
The model was trained on scaled data. If you feed raw data at inference, reconstruction errors will be meaningless. Always store and apply the preprocessing pipeline.

**5. Setting anomaly thresholds without understanding the business context**
A threshold of "top 5% of reconstruction errors = anomaly" is arbitrary. Work with domain experts to understand the acceptable false positive rate in the business context.

### VAE Pitfalls

**6. Posterior collapse**
The VAE learns to ignore the latent code (KL term goes to 0, reconstruction uses no z). Solution: KL annealing — start with β=0 and slowly increase it during training.

**7. Ignoring the KL term interpretation**
A KL of 0 means the encoder is outputting the prior N(0, I) for all inputs — the encoder is useless. Monitor KL during training; it should be positive and non-trivially large.

**8. Expecting VAE to match GAN quality**
VAEs are trained with pixel-wise reconstruction loss, which tends to produce blurry images. For photorealism, GANs are the right tool. Hybrid models (VAE-GAN) exist but are complex.

### GAN Pitfalls

**9. Training D to convergence before training G**
If D is perfect, G gets zero gradient. Train D for 1-5 steps, then train G for 1 step (not the other way around, and not until convergence).

**10. Using the same learning rate for G and D**
G and D often benefit from different learning rates. A common setting: lr_D = 4e-4, lr_G = 1e-4.

**11. Not using label smoothing**
If real labels are exactly 1.0, D becomes overconfident. Use 0.9 for real labels. This is a single-line change with significant stability improvement.

**12. Ignoring FID when evaluating GANs**
Visual inspection of generated samples is insufficient. FID (Fréchet Inception Distance) is the standard metric — it measures both quality and diversity by comparing distributions of Inception features.

**13. Mode collapse misdiagnosis**
If all generated samples look similar *within a batch*, that's mode collapse. If they look similar *across batches*, the random seed might just be similar. Always check diversity across many different noise samples.

**14. Not monitoring D(real) and D(fake) separately**
The combined discriminator loss can look fine even during training failure. Monitor D(real) (should stay ~0.7-0.9 after label smoothing) and D(fake) (should start high and decrease) separately.

## 8. Interview Preparation

### Conceptual Questions

**Q1: What is the fundamental difference between a VAE and a regular autoencoder?**

A regular autoencoder maps each input to a fixed point in latent space. There's no structure imposed on what that space looks like — it can be fragmented and discontinuous. A VAE maps each input to a *distribution* (Gaussian with mean μ and variance σ²) and samples from it during training. The KL divergence term forces all these distributions toward the prior N(0, I), making the latent space smooth, continuous, and fully covered. This is what makes the VAE generative — any point sampled from N(0, I) decodes to a valid output.

**Q2: Explain the reparameterization trick and why it's necessary.**

To train the VAE, we need gradients to flow through the sampling step z ~ N(μ, σ²). But sampling is not a differentiable operation — you can't take a derivative with respect to the parameters of a random variable you're sampling from. The reparameterization trick rewrites z = μ + σ·ε, where ε ~ N(0, I). Now ε is just fixed random noise (no parameters), and the operation z = μ + σ·ε is fully differentiable with respect to μ and σ. Gradients can flow through.

**Q3: What is mode collapse in GANs, and what causes it?**

Mode collapse occurs when the Generator produces only a limited subset of the real data distribution. For example, on MNIST, it might produce perfect-looking 7s and nothing else. It's caused by the Generator finding a "local optimum" in the adversarial game — it discovers that generating one class of outputs consistently fools the Discriminator, and has no gradient signal pushing it to be more diverse. The standard GAN objective doesn't explicitly penalize lack of diversity.

**Q4: Why does the Discriminator use LeakyReLU instead of ReLU?**

Standard ReLU kills negative activations (outputs zero), which can lead to dead neurons in the Discriminator and reduce gradient flow during backpropagation. The Discriminator needs to pass gradients through to update the Generator. LeakyReLU (with a small negative slope, typically 0.2) allows a small gradient even for negative inputs, maintaining gradient flow and helping the Generator learn even when the Discriminator classifies inputs as very negative.

**Q5: What does KL annealing solve in VAE training?**

Posterior collapse: the VAE can shortcut the learning objective by ignoring the latent code entirely. If the decoder is powerful enough, it can reconstruct inputs well without using z — making the encoder produce the prior N(0, I) for all inputs (KL → 0). KL annealing prevents this by starting training with β = 0 (pure reconstruction objective) and slowly increasing β. This forces the encoder to first learn useful reconstruction features, and only gradually introduces the pressure toward the prior, preventing early collapse.

### Practical Scenario Questions

**Q6: You train a GAN for 100 epochs and the Generator loss keeps increasing while Discriminator loss drops to near zero. What's happening and what do you do?**

The Discriminator is too strong — it's perfectly classifying real from fake. The Generator receives near-zero gradient because log(1 - D(G(z))) saturates. Actions:
- Reduce Discriminator learning rate relative to Generator
- Add label smoothing to real labels (0.9 instead of 1.0)
- Add dropout to the Discriminator
- Train Generator for more steps per Discriminator step
- Switch to non-saturating Generator loss: -log(D(G(z)))

**Q7: Your autoencoder-based anomaly detector is flagging 40% of normal samples as anomalies. How do you debug this?**

This means the reconstruction threshold is set too low. Systematic debugging:
1. Plot the distribution of reconstruction errors for validation normal samples — is the threshold within or above this distribution?
2. Check if the autoencoder is actually reconstructing normal samples well (visualize reconstructions)
3. Check if preprocessing at inference matches training (same scaler, same feature engineering)
4. Check if there's data drift between training normal data and current normal data
5. Recalibrate the threshold using a held-out normal sample set, targeting your desired false positive rate

**Q8: When would you choose a VAE over a GAN for a generation task?**

Choose VAE when:
- You need to also reconstruct given inputs (VAE can do both; GAN cannot reconstruct)
- You need a structured latent space for interpolation or semantic arithmetic
- Training stability is critical (VAEs are much more stable)
- You need uncertainty estimates (VAE gives a distribution over z)
- The dataset is relatively small
Choose GAN when: photorealistic quality is the primary goal and you have enough data and compute for stable training.

**Q9: How would you deploy a GAN-based image generation service at scale?**

- Export Generator to ONNX for framework-agnostic serving
- Host on GPU instances; batch requests when possible
- Cache common outputs (seeded generation) for instant retrieval
- API rate limiting per user
- Content moderation pipeline on generated outputs before serving
- Monitoring: track latent space diversity, image quality metrics (FID) on sampled outputs over time
- Canary deployment: A/B test new model versions

**Q10: Explain the CycleGAN cycle consistency loss in your own words.**

Without paired training data, you can't directly train a translator. Instead, CycleGAN trains two generators: G (domain A → B) and F (domain B → A). The cycle consistency loss says: if you translate image x from A to B, then translate it back from B to A, you should get approximately the original x back. This constrains the translation to be meaningful — the generator can't arbitrarily hallucinate content, because that content would need to be "undone" by the reverse generator, and the round trip must be accurate. The loss term is simply L1 distance between x and F(G(x)).
