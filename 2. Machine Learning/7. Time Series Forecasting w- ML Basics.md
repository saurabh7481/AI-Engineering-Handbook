# 1. Time Series Analysis

## 1.1 What is Time-Series Data

### a. Intuition

Imagine you're tracking your daily step count. Each day produces one number. The *order* of those numbers matters — yesterday's count tells you something about today's. That's the fundamental difference between time series data and regular tabular data: **observations are ordered, and that order carries information**.

In a standard ML dataset, you can shuffle rows and nothing is lost. In a time series, shuffling destroys the entire signal.

### b. Mathematical Insight

A time series is formally defined as a sequence of observations indexed by time:

```
Y = {y_t : t = 1, 2, ..., T}
```

Where:
- `y_t` is the observed value at time `t`
- `T` is the total number of observations
- The index `t` has a fixed, meaningful ordering

### c. How It Works

1. Data is collected at each time step (second, day, month, etc.)
2. Each observation is associated with a timestamp
3. The temporal structure creates dependencies: `y_t` is often correlated with `y_{t-1}`, `y_{t-2}`, etc.
4. The goal is to model or exploit these dependencies

### d. Visual Representation

```
Time:  t1   t2   t3   t4   t5   t6   t7
        |    |    |    |    |    |    |
Value: 10   12    9   15   13   18   16
        \_____________________________/
              Time-ordered sequence
```

### e. Python Implementation

```python
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

# Create a simple time series
dates = pd.date_range(start='2023-01-01', periods=365, freq='D')
values = np.cumsum(np.random.randn(365)) + 100  # Random walk starting at 100

ts = pd.Series(values, index=dates, name='value')

# Basic inspection
print(ts.head())
print(f"\nFrequency: {ts.index.freq}")
print(f"Date range: {ts.index[0]} to {ts.index[-1]}")
print(f"Total observations: {len(ts)}")

# Plot
ts.plot(figsize=(12, 4), title='Daily Time Series', ylabel='Value')
plt.tight_layout()
plt.show()
```

### f. When to Use / Avoid

**Use time series methods when:**
- Data has a natural temporal ordering
- Past values help predict future values
- Seasonality or trends exist

**Avoid treating as time series when:**
- Data is cross-sectional (e.g., one survey per person, no repeat measurements)
- Time stamps are arbitrary labels, not meaningful ordering

## 1.2 Time Index & Frequency

### a. Intuition

Think of the time index as the ruler you use to measure your data. A ruler calibrated in centimeters gives different readings than one in inches — even if you're measuring the same object. Frequency defines the "unit" of your time ruler: is it ticking every second, every day, or every quarter?

### b. Mathematical Insight

Frequency `f` determines the number of observations per unit time period. The period `P = 1/f` is the time between observations.

For seasonal analysis, the seasonal period `m` is crucial:
- Daily data with weekly seasonality: `m = 7`
- Monthly data with yearly seasonality: `m = 12`
- Hourly data with daily seasonality: `m = 24`

### c. How It Works

1. Choose the observation granularity (seconds, days, months)
2. Assign a timestamp to each observation
3. Verify regularity: are all gaps equal?
4. Set the Pandas frequency string (e.g., `'D'`, `'M'`, `'H'`)

### d. Pandas Frequency Reference

| Alias | Meaning         | Example Use Case         |
|-------|-----------------|--------------------------|
| `S`   | Second          | IoT sensor data          |
| `T`   | Minute          | Trading tick data        |
| `H`   | Hour            | Energy consumption       |
| `D`   | Calendar day    | Website traffic          |
| `B`   | Business day    | Stock prices             |
| `W`   | Week            | Weekly reports           |
| `M`   | Month end       | Sales summaries          |
| `Q`   | Quarter end     | Financial reporting      |
| `A`   | Year end        | Annual budgets           |

### e. Python Implementation

```python
import pandas as pd

# Setting and verifying frequency
ts = pd.Series(
    [10, 12, 9, 15, 13, 18],
    index=pd.date_range('2023-01-01', periods=6, freq='D')
)

print(f"Inferred frequency: {pd.infer_freq(ts.index)}")

# Resampling to different frequencies
ts_monthly = ts.resample('M').mean()   # Aggregate to monthly
ts_hourly  = ts.resample('H').interpolate()  # Upsample to hourly

# Checking for gaps
expected_dates = pd.date_range(ts.index[0], ts.index[-1], freq='D')
missing = expected_dates.difference(ts.index)
print(f"Missing dates: {list(missing)}")
```

### f. When to Use / Avoid

Always set the frequency explicitly on your DatetimeIndex. Pandas functions like `shift()`, `resample()`, and `rolling()` behave differently (or fail) without a set frequency. Irregular time series require special handling — see Section 1.10.

## 1.3 Components of a Time Series

### a. Intuition

Every time series is like a layered cake. Underneath the noisy surface you observe, there are distinct layers:
- **Trend**: Is the series climbing or falling over the long run?
- **Seasonality**: Does it repeat at regular, calendar-driven intervals?
- **Cyclicity**: Are there longer, irregular boom-bust patterns?
- **Noise**: What's left after accounting for everything else?

### b. Mathematical Insight

In the additive model:

```
y_t = T_t + S_t + C_t + R_t
```

Where:
- `T_t` = Trend component (long-run direction)
- `S_t` = Seasonal component (calendar-driven repeating patterns)
- `C_t` = Cyclical component (multi-year irregular waves)
- `R_t` = Remainder/Residual (random noise)

### Trend

**Intuition:** Trend is the persistent, long-run direction of the series. Think of global temperature rising over decades — that upward drift is the trend, separate from seasonal summer/winter swings.

**Types of trend:**
- Linear trend: constant rate of change
- Exponential trend: accelerating rate (common in tech adoption)
- Stochastic trend: random walk (stock prices)

### Seasonality

**Intuition:** Seasonality is the predictable, calendar-linked repetition. Ice cream sales spike every summer. Tax filings surge in April. These patterns are regular and tied to the calendar.

**Key property:** Fixed, known period (e.g., 12 months, 7 days, 24 hours).

### Cyclicity

**Intuition:** Cyclicity is like seasonality's unpredictable cousin. Business cycles last 2–10 years and don't follow a fixed calendar schedule. They're driven by economic forces, not the calendar.

**Key difference from seasonality:** Cycles have *variable* length and magnitude.

### Noise

**Intuition:** Whatever remains after removing trend, seasonality, and cycles. It's the inherently unpredictable part — measurement error, random shocks, one-off events.

### c. How It Works (Decomposition Preview)

```
Observed = Trend + Seasonality + Cyclicity + Noise

Step 1: Estimate trend (e.g., moving average)
Step 2: Remove trend → detrended series
Step 3: Estimate seasonal pattern from detrended series
Step 4: Remove seasonality → residuals
Step 5: Residuals contain cyclicity + noise
```

### d. Visual Representation

```
Observed:   /\/\/\/\/\/\/\/\/\___/\/\/\/\/\/\/\/\/\/\
            ↑ noisy seasonal waves with upward trend

Trend:      ___________________________
                                       ↑ slow rise

Seasonal:   /\/\/\/\/\/\/\/\/\/\/\/\/\/\/\/
            ↑ regular peaks and troughs

Residual:   ~~~~~wv~~w~~~v~w~~~
            ↑ random noise
```

### e. Python Implementation

```python
from statsmodels.tsa.seasonal import seasonal_decompose
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

# Generate synthetic series with known components
np.random.seed(42)
t = np.arange(120)
trend     = 0.5 * t                          # Linear upward trend
seasonal  = 10 * np.sin(2 * np.pi * t / 12) # Monthly seasonality
noise     = np.random.randn(120) * 2

y = trend + seasonal + noise
dates = pd.date_range('2014-01-01', periods=120, freq='M')
ts = pd.Series(y, index=dates)

# Decompose
result = seasonal_decompose(ts, model='additive', period=12)

fig, axes = plt.subplots(4, 1, figsize=(12, 10))
result.observed.plot(ax=axes[0],  title='Observed')
result.trend.plot(ax=axes[1],     title='Trend')
result.seasonal.plot(ax=axes[2],  title='Seasonal')
result.resid.plot(ax=axes[3],     title='Residual')
plt.tight_layout()
plt.show()
```

## 1.4 Additive vs Multiplicative Series

### a. Intuition

Think of a bakery's weekly sales. If the seasonal swings are always ±100 units regardless of how big the business gets, that's **additive**. If the swings grow proportionally as the business scales (±10% of current volume), that's **multiplicative**.

The test: plot the series. If seasonal amplitudes grow with the level, use multiplicative. If they stay constant, use additive.

### b. Mathematical Insight

**Additive model:**
```
y_t = T_t + S_t + R_t
```
Seasonal effect has constant magnitude regardless of trend level.

**Multiplicative model:**
```
y_t = T_t × S_t × R_t
```
Seasonal effect scales with the trend. Taking log converts it to additive:
```
log(y_t) = log(T_t) + log(S_t) + log(R_t)
```

### c. How It Works

1. Plot the series and observe seasonal amplitudes
2. If amplitudes are constant → additive
3. If amplitudes grow with level → multiplicative
4. For multiplicative: apply log transform, use additive methods, then exponentiate forecasts

### e. Python Implementation

```python
from statsmodels.tsa.seasonal import seasonal_decompose
import pandas as pd
import numpy as np

np.random.seed(42)
t = np.arange(120)

# Multiplicative: seasonal amplitude grows with trend
trend_mult    = np.exp(0.03 * t)
seasonal_mult = 1 + 0.3 * np.sin(2 * np.pi * t / 12)
noise_mult    = np.random.lognormal(0, 0.05, 120)
y_mult        = trend_mult * seasonal_mult * noise_mult

dates  = pd.date_range('2014-01-01', periods=120, freq='M')
ts_m   = pd.Series(y_mult, index=dates)

# Use multiplicative decomposition
result_mult = seasonal_decompose(ts_m, model='multiplicative', period=12)

# Alternatively: log-transform and use additive
ts_log       = np.log(ts_m)
result_log   = seasonal_decompose(ts_log, model='additive', period=12)
```

### f. When to Use / Avoid

| Condition | Model |
|-----------|-------|
| Seasonal swings are constant | Additive |
| Seasonal swings grow with level | Multiplicative |
| Series contains zeros or negatives | Additive only (log undefined) |
| Values span multiple orders of magnitude | Multiplicative / log-additive |

## 1.5 Stationarity & Non-Stationarity

### a. Intuition

A stationary series is one that behaves the same throughout its history — like a balanced scale that always returns to center. Its mean, variance, and autocorrelation don't change over time.

A non-stationary series wanders. Stock prices are non-stationary: they trend, drift, and their variance can explode. Most classical forecasting models (ARIMA) **require stationarity** — so we transform the series until it is.

### b. Mathematical Insight

A series `y_t` is **weakly stationary** (covariance stationary) if:

1. Constant mean: `E[y_t] = μ` for all `t`
2. Constant variance: `Var(y_t) = σ²` for all `t`
3. Autocovariance depends only on lag: `Cov(y_t, y_{t-k}) = γ(k)` — not on `t`

**Unit Root (non-stationarity test):** A random walk `y_t = y_{t-1} + ε_t` has a unit root and is non-stationary.

**ADF Test Hypothesis:**
```
H₀: Series has a unit root (non-stationary)
H₁: Series is stationary

Reject H₀ if p-value < 0.05 → stationary
```

### c. How It Works

1. Plot the series — does mean or variance visually shift?
2. Plot rolling mean and rolling std — are they flat?
3. Run Augmented Dickey-Fuller (ADF) or KPSS test
4. If non-stationary: difference the series, apply log transform, or detrend
5. Re-test until stationarity is achieved

### d. Visual Representation

```
Non-Stationary (trending mean):       Stationary (flat mean):
                                  
  ↑                               ↑ - - - - - - - - - mean
  |        /                      |  ~^~v~~^v~^~v~^^~
  |      /                        |
  |    /                          |
  |  /                            |
  |/                              +-------------------→ t
  +-------------------→ t
```

### e. Python Implementation

```python
import pandas as pd
import numpy as np
from statsmodels.tsa.stattools import adfuller, kpss
import matplotlib.pyplot as plt

def test_stationarity(series: pd.Series, window: int = 12) -> dict:
    """
    Comprehensive stationarity test with rolling statistics and ADF/KPSS tests.
    """
    # Rolling statistics
    roll_mean = series.rolling(window=window).mean()
    roll_std  = series.rolling(window=window).std()
    
    # ADF Test
    adf_result = adfuller(series.dropna(), autolag='AIC')
    adf_pvalue = adf_result[1]
    
    # KPSS Test (opposite null: H0 = stationary)
    kpss_result = kpss(series.dropna(), regression='c', nlags='auto')
    kpss_pvalue = kpss_result[1]
    
    print("=== Stationarity Tests ===")
    print(f"ADF  p-value: {adf_pvalue:.4f}  → {'Stationary ✓' if adf_pvalue < 0.05 else 'Non-Stationary ✗'}")
    print(f"KPSS p-value: {kpss_pvalue:.4f}  → {'Stationary ✓' if kpss_pvalue > 0.05 else 'Non-Stationary ✗'}")
    
    # Plot
    fig, axes = plt.subplots(2, 1, figsize=(12, 6))
    series.plot(ax=axes[0], label='Original', alpha=0.7)
    roll_mean.plot(ax=axes[0], label=f'Rolling Mean ({window})', color='red')
    roll_std.plot(ax=axes[0], label=f'Rolling Std ({window})', color='green')
    axes[0].legend(); axes[0].set_title('Rolling Statistics')
    
    # First difference (common fix)
    series.diff().dropna().plot(ax=axes[1], color='purple')
    axes[1].set_title('First Difference')
    plt.tight_layout()
    plt.show()
    
    return {'adf_pvalue': adf_pvalue, 'kpss_pvalue': kpss_pvalue}

# Example
np.random.seed(42)
non_stationary = pd.Series(np.cumsum(np.random.randn(200)))  # Random walk
test_stationarity(non_stationary)
```

### f. When to Use / Avoid

- Always test before fitting ARIMA/SARIMA — they assume stationarity
- Use ADF and KPSS together (they have opposite null hypotheses, using both reduces error)
- Be cautious of over-differencing: check if `d=1` is enough before trying `d=2`

### g. Key Hyperparameters

- `d` in ARIMA: number of differences needed to achieve stationarity
- ADF `autolag='AIC'`: automatically selects lag order for the test

## 1.6 Autocorrelation & Partial Autocorrelation

### a. Intuition

**Autocorrelation (ACF):** How much does today's value look like yesterday's? Last week's? If you know it rained yesterday, does that tell you anything about today's rainfall? ACF measures this correlation across all lags simultaneously.

**Partial Autocorrelation (PACF):** Like ACF, but removes the "middleman" correlations. The correlation between today and two days ago, *after* controlling for yesterday's effect. This is how you isolate the direct influence of each lag.

Together, ACF and PACF are the X-ray of a time series — they tell you exactly what structure the data has.

### b. Mathematical Insight

**ACF at lag k:**
```
ρ(k) = Cov(y_t, y_{t-k}) / Var(y_t)
     = γ(k) / γ(0)
```

**PACF at lag k:** The partial correlation between `y_t` and `y_{t-k}` after removing linear effects of `y_{t-1}, ..., y_{t-k+1}`.

**Critical values for significance:**
```
±1.96 / √T   (approximately)
```
Bars exceeding this threshold are statistically significant.

### c. How It Works

1. Compute correlation between `y_t` and `y_{t-k}` for each lag `k`
2. Plot as vertical bars against lag number
3. Use confidence bands (dashed lines) to identify significant lags
4. Interpret pattern to determine model order

**Pattern Recognition Guide:**

| Pattern | ACF | PACF | Implication |
|---------|-----|------|-------------|
| AR(p) | Decays gradually (tails off) | Cuts off after lag p | Use AR(p) |
| MA(q) | Cuts off after lag q | Decays gradually (tails off) | Use MA(q) |
| ARMA(p,q) | Tails off | Tails off | Use ARMA |
| White noise | No significant lags | No significant lags | No structure |
| Seasonal | Spikes at seasonal lags (12, 24...) | Spikes at seasonal lags | Add seasonal terms |

### d. Visual Representation

```
ACF for AR(2) process:
Lag:  0    1    2    3    4    5    6    7    8
      |    |    |         |         |
      █    ██   █   .     .    .    .    .    .
           ↑ tails off gradually (typical AR signature)

PACF for AR(2) process:
Lag:  0    1    2    3    4    5    6    7    8
      |    |    |
      █    ██   █    .    .    .    .    .    .
                ↑ cuts off sharply at lag 2 (p=2)
```

### e. Python Implementation

```python
import pandas as pd
import numpy as np
from statsmodels.graphics.tsaplots import plot_acf, plot_pacf
from statsmodels.tsa.stattools import acf, pacf
import matplotlib.pyplot as plt

def plot_acf_pacf(series: pd.Series, lags: int = 40, title: str = 'ACF & PACF Analysis'):
    """
    Plot ACF and PACF side by side for model order identification.
    """
    fig, axes = plt.subplots(1, 2, figsize=(14, 5))
    
    plot_acf(series.dropna(), lags=lags, ax=axes[0], alpha=0.05)
    axes[0].set_title(f'{title} - ACF')
    axes[0].set_xlabel('Lag')
    
    plot_pacf(series.dropna(), lags=lags, ax=axes[1], alpha=0.05, method='ywmle')
    axes[1].set_title(f'{title} - PACF')
    axes[1].set_xlabel('Lag')
    
    plt.tight_layout()
    plt.show()

# Demonstrate with AR(2) process
np.random.seed(42)
from statsmodels.tsa.arima_process import ArmaProcess

ar_params = np.array([1, -0.6, -0.3])  # AR(2): y_t = 0.6*y_{t-1} + 0.3*y_{t-2} + e
ma_params = np.array([1])
ar2_process = ArmaProcess(ar_params, ma_params)
ar2_sample  = pd.Series(ar2_process.generate_sample(nsample=300))

plot_acf_pacf(ar2_sample, lags=20, title='AR(2) Process')

# Print significant lags
acf_values  = acf(ar2_sample, nlags=20)
pacf_values = pacf(ar2_sample, nlags=20, method='ywmle')
conf_int    = 1.96 / np.sqrt(len(ar2_sample))

print("Significant ACF lags:", [i for i, v in enumerate(acf_values) if abs(v) > conf_int and i > 0])
print("Significant PACF lags:", [i for i, v in enumerate(pacf_values) if abs(v) > conf_int and i > 0])
```

### f. When to Use / Avoid

- **Always** plot ACF/PACF before fitting any ARIMA model
- Use on the **differenced** series (after achieving stationarity)
- For seasonal data: also inspect lags at multiples of the seasonal period

## 1.7 Lag Plots & Rolling Statistics

### a. Intuition

A **lag plot** is a scatter plot of `y_t` vs `y_{t-k}`. If there's a clear linear pattern, the series is autocorrelated at lag `k`. It's the simplest visual test for dependence.

**Rolling statistics** track how the mean and variance evolve over time using a sliding window. They're the diagnostic tool for non-stationarity — if the rolling mean drifts or the rolling std changes, the series isn't stationary.

### b. Mathematical Insight

**Rolling mean at time t with window w:**
```
μ_t = (1/w) × Σ y_{t-i}  for i = 0 to w-1
```

**Rolling standard deviation:**
```
σ_t = sqrt[(1/w) × Σ (y_{t-i} - μ_t)²]
```

### c. How It Works

1. For lag plots: plot `(y_{t}, y_{t-k})` as scatter; linear pattern = autocorrelation
2. For rolling stats: slide a window of size `w` and compute mean/std at each position
3. Compare rolling mean to overall mean — divergence = non-stationarity

### e. Python Implementation

```python
import pandas as pd
import numpy as np
from pandas.plotting import lag_plot
import matplotlib.pyplot as plt

def analyze_lags_and_rolling(series: pd.Series, lags: list = [1, 7, 30], window: int = 30):
    """
    Combined lag plot and rolling statistics analysis.
    """
    n_lags = len(lags)
    fig, axes = plt.subplots(2, max(n_lags, 2), figsize=(14, 8))
    
    # Lag plots
    for i, lag in enumerate(lags):
        lag_plot(series, lag=lag, ax=axes[0, i])
        axes[0, i].set_title(f'Lag Plot (k={lag})')
        # Compute and annotate correlation
        corr = series.autocorr(lag=lag)
        axes[0, i].set_xlabel(f'y(t-{lag}) | r={corr:.3f}')
    
    # Rolling statistics
    roll_mean = series.rolling(window=window).mean()
    roll_std  = series.rolling(window=window).std()
    
    series.plot(ax=axes[1, 0], alpha=0.5, label='Observed', color='steelblue')
    roll_mean.plot(ax=axes[1, 0], label=f'Rolling Mean ({window})', color='red', lw=2)
    axes[1, 0].legend(); axes[1, 0].set_title('Rolling Mean')
    
    roll_std.plot(ax=axes[1, 1], color='green', label=f'Rolling Std ({window})')
    axes[1, 1].legend(); axes[1, 1].set_title('Rolling Std (check for variance changes)')
    
    plt.tight_layout()
    plt.show()

# Test
np.random.seed(42)
ts = pd.Series(np.cumsum(np.random.randn(365)) + 50)
analyze_lags_and_rolling(ts, lags=[1, 7, 30])
```

### f. When to Use / Avoid

- Lag plots: quick visual check for serial dependence before modeling
- Rolling statistics: always use during EDA for any time series
- Window size: choose based on domain knowledge (7 for weekly data, 12 for monthly seasonality)

## 1.8 Time Series Decomposition

### a. Intuition

Decomposition is the time series equivalent of breaking white light through a prism — you separate the mixed signal into its pure components. Once separated, you can model each component independently and combine them for forecasting.

### b. Mathematical Insight

Two main approaches:

**Classical Decomposition (moving average based):**
1. Estimate trend using centered moving average
2. Detrend by subtracting (additive) or dividing (multiplicative)
3. Estimate seasonal component by averaging detrended values by season
4. Residual = what's left

**STL Decomposition (Seasonal and Trend using Loess):**
- Uses locally weighted regression (LOESS) for trend and seasonal estimation
- More robust to outliers and allows seasonal component to change over time
- Recommended for production use

### c. How It Works

```
Classical:
  T_t = MA(y_t, window=m)         ← Centered moving average
  D_t = y_t - T_t                 ← Detrended
  S_t = mean(D_t by season)       ← Averaged seasonal pattern
  R_t = D_t - S_t                 ← Residual

STL:
  Iterates between seasonal smoother and trend smoother
  until convergence (more stable for messy data)
```

### e. Python Implementation

```python
import pandas as pd

from statsmodels.tsa.seasonal import seasonal_decompose, STL

    """

        series: Time series with DatetimeIndex

        fig, axes = plt.subplots(4, 1, figsize=(12, 10))
        for ax, (name, data) in zip(axes, [
            ('Observed', result.observed),
            ('Trend',    result.trend),
            ('Seasonal', result.seasonal),
            ('Residual', result.resid)
        ]):
            data.plot(ax=ax)
            ax.set_title(name)
    
    plt.tight_layout()
    plt.show()
    return result
```

## 1.9 Structural Breaks

### a. Intuition

A structural break is when the underlying data-generating process suddenly changes — like a hinge point in history. COVID-19 caused structural breaks in almost every economic time series. A factory upgrade might cause a step change in production efficiency. Models trained before the break can be dangerously wrong after it.

### b. Mathematical Insight

At breakpoint `τ`, the series shifts regime:

```
y_t = α₁ + β₁·t + ε_t    for t < τ
y_t = α₂ + β₂·t + ε_t    for t ≥ τ
```

The Chow test tests whether regression parameters are the same before and after a known breakpoint.

For unknown breakpoints: CUSUM test, Bai-Perron multiple breakpoint test.

### c. How It Works

1. Plot the series and look for obvious step changes or slope changes
2. Use CUSUM or Bai-Perron to detect unknown breakpoints
3. If breakpoints found: either model separately per regime, or include dummy variable at break date
4. Re-evaluate model fit after accounting for breaks

### e. Python Implementation

```python
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
from statsmodels.stats.diagnostic import breaks_cusumolsresid

def detect_structural_break(series: pd.Series, window: int = 20):
    """
    Detect potential structural breaks using rolling mean shift detection.
    """
    roll_mean = series.rolling(window).mean()
    roll_std  = series.rolling(window).std()
    
    # Z-score of rolling mean relative to global mean
    z_scores  = (roll_mean - series.mean()) / series.std()
    
    # Flag dates where rolling mean shifts significantly
    breaks = series.index[np.abs(z_scores) > 2].tolist()
    
    fig, axes = plt.subplots(2, 1, figsize=(12, 8))
    series.plot(ax=axes[0], label='Series', alpha=0.7)
    roll_mean.plot(ax=axes[0], label='Rolling Mean', color='red')
    for b in breaks[:3]:  # Mark first few potential breaks
        axes[0].axvline(x=b, color='orange', linestyle='--', alpha=0.7, label=f'Break: {b}')
    axes[0].legend(); axes[0].set_title('Series with Potential Structural Breaks')
    
    z_scores.plot(ax=axes[1], color='purple')
    axes[1].axhline(y=2,  color='red', linestyle='--', alpha=0.5)
    axes[1].axhline(y=-2, color='red', linestyle='--', alpha=0.5)
    axes[1].set_title('Z-Score of Rolling Mean')
    
    plt.tight_layout()
    plt.show()
    
    return breaks

# Simulate series with structural break at t=150
```

### f. When to Use / Avoid

- Always inspect for breaks when covering long historical periods (5+ years)
- Use domain knowledge: COVID, regulatory changes, product launches are common breaks
- Simple fix: use only data after the most recent break for training

## 1.10 Missing Timestamps & Irregular Series

### a. Intuition

Real-world time series are messy. Stock markets are closed on weekends. Sensors fail. Survey respondents skip weeks. Missing data in time series is more dangerous than in tabular data because the *position* of the missing value matters — the gap itself is informative.

An **irregular time series** has observations at uneven intervals (e.g., event logs, patient vitals recorded "when needed").

### b. Mathematical Insight

For regular series with missing values, imputation must respect temporal ordering. Interpolation assumes the series moves smoothly between observed values — valid only if the gap is short relative to the series dynamics.

### c. How It Works

1. Detect missing timestamps by comparing observed dates to expected regular grid
2. Choose strategy based on gap size and nature:
   - Short gaps (1-3 points): forward fill or linear interpolation
   - Longer gaps: use seasonality-aware imputation or treat as structural break
3. For irregular series: regularize using resampling to a fixed frequency

### e. Python Implementation

```python
import pandas as pd
import numpy as np

def handle_missing_timestamps(series: pd.Series, freq: str = 'D') -> pd.Series:
    """
    Detect and handle missing timestamps in a time series.
    
    Args:
        series: Time series with DatetimeIndex
        freq: Expected frequency string
    
    Returns:
        Imputed series on complete regular grid
    """
    # Step 1: Create complete regular index
    full_index = pd.date_range(
        start=series.index.min(),
        end=series.index.max(),
        freq=freq
    )
    
    # Step 2: Reindex to expose missing timestamps as NaN
    series_complete = series.reindex(full_index)
    
    # Step 3: Report missing timestamps
    n_missing = series_complete.isna().sum()
    pct_missing = 100 * n_missing / len(series_complete)
    print(f"Missing timestamps: {n_missing} ({pct_missing:.1f}%)")
    
    # Step 4: Imputation strategy based on gap size
    # Find gap lengths
    is_missing = series_complete.isna()
    gaps = is_missing.astype(int).groupby(
        (~is_missing).cumsum()
    ).transform('sum')
    
    # Short gaps: linear interpolation
    series_short  = series_complete.copy()
    short_mask    = (is_missing) & (gaps <= 3)
    series_short[short_mask] = np.nan  # ensure they're nan
    series_short  = series_short.interpolate(method='linear', limit=3)
    
    # Remaining long gaps: forward fill then backward fill
    series_filled = series_short.fillna(method='ffill').fillna(method='bfill')
    
    return series_filled

# Example: Create series with artificial missing dates
np.random.seed(42)
dates = pd.date_range('2023-01-01', periods=365, freq='D')
vals  = 50 + np.sin(2 * np.pi * np.arange(365) / 30) * 10 + np.random.randn(365)
ts    = pd.Series(vals, index=dates)

# Remove 15 random dates to simulate missing

ts_fixed = handle_missing_timestamps(ts_missing, freq='D')
print(f"\nOriginal length:  {len(ts)}")
print(f"With gaps length: {len(ts_missing)}")
print(f"Fixed length:     {len(ts_fixed)}")
```

### f. When to Use / Avoid

| Strategy | When to Use |
|----------|-------------|
| Forward fill | Last known value is good estimate (price data) |
| Linear interpolation | Short gaps, smooth series |
| Seasonal interpolation | Strong seasonality, gap < 1 season |
| Drop gaps | Gap is so large that interpolation is meaningless |
| Flag as separate regime | Gap caused by known structural change |

# 2. Classical Forecasting

## 2.1 Naive, Mean & Drift Methods

### a. Intuition

Before building complex models, always build simple baselines. They're surprisingly hard to beat, and they anchor your evaluation. Three essential baselines:

- **Naive:** Tomorrow will be exactly like today. `ŷ_{T+h} = y_T`
- **Mean:** Tomorrow will be the historical average. `ŷ_{T+h} = mean(y)`
- **Drift:** Project the trend between first and last observation forward. Linear extrapolation.

### b. Mathematical Insight

```
Naive:   ŷ_{T+h} = y_T

Mean:    ŷ_{T+h} = (1/T) × Σ y_t

Drift:   ŷ_{T+h} = y_T + h × (y_T - y_1) / (T - 1)
                   slope = (last - first) / time_elapsed
```

**Seasonal Naive:** `ŷ_{T+h} = y_{T+h-m}` — use the value from the same season last period. This beats all above methods for highly seasonal series.

### c. How It Works

1. Observe the last value (or historical values)
2. Apply formula directly — no fitting required
3. Use as lower bound for model performance

### e. Python Implementation

```python
import pandas as pd
import numpy as np

class BaselineForecaster:
    """
    Collection of baseline forecasting methods.
    All methods return point forecasts for h steps ahead.
    """
    
    @staticmethod
    def naive(series: pd.Series, h: int = 1) -> np.ndarray:
        """Forecast = last observed value."""
        return np.repeat(series.iloc[-1], h)
    
    @staticmethod
    def seasonal_naive(series: pd.Series, h: int, m: int = 12) -> np.ndarray:
        """Forecast = same period in last season."""
        forecasts = []
        for step in range(1, h + 1):
            # Look back m steps
            look_back = len(series) - m + ((step - 1) % m)
            if look_back < 0:
                look_back = 0
            forecasts.append(series.iloc[look_back])
        return np.array(forecasts)
    
    @staticmethod
    def mean_forecast(series: pd.Series, h: int = 1) -> np.ndarray:
        """Forecast = historical mean."""
        return np.repeat(series.mean(), h)
    
    @staticmethod
    def drift(series: pd.Series, h: int = 1) -> np.ndarray:
        """Forecast = linear extrapolation from first to last."""
        T     = len(series)
        slope = (series.iloc[-1] - series.iloc[0]) / (T - 1)
        return series.iloc[-1] + slope * np.arange(1, h + 1)
    
    @staticmethod
    def moving_average(series: pd.Series, h: int = 1, window: int = 7) -> np.ndarray:
        """Forecast = mean of last `window` observations."""
        ma = series.iloc[-window:].mean()
        return np.repeat(ma, h)


# Demonstrate
np.random.seed(42)
ts  = pd.Series(
    50 + np.arange(120) * 0.3 + 5 * np.sin(2 * np.pi * np.arange(120) / 12) + np.random.randn(120),
    index=pd.date_range('2014-01-01', periods=120, freq='M')
)

bf = BaselineForecaster()
h  = 12  # Forecast 12 months ahead

print("Baseline Forecasts (next 12 steps):")
print(f"  Naive:         {bf.naive(ts, h).mean():.2f} (constant)")
print(f"  Mean:          {bf.mean_forecast(ts, h).mean():.2f} (constant)")
print(f"  Drift (final): {bf.drift(ts, h)[-1]:.2f}")
print(f"  Seasonal Naive (first): {bf.seasonal_naive(ts, h, m=12)[0]:.2f}")
```

### f. When to Use / Avoid

| Method | Best For |
|--------|----------|
| Naive | Stable, no-trend data; very short horizons |
| Seasonal Naive | Strong seasonality; often the best simple baseline |
| Mean | Stationary series; long-horizon forecasting |
| Drift | Clear linear trend; short-to-medium horizons |

### g. Key Note

Always include at least seasonal naive as a baseline. If your model doesn't beat it on MAPE, the model adds no value.

## 2.2 Moving Average Smoothing

### a. Intuition

A moving average is a filter that smooths out noise by averaging neighboring values. Think of it as a sliding window that always shows you the "local average." It removes high-frequency fluctuations to reveal the underlying trend.

**Important distinction:** Moving average as a *smoother* (described here) vs. Moving Average (MA) as a *model component* in ARMA (Section 2.4) — these are different concepts sharing a name.

### b. Mathematical Insight

**Simple Moving Average (SMA):**
```
MA_t = (1/k) × Σ y_{t-i}    for i = 0 to k-1
```

**Centered Moving Average (for decomposition):**
```
CMA_t = (1/k) × Σ y_{t-i}   for i = -(k-1)/2 to (k-1)/2
```

For even-ordered windows (like k=12): apply 2×12-MA (average of two 12-MAs offset by one) to achieve centering.

**Weighted Moving Average:**
```
WMA_t = Σ w_i × y_{t-i}     where Σ w_i = 1
```

### e. Python Implementation

```python
import pandas as pd
```

- Use as a preprocessing step to visualize trend
- Use in decomposition (centered MA for trend estimation)
- **Do not** use as a forecasting method for data with strong trend or seasonality — exponential smoothing is better
- Larger window = smoother, but loses more recent information

## 2.3 Exponential Smoothing Family

### a. Intuition

Moving averages treat all window observations equally. But recent observations should matter more than old ones. Exponential smoothing solves this by assigning *exponentially decaying* weights: the most recent gets weight α, the one before gets weight α(1-α), and so on.

The family grows progressively more capable:
- **SES:** Handles level only (flat series)
- **Holt's:** Handles level + trend
- **Holt-Winters:** Handles level + trend + seasonality

### b. Mathematical Insight

**Simple Exponential Smoothing (SES):**
```
ŷ_{t+1} = α·y_t + (1-α)·ŷ_t

Level:    l_t = α·y_t + (1-α)·l_{t-1}
Forecast: ŷ_{T+h} = l_T   (flat forecast)
```

**Holt's Linear (Double Exponential Smoothing):**
```
Level: l_t = α·y_t + (1-α)·(l_{t-1} + b_{t-1})
Trend: b_t = β·(l_t - l_{t-1}) + (1-β)·b_{t-1}
Forecast: ŷ_{T+h} = l_T + h·b_T
```

**Holt-Winters Additive:**
```
Level:    l_t = α·(y_t - s_{t-m}) + (1-α)·(l_{t-1} + b_{t-1})
Trend:    b_t = β·(l_t - l_{t-1}) + (1-β)·b_{t-1}
Seasonal: s_t = γ·(y_t - l_{t-1} - b_{t-1}) + (1-γ)·s_{t-m}
Forecast: ŷ_{T+h} = l_T + h·b_T + s_{T+h-m}
```

Where `m` = seasonal period, and `α, β, γ ∈ [0, 1]`.

### c. How It Works

1. Initialize level, trend, and seasonal components
2. At each time step, update components using smoothing equations
3. Forecast by combining current estimates
4. Parameters (α, β, γ) are optimized by minimizing SSE on training data

### d. Visual Representation — Weight Decay

```
α = 0.3:   Most recent observation gets 30% weight
           
t   weight
T   30.0%  ████████████████████████████
T-1 21.0%  █████████████████████
T-2 14.7%  ██████████████
T-3 10.3%  ██████████
T-4  7.2%  ███████
...  ...   (exponentially decaying)
```

### e. Python Implementation

```python
from statsmodels.tsa.holtwinters import ExponentialSmoothing, SimpleExpSmoothing
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

def fit_exponential_smoothing_family(series: pd.Series, period: int = 12, forecast_h: int = 12):
    """
    Fit and compare SES, Holt's, and Holt-Winters models.
    """
    train = series[:-forecast_h]
    test  = series[-forecast_h:]
    
    models = {}
    
    # 1. Simple Exponential Smoothing
    ses = SimpleExpSmoothing(train).fit(optimized=True)
    models['SES'] = ses
    
    # 2. Holt's Linear Trend
    holts = ExponentialSmoothing(
        train,
        trend='add',
        damped_trend=True  # Damping prevents explosive forecasts
    ).fit(optimized=True)
    models["Holt's (damped)"] = holts
    
    # 3. Holt-Winters Additive
    hw_add = ExponentialSmoothing(
        train,
        trend='add',
        seasonal='add',
        seasonal_periods=period,
        damped_trend=True
    ).fit(optimized=True)
    models['Holt-Winters (Add)'] = hw_add
    
    # 4. Holt-Winters Multiplicative
    if (train > 0).all():  # Only valid for positive series
        hw_mul = ExponentialSmoothing(
            train,
            trend='add',
            seasonal='mul',
            seasonal_periods=period,
            damped_trend=True
        ).fit(optimized=True)
        models['Holt-Winters (Mul)'] = hw_mul
    
    # Evaluate
    from sklearn.metrics import mean_absolute_error
    print("=== Model Comparison ===")
    results = {}
    
    fig, ax = plt.subplots(figsize=(14, 6))
    series.plot(ax=ax, label='Observed', color='black', lw=1.5)
    
    for name, model in models.items():
        forecast = model.forecast(forecast_h)
        mae      = mean_absolute_error(test, forecast)
        results[name] = {'MAE': mae, 'params': model.params}
        print(f"  {name:30s} MAE: {mae:.3f}")
        forecast.plot(ax=ax, label=name, linestyle='--')
    
    test.plot(ax=ax, label='Actual Test', color='red', lw=2)
    ax.set_title('Exponential Smoothing Family Comparison')
    ax.legend()
    plt.tight_layout()
    plt.show()
    
    # Print smoothing parameters
    print("\n=== Smoothing Parameters ===")
    for name, info in results.items():
        params = info['params']
        alpha  = params.get('smoothing_level', 'N/A')
        beta   = params.get('smoothing_trend', 'N/A')
        gamma  = params.get('smoothing_seasonal', 'N/A')
        print(f"  {name}: α={alpha}, β={beta}, γ={gamma}")
    
    return models, results

# Generate test data
np.random.seed(42)
t = np.arange(120)
trend    = 50 + 0.5 * t
seasonal = 8 * np.sin(2 * np.pi * t / 12)
noise    = np.random.randn(120) * 2
ts = pd.Series(trend + seasonal + noise,
               index=pd.date_range('2014-01-01', periods=120, freq='M'))

models, results = fit_exponential_smoothing_family(ts, period=12, forecast_h=12)
```

### f. When to Use / Avoid

| Method | Use When |
|--------|----------|
| SES | No trend, no seasonality; short-range |
| Holt's | Clear trend, no seasonality |
| Holt-Winters Add | Trend + constant seasonal amplitude |
| Holt-Winters Mul | Trend + growing seasonal amplitude (positive data) |

### g. Key Hyperparameters

- `α` (smoothing_level): Near 1 = reactive to recent changes; near 0 = relies on history
- `β` (smoothing_trend): Near 1 = quickly adapts trend; near 0 = stable trend estimate
- `γ` (smoothing_seasonal): Near 1 = seasonal pattern updates quickly
- `damped_trend=True`: Almost always use — prevents forecasts from growing forever

## 2.4 AR and MA Models

### a. Intuition

**AR (AutoRegressive):** Predict tomorrow using a weighted sum of past observations. "I look at my past to predict my future." If you consistently sell more units this week when you sold a lot last week, that's an AR relationship.

**MA (Moving Average):** Predict tomorrow using a weighted sum of past *forecast errors*. "I learn from my mistakes." Not a moving average of values — a moving average of *errors*. This is subtle but important.

### b. Mathematical Insight

**AR(p) model:**
```
y_t = c + φ₁·y_{t-1} + φ₂·y_{t-2} + ... + φ_p·y_{t-p} + ε_t

Where: φ_i = autoregressive coefficients
       c = constant (mean level when stationary)
       ε_t = white noise error
```

**MA(q) model:**
```
y_t = μ + ε_t + θ₁·ε_{t-1} + θ₂·ε_{t-2} + ... + θ_q·ε_{t-q}

Where: θ_i = moving average coefficients
       μ = mean of series
       ε_t = white noise error
```

**Stationarity condition for AR(p):**
All roots of the characteristic polynomial `1 - φ₁·z - φ₂·z² - ... - φ_p·z^p = 0` must lie *outside* the unit circle.

### c. How It Works

**AR(1) step by step:**
1. The model is: `y_t = c + φ₁·y_{t-1} + ε_t`
2. At t=100: multiply yesterday's value by φ₁, add constant c, add random shock
3. Forecasting 1-step: `ŷ_{101} = c + φ₁·y_{100}`
4. Forecasting h-steps: substitute previous forecasts iteratively

**MA(1) step by step:**
1. The model is: `y_t = μ + ε_t + θ₁·ε_{t-1}`
2. At t=100: add the current error plus θ₁ times yesterday's error
3. Key: MA forecasts always revert to mean (μ) after q steps

### d. Identification Rules (from ACF/PACF)

```
Model  | ACF            | PACF
-------|----------------|------------------
AR(p)  | Tails off      | Cuts off at lag p
MA(q)  | Cuts off lag q | Tails off
ARMA   | Tails off      | Tails off
```

### e. Python Implementation

```python

    """

# Generate AR(2) process and fit

np.random.seed(42)

# Check stationarity first
from statsmodels.tsa.stattools import adfuller
p_val = adfuller(ar2)[1]
print(f"ADF p-value: {p_val:.4f} ({'Stationary' if p_val < 0.05 else 'Non-Stationary'})")

# Identify order from ACF/PACF
fig, axes = plt.subplots(1, 2, figsize=(14, 5))
plot_acf(ar2, lags=20, ax=axes[0])
plot_pacf(ar2, lags=20, ax=axes[1])

axes[1].set_title('PACF — cuts off at 2 (AR(2) identified)')
plt.tight_layout()
plt.show()

# Fit AR(2)
result = fit_ar_ma(ar2, p=2, q=0)
```

### f. When to Use / Avoid

| Model | Use When |
|-------|----------|
| AR(p) | PACF cuts off at p; series depends on recent values |
| MA(q) | ACF cuts off at q; series absorbs past shocks |
| Neither | No significant lags → no structure to model |

### g. Key Hyperparameters

- `p`: AR order — determined by PACF cutoff lag
- `q`: MA order — determined by ACF cutoff lag
- `d`: Differencing order — determined by stationarity tests

## 2.5 ARMA, ARIMA & SARIMA

### a. Intuition

**ARMA = AR + MA** in one model. Most real series need both past values *and* past errors to be well described.

**ARIMA = ARMA + Differencing** to handle non-stationary series. The "I" stands for Integrated — differencing `d` times makes the series stationary before fitting ARMA.

**SARIMA = ARIMA + Seasonal terms** — the full model that handles seasonal data. It's ARIMA applied at two scales simultaneously: within-season (p, d, q) and across-seasons (P, D, Q).

### b. Mathematical Insight

**ARIMA(p, d, q):**
```
Φ(B)·Δ^d·y_t = Θ(B)·ε_t

Where:
  B = backshift operator: B·y_t = y_{t-1}
  Δ = first difference: Δ·y_t = y_t - y_{t-1} = (1-B)·y_t
  Φ(B) = AR polynomial: 1 - φ₁B - φ₂B² - ... - φ_pB^p
  Θ(B) = MA polynomial: 1 + θ₁B + θ₂B² + ... + θ_qB^q
  d = number of differences to achieve stationarity
```

**SARIMA(p, d, q)(P, D, Q)_m:**
```
Φ(B)·Φ_s(B^m)·Δ^d·Δ_m^D·y_t = Θ(B)·Θ_s(B^m)·ε_t

Where:
  P, D, Q = seasonal AR, I, MA orders
  m = seasonal period
  Φ_s, Θ_s = seasonal polynomials operating at lag m
```

### c. How It Works

**ARIMA fitting process:**
1. Test for stationarity → determine `d`
2. Difference `d` times
3. Inspect ACF/PACF of differenced series → determine `p` and `q`
4. Fit model → check residuals for white noise
5. Iterate if residuals still show structure

**Information Criteria for model selection:**
```
AIC = -2·log(L) + 2k      (penalizes complexity)
BIC = -2·log(L) + k·log(n) (penalizes more strongly for large n)

Lower is better. BIC prefers simpler models.
```

### d. Visual Representation (SARIMA Structure)

```
SARIMA(1,1,1)(1,1,1)_12

Non-seasonal part:  [AR(1)][Diff(1)][MA(1)]
Seasonal part:      [SAR(1)][SDiff(1)][SMA(1)]_12

Both parts multiply together in the lag polynomial
```

### e. Python Implementation

```python

                       max_P: int = 2, max_Q: int = 2, D: int = 1):

    """

            range(max_p + 1), range(max_q + 1),
            range(max_P + 1), range(max_Q + 1)
        ):

                    series,

                    seasonal_order=(P, D, Q, m),

                    enforce_invertibility=False
                )
                fitted = model.fit(disp=False)
                results_list.append({
                    'order': (p, d, q),
                    'seasonal_order': (P, D, Q, m),
                    'AIC': fitted.aic,
                    'BIC': fitted.bic,
                    'model': fitted
                })
            except Exception:
                continue
    else:
        for p, q in itertools.product(range(max_p + 1), range(max_q + 1)):
            try:

                fitted = model.fit()
                results_list.append({
                    'order': (p, d, q),
                    'seasonal_order': None,

                    'BIC': fitted.bic,

                })
            except Exception:
                continue

    print("=== Top 5 Models by AIC ===")
```

## 2.6 Model Assumptions & Order Selection

### a. Key ARIMA Assumptions

1. **Stationarity:** The differenced series must be stationary
2. **No autocorrelation in residuals:** Residuals should look like white noise
3. **Constant residual variance:** No heteroskedasticity
4. **Approximately normal residuals:** For valid confidence intervals

### b. Residual Diagnostics Checklist

After fitting any model, always check:

```
✓ ACF of residuals: no significant spikes at any lag
✓ Ljung-Box test: p-value > 0.05 (fail to reject white noise)
✓ QQ plot: points near the diagonal (normality)
✓ Residuals vs fitted: no pattern (constant variance)
```

### e. Python Implementation

```python
from statsmodels.stats.diagnostic import acorr_ljungbox
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
from scipy import stats

def residual_diagnostics(residuals: pd.Series, lags: int = 20):
    """
    Comprehensive residual diagnostics for fitted time series model.
    """
    fig, axes = plt.subplots(2, 2, figsize=(14, 10))
    
    # 1. Residual time plot
    residuals.plot(ax=axes[0, 0])
    axes[0, 0].axhline(0, color='red', linestyle='--')
    axes[0, 0].set_title('Residuals over Time')
    
    # 2. ACF of residuals
    from statsmodels.graphics.tsaplots import plot_acf
    plot_acf(residuals.dropna(), lags=lags, ax=axes[0, 1])
    axes[0, 1].set_title('ACF of Residuals (should be white noise)')
    
    # 3. Histogram
    axes[1, 0].hist(residuals.dropna(), bins=30, density=True, alpha=0.7)
    x = np.linspace(residuals.min(), residuals.max(), 100)
    axes[1, 0].plot(x, stats.norm.pdf(x, residuals.mean(), residuals.std()), 'r-', lw=2)
    axes[1, 0].set_title('Residual Distribution (vs Normal)')
    
    # 4. QQ plot
    stats.probplot(residuals.dropna(), plot=axes[1, 1])
    axes[1, 1].set_title('Q-Q Plot')
    
    plt.tight_layout()
    plt.show()
    
    # Ljung-Box test
    lb_result = acorr_ljungbox(residuals.dropna(), lags=[10, 20], return_df=True)
    print("=== Ljung-Box Test (p > 0.05 = no autocorrelation ✓) ===")
    print(lb_result[['lb_stat', 'lb_pvalue']].to_string())
    
    # Normality test (Shapiro-Wilk for small samples)
    if len(residuals.dropna()) < 5000:
        _, p_norm = stats.shapiro(residuals.dropna()[:2000])
        print(f"\nShapiro-Wilk normality p-value: {p_norm:.4f} "
              f"({'Normal ✓' if p_norm > 0.05 else 'Non-Normal ✗'})")
```

# 3. Feature Engineering for Time Series

## 3.1 Lag Features

### a. Intuition

If you want to use gradient boosting or any regression model on time series, you need to convert the sequential structure into tabular features. The most fundamental: lag features. `lag_1` = the value from one step ago. `lag_7` = the value from seven steps ago (same day last week, for daily data).

This transforms forecasting into: "Given what I know about the recent past, predict the next value."

### b. Mathematical Insight

A supervised ML formulation with lag features:

```
Input:  X_t = [y_{t-1}, y_{t-2}, ..., y_{t-p}]
Output: y_t

Model learns: f(y_{t-1}, y_{t-2}, ..., y_{t-p}) ≈ y_t
```

This is equivalent to an AR(p) model but allows nonlinear relationships via tree-based methods.

### c. How It Works

1. Choose meaningful lag values based on domain knowledge and ACF
2. Shift the target column by k positions to create `lag_k`
3. Drop rows with NaN (the first `max_lag` rows)
4. **Critical:** Never use future information — always shift forward, never backward

### e. Python Implementation

```python
import pandas as pd
import numpy as np

def create_lag_features(df: pd.DataFrame, 
                         target_col: str,
                         lags: list) -> pd.DataFrame:
    """
    Create lag features for a time series target column.
    
    Args:
        df: DataFrame with DatetimeIndex
        target_col: Column to create lags for
        lags: List of lag periods (e.g., [1, 7, 14, 28])
    
    Returns:
        DataFrame with lag features added
    """
    df_out = df.copy()
    
    for lag in lags:
        col_name = f'{target_col}_lag_{lag}'
        df_out[col_name] = df_out[target_col].shift(lag)
        # .shift(k) shifts DOWN by k → lag_k[t] = y[t-k] ✓
    
    # Report NaN rows introduced
    n_nan = df_out[f'{target_col}_lag_{max(lags)}'].isna().sum()
    print(f"Lag features created. First {n_nan} rows have NaN (will be dropped before training).")
    
    return df_out

# Example
np.random.seed(42)
dates  = pd.date_range('2022-01-01', periods=200, freq='D')
y      = np.cumsum(np.random.randn(200)) + 100
df     = pd.DataFrame({'sales': y}, index=dates)

# Create lags based on ACF (lag 1, 7 for weekly pattern, 30 for monthly)
df_lagged = create_lag_features(df, 'sales', lags=[1, 2, 3, 7, 14, 28])

# Drop rows with NaN before training
df_train = df_lagged.dropna()
print(f"\nFeatures: {[c for c in df_train.columns if 'lag' in c]}")
print(df_train.head(3))
```

### f. When to Use / Avoid

- **Use:** When series has clear autocorrelation (check ACF)
- **Use:** With gradient boosting, random forests, XGBoost for nonlinear AR relationships
- **Avoid:** Very sparse data — lags amplify NaN issues
- **Critical:** Do NOT use `df.shift(-k)` (negative shift) — that's look-ahead leakage

## 3.2 Rolling & Expanding Window Features

### a. Intuition

Beyond raw lag values, summary statistics of recent windows capture useful patterns:
- Rolling mean: smooth local trend
- Rolling std: local volatility
- Rolling min/max: recent range

**Expanding window** grows from the beginning to the current row — useful for "historical cumulative statistics." The difference: rolling uses a fixed-size recent window; expanding uses all history up to now.

### b. Mathematical Insight

```
Rolling mean (window w):  rm_t = mean(y_{t-1}, ..., y_{t-w})
Rolling std:              rs_t = std(y_{t-1}, ..., y_{t-w})

Expanding mean:           em_t = mean(y_1, y_2, ..., y_{t-1})

Key: Both must use only PAST data → shift by 1 before computing!
```

### e. Python Implementation

```python
import pandas as pd
import numpy as np

def create_window_features(df: pd.DataFrame, 
                            target_col: str,
                            rolling_windows: list = [7, 14, 30],
                            stats: list = ['mean', 'std', 'min', 'max']) -> pd.DataFrame:
    """
    Create rolling and expanding window features.
    
    CRITICAL: Always use .shift(1) before .rolling() to prevent leakage.
    """
    df_out = df.copy()
    
    # Shift target first to prevent look-ahead leakage
    shifted = df_out[target_col].shift(1)
    
    # Rolling window features
    for window in rolling_windows:
        rolled = shifted.rolling(window=window, min_periods=1)
        
        for stat in stats:
            col_name = f'{target_col}_rolling_{window}_{stat}'
            if stat == 'mean':
                df_out[col_name] = rolled.mean()
            elif stat == 'std':
                df_out[col_name] = rolled.std()
            elif stat == 'min':
                df_out[col_name] = rolled.min()
            elif stat == 'max':
                df_out[col_name] = rolled.max()
    
    # Expanding window features (always use full history up to t-1)
    expanding = shifted.expanding(min_periods=1)
    df_out[f'{target_col}_expanding_mean'] = expanding.mean()
    df_out[f'{target_col}_expanding_std']  = expanding.std()
    
    return df_out


# Example

df     = pd.DataFrame({'sales': y}, index=dates)

print(df_features.filter(like='rolling').head(5))
print("\nExpanding features:")
print(df_features.filter(like='expanding').head(5))
```

### f. When to Use / Avoid

- Rolling features: excellent for capturing local trends and volatility
- Expanding features: use when long-run cumulative stats matter (e.g., total revenue to date)
- **Always shift(1) before rolling** — this is the single most common source of leakage in time series ML
- Window selection: guided by domain knowledge (7 for weekly cycle, 30 for monthly)

## 3.3 Differencing

### a. Intuition

Differencing is simple but powerful: subtract the previous value from the current one. You transform the question from "what will the absolute level be?" to "how much will it change?"

Regular differencing removes trends. Seasonal differencing removes seasonal patterns.

### b. Mathematical Insight

```
First difference:    Δy_t = y_t - y_{t-1}
Second difference:   Δ²y_t = Δy_t - Δy_{t-1} = y_t - 2y_{t-1} + y_{t-2}

Seasonal difference: Δ_m·y_t = y_t - y_{t-m}

Combined:            Δ·Δ_m·y_t = y_t - y_{t-1} - y_{t-m} + y_{t-m-1}
```

### e. Python Implementation

```python
import pandas as pd
import numpy as np
from statsmodels.tsa.stattools import adfuller

def apply_differencing(series: pd.Series, 
                        regular_d: int = 1, 
                        seasonal_d: int = 1,
                        seasonal_period: int = 12) -> dict:
    """
    Apply regular and/or seasonal differencing and report stationarity.
    """
    result = {'original': series}
    current = series.copy()
    
    # Regular differencing
    for i in range(regular_d):
        current = current.diff().dropna()
        result[f'diff_{i+1}'] = current
    
    result_after_regular = current.copy()
    
    # Seasonal differencing
    for i in range(seasonal_d):
        current = current.diff(periods=seasonal_period).dropna()
        result[f'seasonal_diff_{i+1}'] = current
    
    # Stationarity test at each stage
    print("=== Stationarity at Each Differencing Stage ===")
    for name, s in result.items():
        if len(s.dropna()) < 5:
            continue
        p_val = adfuller(s.dropna())[1]
        status = '✓ Stationary' if p_val < 0.05 else '✗ Non-Stationary'
        print(f"  {name:30s}  ADF p={p_val:.4f}  {status}")
    
    return result

# Demo
np.random.seed(42)
t   = np.arange(120)
ts  = pd.Series(
    50 + 0.5*t + 8*np.sin(2*np.pi*t/12) + np.random.randn(120)*2,
    index=pd.date_range('2014-01-01', periods=120, freq='M')
)

results = apply_differencing(ts, regular_d=1, seasonal_d=1, seasonal_period=12)
```

### f. When to Use / Avoid

- Use d=1 for most economic/business series with trend
- Use D=1 for strong seasonality (monthly retail, weekly traffic)
- **Avoid over-differencing:** check ADF at each step; stop when stationary
- If variance is unstable: log-transform first, then difference

## 3.4 Trend & Seasonal Features

### a. Intuition

For supervised ML models (XGBoost, LightGBM), we need to encode trend and seasonality as explicit features — these models don't automatically "see" time structure.

**Trend features:** A simple integer time index, or polynomial terms. Tells the model "data is at time step 100, not step 5."

**Seasonal indicators:** Dummy variables for each season (month 1, 2, ..., 12), or interactions.

### e. Python Implementation

```python
import pandas as pd
import numpy as np

def create_trend_seasonal_features(df: pd.DataFrame, 
                                    target_col: str = None) -> pd.DataFrame:
    """
    Create explicit trend and seasonal indicator features from DatetimeIndex.
    """
    df_out = df.copy()
    
    # Trend features
    df_out['time_index']  = np.arange(len(df_out))           # Linear trend proxy
    df_out['time_index2'] = df_out['time_index'] ** 2        # Quadratic term
    df_out['log_time']    = np.log1p(df_out['time_index'])   # Log growth
    
    # Month-of-year seasonal dummies (drop first to avoid multicollinearity)
    if hasattr(df_out.index, 'month'):
        month_dummies = pd.get_dummies(df_out.index.month, prefix='month', drop_first=True)
        month_dummies.index = df_out.index
        df_out = pd.concat([df_out, month_dummies], axis=1)
        df_out.columns = df_out.columns.astype(str)
    
    # Quarter dummies
    if hasattr(df_out.index, 'quarter'):
        qtr_dummies = pd.get_dummies(df_out.index.quarter, prefix='quarter', drop_first=True)
        qtr_dummies.index = df_out.index
        df_out = pd.concat([df_out, qtr_dummies], axis=1)
        df_out.columns = df_out.columns.astype(str)
    
    print(f"Added trend/seasonal features. Total columns: {len(df_out.columns)}")
    return df_out
```

## 3.5 Date-Time Features & Cyclical Encoding

### a. Intuition

Raw date parts (month=12, day=31) are treated as arbitrary integers by tree-based models. But month 12 is close to month 1 (it's right before January), not far from it. A model treating 12 as "large" and 1 as "small" misses this cyclical proximity.

**Cyclical encoding** using sine and cosine transforms preserves this circular structure.

### b. Mathematical Insight

For a feature `x` with period `P`:
```
sin_encoded = sin(2π × x / P)
cos_encoded = cos(2π × x / P)

Month (P=12): month_sin = sin(2π × month / 12)
              month_cos = cos(2π × month / 12)
```

Together, `(sin, cos)` form a 2D point on a unit circle. December (12) and January (1) end up near each other on this circle. ✓

### e. Python Implementation

```python
import pandas as pd
import numpy as np

def create_datetime_features(df: pd.DataFrame, 
                               use_cyclical: bool = True) -> pd.DataFrame:
    """
    Extract and encode date-time features from DatetimeIndex.
    
    Args:
        df: DataFrame with DatetimeIndex
        use_cyclical: If True, encode cyclic features with sin/cos
    
    Returns:
        DataFrame with date-time features added
    """
    df_out = df.copy()
    idx    = df_out.index
    
    # Basic date-time components
    df_out['hour']         = idx.hour if hasattr(idx, 'hour') else 0
    df_out['day_of_week']  = idx.dayofweek          # 0=Monday, 6=Sunday
    df_out['day_of_month'] = idx.day
    df_out['week_of_year'] = idx.isocalendar().week.astype(int)
    df_out['month']        = idx.month
    df_out['quarter']      = idx.quarter
    df_out['year']         = idx.year
    
    # Binary flags
    df_out['is_weekend']   = (idx.dayofweek >= 5).astype(int)
    df_out['is_month_end'] = idx.is_month_end.astype(int)
    df_out['is_quarter_end'] = idx.is_quarter_end.astype(int)
    
    if use_cyclical:
        # Cyclical encoding for periodic features
        cyclical_features = {
            'hour':         24,
            'day_of_week':  7,
            'day_of_month': 31,
            'week_of_year': 52,
            'month':        12,
        }
        
        for feature, period in cyclical_features.items():
            if feature in df_out.columns:
                df_out[f'{feature}_sin'] = np.sin(2 * np.pi * df_out[feature] / period)
                df_out[f'{feature}_cos'] = np.cos(2 * np.pi * df_out[feature] / period)
                # Drop raw feature to avoid redundancy (optional)
                # df_out.drop(columns=[feature], inplace=True)
    
    print(f"Date-time features created: {[c for c in df_out.columns if c not in df.columns]}")
    return df_out


# Example
dates  = pd.date_range('2022-01-01', periods=365, freq='D')
df     = pd.DataFrame({'sales': np.random.randn(365) * 10 + 100}, index=dates)

# Verify cyclical encoding: Dec (month=12) should be near Jan (month=1)
print("\nCyclical month encoding:")

    df_dt['month'].isin([1, 12])
].drop_duplicates())
```

### f. When to Use / Avoid

- Always use datetime features for daily/hourly data
- Use cyclical encoding when your model can't learn cyclic relationships (linear models, neural networks)
- Tree-based models (XGBoost) are less sensitive to cyclical encoding but it still helps
- `is_weekend`, `is_month_end` are often among the most predictive features

## 3.6 Supervised Learning Formulation

### a. Intuition

This is the bridge between time series analysis and standard ML. Instead of using specialized time series models (ARIMA), we reformulate the problem as a regression:

"Use features derived from the past to predict the future value."

Once reformulated, you can apply XGBoost, LightGBM, Random Forests, or any regression model.

### b. Mathematical Insight

```
Standard forecasting: ŷ_{T+h} = f(y_1, y_2, ..., y_T)

Supervised ML formulation:
  X_t = [lag_1, lag_7, rolling_7_mean, month_sin, month_cos, ...]
  y_t = y_{t+h}  ← the h-step-ahead target

Model: ŷ_{t+h} = f(X_t)

This is trained on (X_1, y_{1+h}), (X_2, y_{2+h}), ..., (X_{T-h}, y_T)
```

### e. Full Pipeline Implementation

```python
import pandas as pd
import numpy as np
from sklearn.ensemble import GradientBoostingRegressor
from sklearn.metrics import mean_absolute_error
import matplotlib.pyplot as plt

def build_supervised_ts_dataset(series: pd.Series,
                                  forecast_horizon: int = 1,
                                  lag_periods: list = [1, 7, 14, 28],
                                  rolling_windows: list = [7, 14]) -> pd.DataFrame:
    """
    Transform time series into supervised ML dataset.
    
    Args:
        series: Target time series
        forecast_horizon: h steps ahead to predict
        lag_periods: Which lags to use as features
        rolling_windows: Rolling window sizes for summary stats
    
    Returns:
        DataFrame with features (X) and target (y)
    """
    df = pd.DataFrame({'target': series})
    
    # Lag features (shifted by horizon to avoid leakage)
    for lag in lag_periods:
        actual_lag = lag + forecast_horizon - 1  # Account for forecast horizon
        df[f'lag_{lag}'] = series.shift(actual_lag)
    
    # Rolling features (also shifted by horizon)
    shifted = series.shift(forecast_horizon)
    for w in rolling_windows:
        df[f'roll_{w}_mean'] = shifted.rolling(w).mean()
        df[f'roll_{w}_std']  = shifted.rolling(w).std()
    
    # Datetime features
    df['month']         = df.index.month
    df['month_sin']     = np.sin(2 * np.pi * df.index.month / 12)
    df['month_cos']     = np.cos(2 * np.pi * df.index.month / 12)
    df['day_of_week']   = df.index.dayofweek
    df['dow_sin']       = np.sin(2 * np.pi * df.index.dayofweek / 7)
    df['dow_cos']       = np.cos(2 * np.pi * df.index.dayofweek / 7)
    df['is_weekend']    = (df.index.dayofweek >= 5).astype(int)
    
    # Target: h-step-ahead value
    df['y'] = series.shift(-forecast_horizon)
    
    # Drop rows with NaN (introduced by lags and target shift)
    df.dropna(inplace=True)
    
    return df

np.random.seed(42)
dates  = pd.date_range('2020-01-01', periods=500, freq='D')
noise  = np.random.randn(500)

# Time-based split (NEVER random split for time series!)

# Evaluate

```

### f. When to Use / Avoid

- **Use:** When you have many features beyond the target (exogenous variables)
- **Use:** When relationships are nonlinear (tree-based models shine here)
- **Use:** For multi-step forecasting via direct strategy (one model per horizon)
- **Avoid:** When you have very short series (<100 observations) — lags consume too many rows
- **Critical:** Always validate with walk-forward CV (Section 4.2), not random splits

# 4. Evaluation Techniques

## 4.1 Train-Test Splits for Time Series

### a. Intuition

The golden rule: **never shuffle time series data.** A train-test split must respect temporal ordering — train on the past, test on the future. Shuffling lets the model "see the future" during training, which makes evaluation metrics meaningless.

Think of it this way: would you train a stock price predictor by randomly mixing years together? The resulting model would "know" 2025 prices while predicting 2022 prices — that's cheating.

### b. Mathematical Insight

```
Correct split (temporal):
  Train: y_1, y_2, ..., y_{T×0.8}
  Test:  y_{T×0.8 + 1}, ..., y_T
  
  → Test set is always in the future relative to training

Wrong split (random):
  Train: {y_5, y_120, y_3, y_88, ...} ← mixes future and past
  Test:  {y_2, y_77, y_91, ...}       ← some "past" relative to train
```

### c. How It Works

```
Timeline: |─────── TRAIN ──────────|───── TEST ─────|
              past ────────────────────────────► future
              
              ↑ Train on everything before test window
                                    ↑ Evaluate on held-out future
```

### e. Python Implementation

```python
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt

def temporal_train_test_split(series: pd.Series, 
                               test_size: float = 0.2,
                               test_periods: int = None) -> tuple:
    """
    Proper temporal train-test split for time series.
    
    Args:
        series: Time series (must be ordered)
        test_size: Fraction for test (if test_periods not specified)
        test_periods: Exact number of periods for test set
    
    Returns:
        (train, test) tuple
    """
    n = len(series)
    
    if test_periods is not None:
        split_idx = n - test_periods
    else:
        split_idx = int(n * (1 - test_size))
    
    train = series.iloc[:split_idx]
    test  = series.iloc[split_idx:]
    
    print(f"Train: {train.index[0]} to {train.index[-1]} ({len(train)} obs)")
    print(f"Test:  {test.index[0]} to {test.index[-1]} ({len(test)} obs)")
    
    # Visualize
    fig, ax = plt.subplots(figsize=(14, 5))
    train.plot(ax=ax, label='Train', color='steelblue')
    test.plot(ax=ax, label='Test', color='tomato')
    ax.axvline(x=train.index[-1], color='black', linestyle='--', label='Split point')
    ax.legend(); ax.set_title('Temporal Train-Test Split')
    plt.tight_layout()
    plt.show()
    
    return train, test

# Example
np.random.seed(42)
ts   = pd.Series(np.cumsum(np.random.randn(365)) + 100,
                 index=pd.date_range('2022-01-01', periods=365, freq='D'))

```

### f. When to Use / Avoid

| Strategy | Use When |
|----------|----------|
| Single temporal split | Quick baseline evaluation; large datasets |
| Walk-forward CV | Production model validation; limited data |
| Multiple test windows | Testing stability across different time periods |

## 4.2 Walk-Forward & Rolling Validation

### a. Intuition

A single train-test split only tells you how the model performs on one specific future window. What if that window happened to be unusually easy or hard?

**Walk-forward validation** simulates real forecasting: train on all data up to time t, predict the next window, advance t by one step, repeat. You get performance across many different "present moments."

**Rolling vs Expanding window:**
- **Rolling:** Fixed-size training window slides forward (recent data only)
- **Expanding:** Training window grows over time (all historical data)

### b. Mathematical Insight

```
Walk-Forward Validation:

Step 1: Train on [1..100],    predict [101..112]
Step 2: Train on [1..112],    predict [113..124]  ← expanding
         or on [13..112],     predict [113..124]  ← rolling (fixed size)
Step 3: Train on [1..124],    predict [125..136]
...

Final score: mean across all validation windows
```

### c. Visual Representation

```
Expanding (Anchored):
  |████████|░░░|
  |████████████|░░░|
  |████████████████|░░░|
   Train      Test (horizon)

Rolling (Sliding):
  |████████|░░░|
     |████████|░░░|
        |████████|░░░|
        Fixed size Train  Test
```

### e. Python Implementation

```python
import pandas as pd
import numpy as np
from sklearn.metrics import mean_absolute_error
from typing import Generator, Tuple
import matplotlib.pyplot as plt

def walk_forward_splits(series: pd.Series,
                         initial_train_size: int,
                         test_size: int = 1,
                         step_size: int = 1,
                         rolling: bool = False,
                         rolling_window: int = None) -> Generator:
    """
    Generate (train, test) index pairs for walk-forward validation.
    
    Args:
        series: Time series
        initial_train_size: Minimum training observations
        test_size: Number of steps to predict per fold
        step_size: How many steps to advance each fold
        rolling: If True, use fixed-size rolling window
        rolling_window: Size of rolling training window
    
    Yields:
        (train_idx, test_idx) slices
    """
    n = len(series)
    start = initial_train_size
    
    while start + test_size <= n:
        if rolling and rolling_window:
            train_start = max(0, start - rolling_window)
        else:
            train_start = 0
        
        train_idx = slice(train_start, start)
        test_idx  = slice(start, start + test_size)
        
        yield train_idx, test_idx
        start += step_size


def walk_forward_evaluate(series: pd.Series,
                            model_fn,          # callable(train) -> forecast function
                            initial_train: int = 60,
                            horizon: int = 12,
                            step: int = 12,
                            rolling: bool = False) -> pd.DataFrame:
    """
    Evaluate a forecasting model using walk-forward validation.
    
    Args:
        series: Full time series
        model_fn: Function that accepts train series, returns (fitted_model, forecast_fn)
        initial_train: Minimum training periods
        horizon: Forecast horizon
        step: Walk-forward step size
        rolling: Use rolling window?
    
    Returns:
        DataFrame of errors per fold
    """
    results = []
    
    splits = list(walk_forward_splits(
        series, initial_train_size=initial_train,
        test_size=horizon, step_size=step, rolling=rolling
    ))
    
    print(f"Walk-forward validation: {len(splits)} folds")
    
    for fold_idx, (train_idx, test_idx) in enumerate(splits):
        train = series.iloc[train_idx]
        test  = series.iloc[test_idx]
        
        try:
            model, forecast_fn = model_fn(train)
            forecast = forecast_fn(horizon)
            
            # Align lengths
            min_len  = min(len(test), len(forecast))
            mae      = mean_absolute_error(test[:min_len], forecast[:min_len])
            mape     = np.mean(np.abs((test[:min_len] - forecast[:min_len]) / test[:min_len])) * 100
            
            results.append({
                'fold': fold_idx + 1,
                'train_end': train.index[-1],
                'test_start': test.index[0],
                'test_end': test.index[-1],
                'train_size': len(train),
                'MAE': mae,
                'MAPE': mape
            })
        except Exception as e:
            print(f"  Fold {fold_idx + 1} failed: {e}")
    
    results_df = pd.DataFrame(results)
    
    print("\n=== Walk-Forward Validation Results ===")
    print(results_df[['fold', 'train_size', 'MAE', 'MAPE']].to_string(index=False))
    print(f"\nMean MAE:  {results_df['MAE'].mean():.4f} ± {results_df['MAE'].std():.4f}")
    print(f"Mean MAPE: {results_df['MAPE'].mean():.2f}% ± {results_df['MAPE'].std():.2f}%")
    
    return results_df


# Demo: walk-forward with exponential smoothing

    index=pd.date_range('2014-01-01', periods=120, freq='M')

    """Holt-Winters model factory."""
    model  = ExponentialSmoothing(train, trend='add', seasonal='add', 
                                   seasonal_periods=12, damped_trend=True)
    fitted = model.fit(optimized=True)
    
    def forecast_fn(h):
        return fitted.forecast(h).values
```

## 4.3 Forecast Horizon & Step Types

### a. Intuition

**Forecast horizon h:** How many steps into the future you're predicting. Predicting next day (h=1) is much easier than predicting next quarter (h=90).

**One-step forecasting:** Predict one step at a time, using actual observed values as inputs. Easier, but not realistic for operational forecasting.

**Multi-step forecasting:** Predict h steps at once. Three strategies:

1. **Direct:** Train one model per horizon (`model_h1`, `model_h2`, ..., `model_h12`)
2. **Recursive:** Use model_h1 iteratively; previous forecast becomes next input
3. **MIMO (Multi-Input Multi-Output):** One model predicts all h steps simultaneously

### b. Trade-offs

```
Strategy    | Complexity | Error accumulation | Memory |
------------|------------|-------------------|--------|
Direct      | High       | None (independent)| High   |
Recursive   | Low        | Yes (errors stack)| Low    |
MIMO        | Medium     | None              | Medium |
```

### e. Python Implementation

```python
import numpy as np
import pandas as pd
from sklearn.linear_model import Ridge

def direct_multistep_forecast(series: pd.Series, 
                               horizon: int = 12, 
                               lag_periods: list = [1, 2, 3, 12]) -> np.ndarray:
    """
    Direct multi-step forecasting: one model per horizon step.
    """
    X_all = pd.DataFrame()
    for lag in lag_periods:
        X_all[f'lag_{lag}'] = series.shift(lag)
    
    X_all.dropna(inplace=True)
    
    models   = []
    forecasts = []
    
    for h in range(1, horizon + 1):
        # Target for this horizon
        y_h = series.shift(-h).reindex(X_all.index).dropna()
        X_h = X_all.reindex(y_h.index)
        
        # Train/test split (last 20%)
        split = int(0.8 * len(X_h))
        
        model = Ridge()
        model.fit(X_h.iloc[:split], y_h.iloc[:split])
        models.append(model)
    
    # Generate actual forecasts using last known values
    last_features = X_all.iloc[-1:].values
    for model in models:
        forecasts.append(model.predict(last_features)[0])
    
    return np.array(forecasts)


def recursive_multistep_forecast(series: pd.Series, 
                                   horizon: int = 12,
                                   lag_periods: list = [1, 2, 3, 12]) -> np.ndarray:
    """
    Recursive multi-step forecasting: use previous predictions as inputs.
    """
    from sklearn.ensemble import GradientBoostingRegressor
    
    X_all = pd.DataFrame()
    for lag in lag_periods:
        X_all[f'lag_{lag}'] = series.shift(lag)
    
    X_all.dropna(inplace=True)
    y_all = series.reindex(X_all.index)
    
    split = int(0.8 * len(X_all))
    model = GradientBoostingRegressor(n_estimators=100, max_depth=3)
    model.fit(X_all.iloc[:split], y_all.iloc[:split])
    
    # Recursive prediction
    history   = list(series.values)
    forecasts = []
    
    for _ in range(horizon):
        # Build feature vector from recent history
        features = [history[-(lag)] if lag <= len(history) else 0 
                    for lag in lag_periods]
        pred = model.predict([features])[0]
        forecasts.append(pred)
        history.append(pred)  # Add prediction to history for next step
    
    return np.array(forecasts)
```

### f. When to Use / Avoid

| Strategy | Best For |
|----------|----------|
| Direct | When models for different horizons behave differently; avoids error accumulation |
| Recursive | Operationally simpler; works well for short horizons (<7 steps) |
| MIMO | When correlations between future steps matter |

## 4.4 Error Metrics

### a. Intuition

Choosing the wrong metric can lead you to select a terrible model. Each metric has specific strengths:

- **MAE:** Average absolute error. Robust to outliers. Easy to explain.
- **MSE/RMSE:** Penalizes large errors more. Sensitive to outliers. RMSE is in same units as target.
- **MAPE:** Percentage error. Scale-independent. **Fails when target can be zero or near-zero.**
- **SMAPE:** Symmetric MAPE. More robust than MAPE but has its own quirks.

### b. Mathematical Insight

```
MAE  = (1/n) × Σ |y_t - ŷ_t|

MSE  = (1/n) × Σ (y_t - ŷ_t)²

RMSE = √MSE

MAPE = (100/n) × Σ |y_t - ŷ_t| / |y_t|
       (undefined when y_t = 0; biased toward under-forecasting)

SMAPE = (100/n) × Σ 2|y_t - ŷ_t| / (|y_t| + |ŷ_t|)
        (bounded in [0%, 200%])

MASE = MAE / MAE_naive
       (scale-free; benchmarks against naive forecast)
```

### e. Python Implementation

```python
import numpy as np
import pandas as pd
from sklearn.metrics import mean_absolute_error, mean_squared_error

def evaluate_forecast(actual: np.ndarray, 
                       forecast: np.ndarray,
                       model_name: str = 'Model') -> dict:
    """
    Comprehensive forecast evaluation with multiple metrics.
    
    Args:
        actual: Array of actual observed values
        forecast: Array of point forecasts
        model_name: Name for display
    
    Returns:
        Dict of metric values
    """
    actual   = np.array(actual)
    forecast = np.array(forecast)
    errors   = actual - forecast
    
    # Core metrics
    mae  = mean_absolute_error(actual, forecast)
    mse  = mean_squared_error(actual, forecast)
    rmse = np.sqrt(mse)
    
    # Percentage metrics (only when actual ≠ 0)
    nonzero_mask = actual != 0
    if nonzero_mask.sum() > 0:
        mape  = np.mean(np.abs(errors[nonzero_mask] / actual[nonzero_mask])) * 100
        smape = np.mean(2 * np.abs(errors) / (np.abs(actual) + np.abs(forecast))) * 100
    else:
        mape  = np.nan
        smape = np.nan
    
    # MASE: compare to naive forecast (shift by 1)
    naive_mae = mean_absolute_error(actual[1:], actual[:-1])
    mase      = mae / naive_mae if naive_mae > 0 else np.nan
    
    # Bias: positive = over-forecasting, negative = under-forecasting
    bias = errors.mean()
    
    metrics = {
        'Model': model_name,
        'MAE':   round(mae, 4),
        'RMSE':  round(rmse, 4),
        'MAPE':  round(mape, 2) if not np.isnan(mape) else 'N/A',
        'SMAPE': round(smape, 2) if not np.isnan(smape) else 'N/A',
        'MASE':  round(mase, 4) if not np.isnan(mase) else 'N/A',
        'Bias':  round(bias, 4)
    }
    
    print(f"\n=== {model_name} Forecast Evaluation ===")
    for k, v in metrics.items():
        if k != 'Model':
            print(f"  {k:10s}: {v}")
    
    return metrics


def compare_models(actuals: np.ndarray, forecasts_dict: dict) -> pd.DataFrame:
    """
    Compare multiple models side-by-side.
    """
    results = []
    for name, fcst in forecasts_dict.items():
        metrics = evaluate_forecast(actuals, fcst, model_name=name)
        results.append(metrics)
    
    df = pd.DataFrame(results).set_index('Model')
    print("\n=== Model Comparison ===")
    print(df.to_string())
    return df


# Demo
np.random.seed(42)
actual = np.array([100, 110, 105, 120, 115, 125, 130, 128, 140, 135, 150, 145])
forecasts = {
    'Naive':   np.array([100] + actual[:-1].tolist()),
    'HW':      actual + np.random.randn(12) * 5,
    'ARIMA':   actual + np.random.randn(12) * 3,
}

compare_models(actual, forecasts)
```

### f. Metric Selection Guide

| Situation | Recommended Metric |
|-----------|-------------------|
| Outliers present | MAE (not RMSE) |
| Large errors critical | RMSE (penalizes more) |
| Need % error, no zeros | MAPE |
| Zero values possible | SMAPE or MAE |
| Comparing across series | MASE or SMAPE |
| Business reporting | MAPE (intuitive) |
| Model selection | Multiple metrics + domain sense |

## 4.5 Baseline Comparison & Residual Diagnostics

### a. Intuition

A model that achieves 5% MAPE sounds impressive — until you realize the seasonal naive baseline achieves 4.5%. Always compare against baselines.

**Residual diagnostics** check whether the model has extracted all available signal. If residuals have autocorrelation, the model is leaving money on the table.

### b. Residual Checklist

```
✓ Zero mean (unbiased)
✓ No autocorrelation (white noise)
✓ Constant variance (homoskedastic)
✓ Approximately normal (for valid prediction intervals)
✓ No correlation with fitted values
```

### e. Python Implementation

```python
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
from statsmodels.stats.diagnostic import acorr_ljungbox
from scipy import stats

def residual_diagnostics_full(actual: pd.Series, 
                               forecast: pd.Series,
                               model_name: str = 'Model') -> dict:
    """
    Full residual diagnostics suite.
    """
    residuals = actual - forecast
    
    print(f"\n=== Residual Diagnostics: {model_name} ===")
    print(f"Mean residual:   {residuals.mean():.4f} (should be ~0)")
    print(f"Residual std:    {residuals.std():.4f}")
    print(f"Skewness:        {residuals.skew():.4f}")
    print(f"Kurtosis:        {residuals.kurtosis():.4f}")
    
    # Ljung-Box test
    lb = acorr_ljungbox(residuals.dropna(), lags=[5, 10, 20], return_df=True)
    print("\nLjung-Box p-values (>0.05 = no autocorrelation ✓):")
    print(lb['lb_pvalue'].to_string())
    
    # Normality
    _, p_norm = stats.shapiro(residuals.dropna()[:200])
    print(f"\nShapiro-Wilk p-value: {p_norm:.4f} "
          f"({'Normal ✓' if p_norm > 0.05 else 'Non-Normal (check outliers)'})")
    
    # Plots
    fig, axes = plt.subplots(2, 2, figsize=(14, 10))
    
    residuals.plot(ax=axes[0,0])
    axes[0,0].axhline(0, color='red', ls='--')
    axes[0,0].set_title('Residuals over Time')
    
    from statsmodels.graphics.tsaplots import plot_acf
    plot_acf(residuals.dropna(), lags=30, ax=axes[0,1])
    axes[0,1].set_title('ACF of Residuals')
    
    axes[1,0].hist(residuals.dropna(), bins=20, density=True, alpha=0.7)
    x = np.linspace(residuals.min(), residuals.max(), 100)
    axes[1,0].plot(x, stats.norm.pdf(x, residuals.mean(), residuals.std()), 'r-')
    axes[1,0].set_title('Residual Distribution')
    
    stats.probplot(residuals.dropna(), plot=axes[1,1])
    axes[1,1].set_title('Q-Q Plot')
    
    plt.suptitle(f'Residual Diagnostics: {model_name}', y=1.02)
    plt.tight_layout()
    plt.show()
    
    return {'mean': residuals.mean(), 'std': residuals.std(), 'lb_pvals': lb['lb_pvalue'].values}
```

```
      │
      ├─► Decomposition (Trend + Seasonal + Residual)
      │         │
      │         ├─► Is there seasonality? → m value → SARIMA seasonal period
      │         ├─► Is there trend? → d=1 or log transform
      │         └─► Residual structure → p, q order
      │
      ├─► ACF/PACF → p, q identification
      │
      ├─► Stationarity test → d determination
      │
      └─► Classical Models vs. Supervised ML Feature Engineering
```

## The Classical → ML Bridge

```
Classical ARIMA                    Supervised ML Equivalent
─────────────────                  ─────────────────────────
AR(p) component         ←→         Lag features [lag_1..lag_p]
d differencing          ←→         Differencing feature engineering
Seasonal AR(P)          ←→         Lag features at seasonal period
Trend component         ←→         time_index feature
Seasonal component      ←→         month_sin, month_cos, dummies
                        
ARIMA constraint:                  ML advantage:
  Linear relationships only          Nonlinear relationships (trees)
  Univariate only                    Multiple exogenous features
  Assumes normality                  Distribution-free
```

## Evaluation Connects Everything

```
Feature Engineering choices affect:
  → Model accuracy (better features = better MAE)
  → Residual autocorrelation (if features miss a pattern)
  → Training data size (more lags = fewer training rows)

Evaluation strategy validates:
  → Walk-forward CV mirrors production forecasting
  → Baseline comparison tests if complexity is justified
  → Residual diagnostics verify assumptions
```

| Generation | Methods | What They Fixed |
|------------|---------|-----------------|
| 1st: Simple baselines | Naive, Mean, Drift | Nothing — they ARE the baseline |
| 2nd: Smoothing | SES → Holt → Holt-Winters | Add components incrementally |
| 3rd: ARIMA family | AR → MA → ARIMA → SARIMA | Full statistical framework |
| 4th: ML on time series | Lag + rolling features + XGBoost | Nonlinearity + exogenous variables |

```python
def generate_retail_dataset(n_stores: int = 3, 
                             n_categories: int = 5,
                             weeks: int = 156,
                             seed: int = 42) -> pd.DataFrame:
    """
    Generate synthetic retail sales dataset.
    
    Simulates:
    - Weekly sales with trend + seasonality
    - Store-level and category-level effects
    - Holiday promotions
    - Random noise
    """
    np.random.seed(seed)
    
    stores     = [f'Store_{i+1}' for i in range(n_stores)]
    categories = [f'Cat_{c}' for c in ['Electronics', 'Clothing', 'Food', 'Home', 'Sports']][:n_categories]
    dates      = pd.date_range('2021-01-04', periods=weeks, freq='W-MON')
    
    rows = []
    for store in stores:
        store_multiplier = np.random.uniform(0.8, 1.4)
        
        for cat in categories:
            cat_base = np.random.randint(200, 1000)
            
            # Trend component
            trend = cat_base + np.arange(weeks) * np.random.uniform(0.5, 3.0)
            
            # Weekly seasonality (Christmas season in weeks 45-52)
            seasonal = np.zeros(weeks)
            for w in range(weeks):
                week_num = dates[w].isocalendar()[1]
                if 45 <= week_num <= 52:
                    seasonal[w] = cat_base * 0.4  # Holiday bump
                elif 20 <= week_num <= 30:
                    seasonal[w] = cat_base * 0.1 if cat == 'Cat_Sports' else -cat_base * 0.05
            
            # Promotion indicator (random 10% of weeks)
            promo = np.random.binomial(1, 0.1, weeks) * cat_base * 0.25
            
            # Noise
            noise = np.random.randn(weeks) * cat_base * 0.08
            
            # Final sales (floor at 0)
            sales = np.maximum(0, (trend + seasonal + promo + noise) * store_multiplier).astype(int)
            
            for i, (date, sale, p) in enumerate(zip(dates, sales, promo)):

                    'year':           date.year,
                })
    
    df = pd.DataFrame(rows)
    df = df.sort_values(['store', 'category', 'date']).reset_index(drop=True)
```

```python
def engineer_retail_features(df: pd.DataFrame) -> pd.DataFrame:
    """
    Create ML-ready features for retail forecasting.
    """
    df = df.sort_values(['store', 'category', 'date']).copy()
    df = df.set_index('date')
    
    feature_groups = []
    
    for (store, cat), group in df.groupby(['store', 'category']):
        group = group.copy().sort_index()
        s     = group['sales']
        
        # --- Lag features ---
        for lag in [1, 2, 3, 4, 8, 13, 26, 52]:
            group[f'lag_{lag}'] = s.shift(lag)
        
        # --- Rolling features (shift 1 to avoid leakage) ---
        shifted = s.shift(1)
        for w in [4, 8, 13, 26]:
            group[f'roll_{w}_mean'] = shifted.rolling(w, min_periods=1).mean()
            group[f'roll_{w}_std']  = shifted.rolling(w, min_periods=1).std()
            group[f'roll_{w}_max']  = shifted.rolling(w, min_periods=1).max()
            group[f'roll_{w}_min']  = shifted.rolling(w, min_periods=1).min()
        
        # --- Expanding mean ---
        group['expanding_mean'] = shifted.expanding(min_periods=4).mean()
        
        # --- Date-time features ---
        group['week_of_year'] = group.index.isocalendar().week.astype(int)
        group['month']        = group.index.month
        group['quarter']      = group.index.quarter
        group['year']         = group.index.year
        group['year_idx']     = group.index.year - group.index.year.min()
        
        # Cyclical encoding
        group['week_sin'] = np.sin(2 * np.pi * group['week_of_year'] / 52)
        group['week_cos'] = np.cos(2 * np.pi * group['week_of_year'] / 52)
        group['month_sin'] = np.sin(2 * np.pi * group['month'] / 12)
        group['month_cos'] = np.cos(2 * np.pi * group['month'] / 12)
        
        # Holiday flag (Christmas season)
        group['is_holiday_season'] = group['week_of_year'].between(45, 52).astype(int)
        
        # Store & category encoding (label encoding for tree models)
        group['store_id']    = hash(store) % 100
        group['category_id'] = hash(cat) % 100
        
        feature_groups.append(group)
    
    df_feat = pd.concat(feature_groups).sort_values(['store', 'category'])
    df_feat = df_feat.reset_index()
    df_feat = df_feat.dropna()
    
    print(f"Feature-engineered dataset: {df_feat.shape}")
    print(f"Features: {[c for c in df_feat.columns if c not in ['date','store','category','sales']]}")
    return df_feat
```

```python
def train_and_evaluate_retail(df_features: pd.DataFrame, 
                               forecast_horizon_weeks: int = 8) -> dict:
    """
    Train multiple models and evaluate using temporal split.
    
    Returns dict of model results.
    """
    feature_cols = [c for c in df_features.columns if c not in 
                    ['date', 'store', 'category', 'sales']]
    
    # Temporal split: last 8 weeks for test
    cutoff = df_features['date'].max() - pd.Timedelta(weeks=forecast_horizon_weeks)
    
    train_df = df_features[df_features['date'] <= cutoff].copy()
    test_df  = df_features[df_features['date'] >  cutoff].copy()
    
    X_train = train_df[feature_cols].values
    y_train = train_df['sales'].values
    X_test  = test_df[feature_cols].values
    y_test  = test_df['sales'].values
    
    print(f"Train size: {len(X_train)}, Test size: {len(X_test)}")
    
    # --- Models ---
    models = {
        'Ridge': Ridge(alpha=100.0),
        'RandomForest': RandomForestRegressor(n_estimators=200, max_depth=8, 
                                               min_samples_leaf=5, n_jobs=-1, random_state=42),
        'GradientBoosting': GradientBoostingRegressor(n_estimators=200, max_depth=4, 
                                                       learning_rate=0.05, random_state=42),
        'XGBoost': xgb.XGBRegressor(n_estimators=300, max_depth=5, learning_rate=0.05,
                                     subsample=0.8, colsample_bytree=0.8, 
                                     random_state=42, verbosity=0),
        'LightGBM': lgb.LGBMRegressor(n_estimators=300, max_depth=5, learning_rate=0.05,
                                        subsample=0.8, colsample_bytree=0.8,
                                        random_state=42, verbose=-1),
    }
    
    results = {}
    
    for name, model in models.items():
        # Ridge needs scaling
        if name == 'Ridge':
            scaler  = StandardScaler()
            X_tr    = scaler.fit_transform(X_train)
            X_te    = scaler.transform(X_test)
        else:
            X_tr, X_te = X_train, X_test
        
        model.fit(X_tr, y_train)
        preds = np.maximum(0, model.predict(X_te))  # Clip negative predictions
        
        mae    = mean_absolute_error(y_test, preds)
        mape   = np.mean(np.abs((y_test - preds) / (y_test + 1))) * 100
        rmse   = np.sqrt(np.mean((y_test - preds)**2))
        
        results[name] = {
            'model':  model,
            'preds':  preds,
            'MAE':    mae,
            'RMSE':   rmse,
            'MAPE':   mape,
        }
        
        print(f"{name:20s} MAE={mae:.1f}, RMSE={rmse:.1f}, MAPE={mape:.2f}%")
    
    # Comparison table
    comparison = pd.DataFrame({name: {k: v for k, v in vals.items() if k not in ['model','preds']}
                                for name, vals in results.items()}).T
    print("\n=== Model Comparison ===")
    print(comparison.sort_values('MAPE').to_string())
    
    return results, test_df, y_test
```

```python
from sklearn.model_selection import TimeSeriesSplit
import xgboost as xgb

def tune_xgboost_retail(df_features: pd.DataFrame) -> xgb.XGBRegressor:
    """
    Tune XGBoost using TimeSeriesSplit cross-validation.
    """
    feature_cols = [c for c in df_features.columns if c not in 
                    ['date', 'store', 'category', 'sales']]
    
    # Ensure temporal ordering
    df_sorted = df_features.sort_values('date')
    X = df_sorted[feature_cols].values
    y = df_sorted['sales'].values
    
    # TimeSeriesSplit
    tscv = TimeSeriesSplit(n_splits=5, gap=8)
    
    # Manual grid (use Optuna or Hyperopt in production)
    param_grid = [
        {'n_estimators': 200, 'max_depth': 4, 'learning_rate': 0.05, 'subsample': 0.8},
        {'n_estimators': 300, 'max_depth': 6, 'learning_rate': 0.05, 'subsample': 0.8},
        {'n_estimators': 300, 'max_depth': 4, 'learning_rate': 0.01, 'subsample': 0.9},
    ]
    
    best_mae  = np.inf
    best_params = None
    
    for params in param_grid:
        fold_maes = []
        
        model = xgb.XGBRegressor(**params, random_state=42, verbosity=0)
        
        for train_idx, val_idx in tscv.split(X):
            X_tr, X_val = X[train_idx], X[val_idx]
            y_tr, y_val = y[train_idx], y[val_idx]
            
            model.fit(X_tr, y_tr)
            preds = np.maximum(0, model.predict(X_val))
            fold_maes.append(mean_absolute_error(y_val, preds))
        
        mean_mae = np.mean(fold_maes)
        print(f"  Params {params} → CV MAE: {mean_mae:.2f} ± {np.std(fold_maes):.2f}")
        
        if mean_mae < best_mae:
            best_mae    = mean_mae
            best_params = params
    
    print(f"\nBest params: {best_params}")
    print(f"Best CV MAE: {best_mae:.2f}")
    
    # Retrain on all data with best params
    best_model = xgb.XGBRegressor(**best_params, random_state=42, verbosity=0)
    best_model.fit(X, y)
    
    return best_model
```

```python
import pandas as pd
import numpy as np

def generate_energy_dataset(years: int = 3, seed: int = 42) -> pd.DataFrame:
    """
    Generate synthetic hourly energy demand data.
    
    Captures:
    - Daily pattern (peaks at 8am and 6pm)
    - Weekly pattern (weekdays > weekends)
    - Annual seasonality (summer AC, winter heating)
    - Long-term trend (efficiency improvements → slight decline)
    - Temperature correlation
    - Random noise
    """
    np.random.seed(seed)
    
    hours  = years * 365 * 24
    dates  = pd.date_range('2021-01-01', periods=hours, freq='H')
    t      = np.arange(hours)
    
    # Annual seasonality (summer heat, winter cold → higher demand)
    annual = 300 * np.sin(2 * np.pi * (t / (24 * 365) - 0.1))  # Peak in summer
    
    # Daily pattern (two peaks: morning and evening)
    hour_of_day = dates.hour
    daily = (150 * np.sin(2 * np.pi * (hour_of_day - 6) / 24) +
             80  * np.sin(2 * np.pi * (hour_of_day - 18) / 24))
    
    # Weekly pattern (weekends lower by ~15%)
    is_weekend = (dates.dayofweek >= 5).astype(float)
    weekly     = -200 * is_weekend
    
    # Long-term trend (slight efficiency improvement)
    trend = -0.01 * t
    
    # Simulated temperature (correlated with demand)
    temperature = 15 + 15 * np.sin(2 * np.pi * t / (24 * 365)) + np.random.randn(hours) * 3
    temp_effect  = 50 * np.abs(temperature - 18)  # Comfort zone at 18°C
    
    # Base demand
    base_demand = 2800
    
    # Random noise
    noise = np.random.randn(hours) * 50
    
    # Final demand (floor at 1000)
    demand = np.maximum(1000, base_demand + trend + annual + daily + weekly + 
                         0.3 * temp_effect + noise)
    
    df = pd.DataFrame({
        'timestamp':   dates,
        'demand_mwh':  demand.round(1),
        'temperature': temperature.round(1),
        'hour':        dates.hour,
        'day_of_week': dates.dayofweek,
        'month':       dates.month,
        'is_weekend':  (dates.dayofweek >= 5).astype(int),
    })
    
    return df.set_index('timestamp')
```

#### Step 4: Feature Engineering

```python
def engineer_energy_features(df: pd.DataFrame, forecast_horizon: int = 48) -> pd.DataFrame:
    """
    Feature engineering for hourly energy demand forecasting.
    Forecast horizon = 48 hours → features must not use data from t to t+47.
    """
    df = df.copy().sort_index()
    s  = df['demand_mwh']
    
    # Lag features (minimum lag = forecast_horizon to avoid leakage)
    for lag in [48, 49, 50, 72, 96, 168, 336]:  # 2d, 3d, 4d, 7d, 14d ahead
        df[f'lag_{lag}'] = s.shift(lag)
    
    # Rolling features (shifted by horizon)
    shifted = s.shift(forecast_horizon)
    for window in [24, 48, 168]:
        df[f'roll_{window}_mean'] = shifted.rolling(window, min_periods=1).mean()
        df[f'roll_{window}_std']  = shifted.rolling(window, min_periods=1).std()
        df[f'roll_{window}_max']  = shifted.rolling(window, min_periods=1).max()
    
    # Expanding mean (historical average)
    df['expanding_mean'] = shifted.expanding(min_periods=24).mean()
    
    # Cyclical date-time features
    df['hour_sin']  = np.sin(2 * np.pi * df.index.hour / 24)
    df['hour_cos']  = np.cos(2 * np.pi * df.index.hour / 24)
    df['dow_sin']   = np.sin(2 * np.pi * df.index.dayofweek / 7)
    df['dow_cos']   = np.cos(2 * np.pi * df.index.dayofweek / 7)
    df['month_sin'] = np.sin(2 * np.pi * df.index.month / 12)
    df['month_cos'] = np.cos(2 * np.pi * df.index.month / 12)
    
    # Raw time features (useful for tree models)
    df['hour']        = df.index.hour
    df['day_of_week'] = df.index.dayofweek
    df['month']       = df.index.month
    df['year']        = df.index.year
    
    # Temperature lags (if available)
    if 'temperature' in df.columns:
        for lag in [48, 168]:
            df[f'temp_lag_{lag}'] = df['temperature'].shift(lag)
    
    df = df.dropna()
    print(f"Energy features created: {df.shape}")
    return df

df_energy_feat = engineer_energy_features(df_energy, forecast_horizon=48)
```

# 8. Common Mistakes & Pitfalls

## 1. Data Leakage (The Most Costly Mistake)

**What happens:** Rolling features or lags are computed without shifting, so features at time t include data from t or later.

**Example:** `df['rolling_7_mean'] = df['sales'].rolling(7).mean()` — this includes `y_t` in the feature for predicting `y_t`.

**Fix:** Always `df['sales'].shift(1).rolling(7).mean()` — shift before rolling.

## 2. Random Train-Test Split

**What happens:** Data is shuffled, model sees future data during training, evaluation is meaninglessly optimistic.

**Fix:** Use `temporal_train_test_split()` or `TimeSeriesSplit` from sklearn.

## 3. No Baseline Comparison

**What happens:** You report 8% MAPE and call it a win, not knowing seasonal naive achieves 6%.

**Fix:** Always compute seasonal naive as minimum viable baseline before fitting any model.

## 4. Testing for Stationarity Only with ADF

**What happens:** ADF has low power in small samples and specific failure modes.

**Fix:** Use both ADF and KPSS (opposite null hypotheses). If they disagree, investigate further.

## 5. Over-Differencing

**What happens:** Applying d=2 when d=1 is sufficient. Over-differencing inflates model complexity and hurts forecast accuracy.

**Fix:** Test after each difference. Stop at the first order where the series becomes stationary.

## 6. Ignoring Structural Breaks

**What happens:** Training model on pre-COVID data and using it to forecast post-COVID demand. The distribution has fundamentally shifted.

**Fix:** Plot rolling statistics. If you see clear regime changes, train only on the current regime.

## 7. Using MAPE with Near-Zero Values

**What happens:** MAPE = |actual - forecast| / |actual| explodes when actual is near 0, producing misleading metrics.

**Fix:** Use MAE, SMAPE, or MASE when series contains zeros or near-zeros.

## 8. Multi-Step Forecasting with Recursive Strategy on Long Horizons

**What happens:** Recursive forecasting compounds errors — each step uses a prediction (not actual) as input. Over 50+ steps, errors accumulate catastrophically.

**Fix:** Use direct strategy for horizons > 10-15 steps. Train one model per horizon.

## 9. Choosing Window Size Arbitrarily

**What happens:** Rolling window of 7 days for monthly data, or 30 days for data with a 7-day cycle — misses the actual periodicity.

**Fix:** Derive window sizes from ACF analysis and domain knowledge. Window size should align with the main seasonal periods.

## 10. Forgetting to Handle Missing Timestamps Before Feature Engineering

**What happens:** Feature engineering (lags, rolling) silently skips missing dates, creating wrong lag alignments.

**Fix:** Always reindex to a complete time grid and impute before creating any features.

# 9. Interview Preparation

## Conceptual Questions

**Q1: What is the difference between seasonality and cyclicity?**

Seasonality has a **fixed, known period** tied to the calendar (every 12 months, every 7 days). Cyclicity has a **variable, unknown period** driven by economic or business cycles that repeat over multi-year horizons. We can model seasonality explicitly; cyclicity is harder to capture and often ends up in the residual.

**Q2: Why must time series data not be shuffled for train-test splits?**

Shuffling violates the temporal ordering that the series depends on. A model trained on shuffled data can see "future" values during training, making it appear to perform well while having no real predictive power. The model learns the distribution, not the temporal dependencies.

**Q3: When would you prefer SARIMA over Holt-Winters?**

Both handle trend and seasonality. Prefer **SARIMA** when you need formal statistical inference (confidence intervals, hypothesis tests about parameters), when you want to diagnose the exact AR and MA structure, or when you need to incorporate exogenous variables via SARIMAX. Prefer **Holt-Winters** when you want a simpler, robust model that's faster to fit and easier to explain to non-technical stakeholders.

**Q4: What does an ACF that cuts off at lag q mean? What does it imply about the model?**

It means the series has a **MA(q) structure** — only the first q lags of the error term are relevant. Beyond q, the autocorrelation is zero. The data-generating process is: current value = mean + weighted sum of the last q errors. PACF for a MA process tails off rather than cutting off.

**Q5: What are the consequences of using MAPE when the target contains zeros?**

MAPE divides by the actual value. Division by zero or values near zero produces astronomically large (or undefined) percentage errors, making the metric meaningless. MAPE also systematically penalizes over-forecasting more than under-forecasting (asymmetric). Use SMAPE or MAE for data with zeros.

**Q6: What is the difference between rolling window and expanding window cross-validation for time series?**

- **Expanding (anchored):** Training set grows with each fold. Models see all available history. Good when older data is still relevant.
- **Rolling (sliding):** Training set stays fixed size. Only recent data is used. Good when the time series has non-stationary properties or when older data is irrelevant (distribution shift).

**Q7: Explain why the "I" in ARIMA stands for "Integrated."**

An **Integrated** process is one where differencing is needed to achieve stationarity. Specifically, a series is integrated of order d (written I(d)) if it needs to be differenced d times to become stationary. An ARIMA(p,d,q) model first differences the series d times, then fits an ARMA(p,q) to the result. The name comes from the fact that a differenced stationary series must be "integrated" (summed/cumulated) back to produce the original series.

## Scenario-Based Questions

**Q8: You fit an ARIMA model and the residuals still show significant autocorrelation at lag 12. What do you do?**

The model hasn't captured the seasonal pattern. I would:
1. Check if lag 12 is the seasonal period (monthly data? → m=12)
2. Add seasonal differencing (D=1) to remove seasonal unit root
3. Upgrade to SARIMA(p,d,q)(P,1,Q)₁₂ and check if residual autocorrelation disappears
4. Re-run Ljung-Box test to verify residuals are white noise

**Q9: Your model performs well on the training set but much worse on the test set — what could be wrong?**

Several possibilities:
1. **Data leakage:** Features computed without proper shifting, allowing future information into training
2. **Overfitting:** Model too complex for the data (high p,q orders in ARIMA; too many lags in ML model)
3. **Distribution shift:** The test period has a different distribution (structural break, COVID, holiday anomaly)
4. **Wrong validation strategy:** Random split was used; switch to temporal split
5. **Over-tuned:** Hyperparameters tuned on the test set (or a proxy of it)

**Q10: A stakeholder asks why MAPE for your model is 4%, but the seasonal naive baseline achieves 3.5%. How do you proceed?**

Honestly: the model is not adding value over the baseline. I would:
1. Acknowledge the finding immediately — don't hide it
2. Investigate: is the feature engineering correct? Are we missing a key predictor?
3. Try more powerful models (LightGBM with more features)
4. Consider if the 0.5% gap matters financially — if 4% vs 3.5% MAPE is not practically significant, use the simpler baseline in production
5. Report both models to stakeholders with their trade-offs

**Q11: How would you forecast hourly energy demand for 72 hours ahead in production?**

1. Use **direct multi-step strategy**: one model per horizon step (h=1, 2, ..., 72)
2. Features: lag_72, lag_73, ..., lag_168 (safe for all horizons), rolling statistics from t-72 onward, calendar features (hour_sin/cos, day_sin/cos, month_sin/cos), weather forecast features
3. Validate using walk-forward CV with 72-hour test windows and 1-week step
4. Deploy as hourly batch job: runs at minute 0 of each hour, produces 72 predictions
5. Monitor: compute MAPE on realized vs predicted for previous 72 hours; alert if threshold exceeded

```
ACF/PACF Pattern Identification:
  AR(p)   → PACF cuts at p, ACF tails off
```
