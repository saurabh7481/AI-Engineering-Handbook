Environment Setup
Choose modern tooling. Polars is 5–20× faster than pandas for large datasets, uses Apache Arrow memory format, and supports lazy evaluation. Plotly produces interactive charts natively.

polars

Rust-based DataFrame library. LazyFrame API applies predicate/projection pushdown. Use for any dataset over 100K rows. Full Pandas compatibility via .to_pandas().

plotly

Interactive, publication-quality charts. Exports to HTML, PNG, SVG. Supports scatter, box, violin, heatmap, 3D, geographic, and custom charts natively.

ydata-profiling

One-line automated EDA report with distributions, correlations, missing values, and interactions. Best for initial exploration before custom analysis.

scipy + statsmodels

Statistical tests (normality, ANOVA, chi-square), seasonal decomposition, ACF/PACF, VIF. Essential for rigorous statistical reasoning.

scikit-learn

PCA, t-SNE, Isolation Forest, mutual information, feature importance, StandardScaler. The ML glue for dimensionality reduction and anomaly detection.

missingno

Matrix, bar, and heatmap visualizations of missing data patterns. Reveals whether missingness is MCAR, MAR, or MNAR at a glance.

03
Missing Value Analysis
Missing data is never truly "missing" — it carries information. The pattern of missingness tells a story. Before imputing, understand why values are absent.

MCAR — Missing Completely At Random

Missingness is independent of any variable. A random sensor failure. Strategy: Listwise deletion is valid. Simple imputation (mean/median) works. Little's MCAR test: p > 0.05.

MAR — Missing At Random

Missingness depends on other observed variables. Survey non-response correlated with age. Strategy: Multiple imputation (MICE), model-based imputation, KNN imputation.

MNAR — Missing Not At Random

Missingness depends on the unobserved value itself. High earners skip income questions. Strategy: Domain-specific logic, add "is_missing" indicator feature, sensitivity analysis.

Reading Distribution Shapes
  • Normal bell curve: Symmetric, single peak. Mean ≈ median ≈ mode. Most ML algorithms assume this.
  • Right skew (positive): Long tail to the right. Mean > median. Common in: prices, income, wait times. Apply log(1+x) transform.
  • Bimodal (two peaks): Often two subpopulations mixed. Split by segment, or add a cluster label feature.
  • Uniform: All values equally likely. Could be: random IDs, index columns. Usually not informative.
  • Spike at zero: Zero-inflated distribution (e.g., tip_amount for cash rides). Model separately or use zero-inflation technique.

Outlier Detection
Outliers can be errors (remove), rare events (keep + flag), or the signal itself (anomaly detection tasks). Never blindly delete — always investigate first.
PYTHONoutliers.py
# ── Method 1: IQR Fences (Tukey) ── def iqr_outliers(df: pl.DataFrame, col: str) -> dict: q1, q3 = df[col].quantile(0.25), df[col].quantile(0.75) iqr = q3 - q1 lower, upper = q1 - 1.5 * iqr, q3 + 1.5 * iqr n_out = df.filter((pl.col(col) < lower) | (pl.col(col) > upper)).height return {"lower": lower, "upper": upper, "n_outliers": n_out, "pct": n_out / df.height * 100} # ── Method 2: Z-Score (assumes normality) ── def zscore_outliers(df: pl.DataFrame, col: str, threshold: float = 3.0): mean, std = df[col].mean(), df[col].std() z = ((df[col] - mean) / std).abs() return df.filter(z > threshold) # threshold=3 → 0.3% false positive rate # ── Method 3: Modified Z-Score (robust, uses MAD) ── # Use this when data is skewed (preferred over standard Z-score) def modified_zscore(df: pl.DataFrame, col: str, threshold: float = 3.5): median = df[col].median() mad = (df[col] - median).abs().median() mz = (0.6745 * (df[col] - median) / mad).abs() return df.filter(mz > threshold) # ── Method 4: Isolation Forest (multivariate, non-parametric) ── from sklearn.ensemble import IsolationForest pdf = df.select(num_cols).drop_nulls().to_pandas().sample(50000) iforest = IsolationForest(n_estimators=100, contamination=0.05, random_state=42, n_jobs=-1) labels = iforest.fit_predict(pdf) # -1 = outlier, 1 = normal scores = iforest.decision_function(pdf) # more negative = more anomalous pdf['is_outlier'] = labels == -1 pdf['anomaly_score'] = scores # ── Method 5: Local Outlier Factor (density-based) ── from sklearn.neighbors import LocalOutlierFactor lof = LocalOutlierFactor(n_neighbors=20, contamination=0.05) lof_labels = lof.fit_predict(pdf[num_cols]) # Better for clusters of different densities # ── Visualize: standardized box plots ── from sklearn.preprocessing import StandardScaler X_scaled = StandardScaler().fit_transform(pdf[num_cols]) pdf_scaled = pd.DataFrame(X_scaled, columns=num_cols) pdf_melted = pdf_scaled.melt(var_name="Feature", value_name="Standardized") fig_box = px.box(pdf_melted, x="Feature", y="Standardized", title="Outlier Box Plots (standardized)", points="outliers")

Keep the outlier if…

It represents a real, valid extreme event. E.g., a $200 taxi fare for a JFK→upstate trip. Flag it with an indicator feature is_high_fare=1 instead of deleting.

Remove the outlier if…

It's clearly a data error: negative fare_amount, trip_distance = 5000 miles, year 1970 timestamp. Log all removals and their justification in your EDA notes.

Winsorize/Cap if…

Outlier is real but disproportionately influences models. Cap at 99th percentile: df[col].clip(upper=df[col].quantile(0.99)). Common in price/revenue data.

  • Friday & Saturday nights: 2–4× higher trip volume 10 PM–2 AM vs weekdays

Target Variable Analysis
When a target exists, shift from descriptive to predictive EDA. Measure each feature's relevance to the target using model-free (MI) and model-based (RF) methods.
PYTHONtarget_analysis.py
# ── Mutual Information: captures ANY kind of dependency ── from sklearn.feature_selection import mutual_info_regression, mutual_info_classif # Create target: tip percentage df_clean = df.filter( (pl.col('fare_amount') > 2) & (pl.col('fare_amount') < 200) & (pl.col('trip_distance') > 0) & (pl.col('trip_distance') < 50) ).with_columns( (pl.col('tip_amount') / pl.col('fare_amount') * 100).alias('tip_pct') ) feature_cols = ['trip_distance', 'passenger_count', 'hour', 'weekday'] pdf = df_clean.select(feature_cols + ['tip_pct']).drop_nulls().to_pandas() pdf = pdf.sample(50000, random_state=42) mi_scores = mutual_info_regression(pdf[feature_cols], pdf['tip_pct']) mi_df = pd.DataFrame({"feature": feature_cols, "mi_score": mi_scores}) mi_df = mi_df.sort_values("mi_score", ascending=False) fig_mi = px.bar(mi_df, x="mi_score", y="feature", orientation='h', title="Mutual Information: Feature → Target", color="mi_score", color_continuous_scale="Blues") # ── Random Forest importance ── from sklearn.ensemble import RandomForestRegressor from sklearn.inspection import permutation_importance rf = RandomForestRegressor(n_estimators=100, n_jobs=-1, random_state=42, max_depth=5) rf.fit(pdf[feature_cols], pdf['tip_pct']) # MDI importance (fast, slightly biased) mdi_imp = pd.DataFrame({"feature": feature_cols, "importance": rf.feature_importances_}) # Permutation importance (slower, unbiased — preferred) perm = permutation_importance(rf, pdf[feature_cols], pdf['tip_pct'], n_repeats=10, random_state=42) perm_df = pd.DataFrame({"feature": feature_cols, "perm_mean": perm.importances_mean, "perm_std": perm.importances_std}) # ── Target vs each feature ── for feat in feature_cols: pdf_temp = pdf.copy() pdf_temp["bin"] = pd.cut(pdf_temp[feat], bins=10) grouped = pdf_temp.groupby("bin", observed=True)['tip_pct'].agg(['mean','count']) fig = px.bar(grouped.reset_index(), x="bin", y="mean", title=f"Mean tip_pct by {feat} bin")

Data Quality Audit
Industrial datasets are always messy. A rigorous quality audit prevents "garbage in, garbage out" — and catches issues that can silently corrupt your model.

# ── 1. Duplicates ── full_dups = df.height - df.unique().height print(f"Full duplicates: {full_dups:,} ({full_dups/df.height*100:.2f}%)") # Key-based duplicates (same trip start/end time + location) key_cols = ['tpep_pickup_datetime', 'PULocationID', 'DOLocationID'] key_dups = df.height - df.unique(subset=key_cols).height print(f"Key duplicates: {key_dups:,}") # ── 2. Range validation ── range_rules = { 'fare_amount': (0, 1000), 'trip_distance': (0, 200), 'passenger_count': (1, 8), 'tip_amount': (0, 300), } for col, (lo, hi) in range_rules.items(): if col in df.columns: n = df.filter((pl.col(col) < lo) | (pl.col(col) > hi)).height print(f"{col} violations: {n:,} ({n/df.height*100:.2f}%)") # ── 4. Zero-variance columns ── zero_var = [c for c in df.columns if df[c].dtype in (pl.Float64, pl.Int64) and df[c].n_unique() == 1] print(f"Zero variance cols: {zero_var}") # → drop these # ── 5. Generate clean dataset ── df_clean = (df .filter( (pl.col('fare_amount').is_between(0.5, 500)) & (pl.col('trip_distance').is_between(0.01, 100)) & (pl.col('passenger_count').is_between(1, 8)) ) .drop_nulls(subset=['fare_amount', 'trip_distance']) ) print(f"After cleaning: {df_clean.height:,} rows ({df_clean.height/df.height*100:.1f}% retained)")

Returns a dict of all figures and statistics. """ Path(output_dir).mkdir(exist_ok=True) results = {} # ── Auto-detect column types ── num_cols = [c for c in df.columns if df[c].dtype in (pl.Float32, pl.Float64, pl.Int32, pl.Int64) and c not in [target_col]] cat_cols = [c for c in df.columns if df[c].dtype == pl.Utf8 or df[c].n_unique() < 30] # 1. Industrial EDA Checklist
Print this and check off each item before calling EDA "complete." Every unchecked item is a potential blind spot in your analysis.

Ingestion & Shape

  • Rows, columns, memory usage reported
  • Schema reviewed (dtypes, column names)
  • Sample rows inspected (head + tail)
  • Cardinality of every column measured
  • Semantic type detection run
  • Lazy loading used for large files

Data Quality

  • Missing values: count, %, pattern
  • MCAR/MAR/MNAR classification attempted
  • Full and key-based duplicates checked
  • Impossible values detected (negatives, future dates)
  • Value range validation against domain rules
  • Zero-variance columns identified
  • String quality issues checked (whitespace, empty)
  • Type mismatches checked (numbers stored as strings)

Univariate Analysis

  • Distribution histograms for all numeric cols
  • Five-number summary + skew + kurtosis
  • Normality tests (Shapiro / D'Agostino / KS)
  • QQ plots for key numeric features
  • Box / violin plots for spread visualization
  • Frequency tables for all categoricals
  • Pareto charts for high-cardinality categoricals

Bivariate Analysis

  • Pearson correlation matrix (numeric)
  • Spearman correlation (non-normal / robust)
  • Scatter / hexbin for dense pairs
  • Group stats for all numeric × categorical pairs
  • ANOVA / Kruskal-Wallis test with effect size
  • Chi-square + Cramér's V for cat × cat
  • Contingency heatmaps for key pairs

Multivariate Analysis

  • PCA: scree plot, loadings, biplot
  • t-SNE or UMAP for cluster structure
  • Parallel coordinates for multi-dim patterns
  • VIF computed (multicollinearity check)
  • Scatter plot matrix (SPLOM)

Outlier Detection

  • IQR fences computed for all numeric cols
  • Z-score / Modified Z-score (MAD)
  • Isolation Forest (multivariate)
  • Each outlier investigated: error vs real?
  • Outlier treatment decision documented

Temporal (if datetimes exist)

  • Temporal features extracted (hour, day, month)
  • Time series plot with rolling stats
  • Hour × day-of-week heatmap
  • Seasonal decomposition (trend + seasonal + residual)
  • ACF / PACF plots
  • Data drift checked across time periods

Target Analysis

  • Class balance (classification) / target distribution
  • Imbalance ratio computed
  • Mutual information: all features vs target
  • Random Forest permutation importance
  • Target vs each feature plotted
  • Point-biserial for binary features

Documentation & Handoff

  • Every anomaly noted with row count
  • Data quality issues list created for team
  • Feature engineering ideas logged
  • Columns recommended for dropping
  • Surprising findings highlighted
  • EDA summary report written
  • ydata-profiling HTML report generated

Geospatial (if lat/lon exist)

  • Coordinate validity checked (within bounds)
  • Density heatmap generated
  • Scatter map with key metric as color
  • Centroid and spatial spread computed
  • Zone-level aggregations analyzed

EDA is iterative, not linear. Start with automated profiling to find the most interesting leads. Then drill deep on those. Each finding suggests new questions. You're done when you can explain the dataset's key characteristics, quirks, and predictive signals to a colleague without referring to your notes.
